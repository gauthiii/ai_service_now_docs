# Perform Bulk Assessment in Stacked View

Perform assessments on multiple risks and controls simultaneously in a risk assessment project using AI Risk and Compliance Workspace. You can assess inherent risks, effectiveness of controls, residual risks, and target risks. You can also define risk responses that enable you to manage and mitigate the risks identified during the risk assessment process.

## What Is the Stacked View?

The stacked view is a mode within a risk assessment project that lets you assess multiple risks and controls together in a single, consolidated assessment experience, covering inherent, control, and residual assessments along with risk response strategies.

## About This Task

You can perform the bulk risk assessment from either of the following:

- Employee Center
- AI Control Tower Workspace

## Procedure

### 1. Open the project from the Employee Center

To perform bulk risk assessment from the Employee Center:

1. Navigate to **All > Self-Service > Employee Center**.
2. On the Employee Center landing page, select **GRC Tasks**.
3. Select **Open** and navigate to **Bulk risk assessment**.
4. Open the risk assessment project that you want to assess.

### 2. (Alternative) Open the project from the AI Control Tower Workspace

To perform bulk risk assessment from the AI Control Tower Workspace:

1. Navigate to **All > AI Control Tower**.
2. Open the risk assessment project either from the **Tasks** list or from the **AI asset** record.

### 3. Select the stacked view

Select **Stacked view** to assess the risk assessment project.

### 4. Open the assessment

Select **Open assessment**.

> If more than one assessor is defined in the **Stakeholder** section, the assessor who opens the assessment first becomes the current assessor of the risk assessment project.

### 5. (Optional) Reassign the project

To reassign the project, select the **more actions** icon, and select **Reassign**.

### 6. (Optional) Add risks to the project

You can do any of the following:

- **Create from risk statements** — To create risks from the risk statement, select the **more actions** icon, and select **Create from risk statements**.
- **Create ad hoc risk** — To add a risk that is not in the library, select the **more actions** icon, and select **Create ad hoc risk**.
- **Add risk** — To add any underlying risks that have been removed, select the **more actions** icon, and select **Add risk**.

> The risks with an in-progress assessment can't be added as part of the risk assessment project.

### 7. (Optional) Remove risks from the project

> Removed scoped risks remain part of the project but are marked as not applicable for reporting purposes. However, removed ad hoc risks are completely deleted.

1. Select **Actions**.
2. Select **Remove risk**. The **Remove risk** dialog box appears.
3. In the **Remove risk** dialog box, provide justification and select **Remove**. The selected risk is deleted, along with all associated responses.

### 8. Respond to assessment factors

Respond to all the factors in the **Inherent**, **Control**, and **Residual** assessments, and define a risk response strategy for each risk within the project.

> **Note:** The application automatically saves your responses and calculates the overall risk score.

### 9. View the assessment summary

To view the complete summary of the assessment, select **View summary**.

### 10. Validate the assessment

To check for any errors in the assessment before you submit it, select the **more actions** icon, and select **Validate**.

You can see the list of errors, if any, that must be resolved in the side panel for the selected risk.

### 11. Resolve errors

Identify the errors and resolve them.

### 12. Re-validate

Select **Validate** to check if the errors have been resolved.

### 13. Submit for approval

To submit the risk assessment project for approval, select **Submit**, and select **Confirm Submission**.

> Assessments are validated again in this stage. Review the assessment summary before you submit. **You can't make any changes after submission.**

## Results

The risk assessment project is submitted for approval, with inherent, control, residual, and target risks assessed and risk response strategies defined for each risk. The application saves responses automatically and calculates the overall risk score.
