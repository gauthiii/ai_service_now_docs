# Configuring MCP Server Console

_Source: ServiceNow Now Assist documentation, pages 1033-1043._

## Overview

Configuring MCP Server Console means creating a Model Context Protocol (MCP) server and configuring the tools and inputs it exposes to MCP clients.

**Configuration overview:**

1. **Create a Model Context Protocol server** — An AI administrator creates a server and adds tools to the server.
2. **Create a tool for a Model Context Protocol server** — If additional tools are needed, the AI administrator identifies which functionality to expose and creates tools based on Now Assist skills (and other capabilities). From the tools, they configure which fields are exposed to clients as tool inputs.

After configuring a server, the AI administrator creates an OAuth inbound integration for each client and configures clients to connect to the server using the server and authentication details. For more information, see *Connecting to an MCP server from an MCP client*.

This page also covers the preconfigured **Quickstart Server**, which helps you get started learning about MCP servers and connecting MCP clients to a ServiceNow instance.

## Features

- Create one or more MCP servers that expose different tools for different use cases (such as HR or IT workflows) or for different clients.
- Use a preconfigured Quickstart Server with ready-made incident and case tools.
- Create tools from multiple categories: REST API, Action, Knowledge graph, Subflow, and Now Assist skill.
- Configure which capability fields are exposed to clients as tool inputs (enable or disable individual inputs).
- Save servers as drafts and activate or deactivate them later.
- Explore search, refresh, and many other page view options on the server pages.

### Quickstart Server tools

The Quickstart Server includes tools for looking up and summarizing incident and case records.

| Name | Description |
| --- | --- |
| Case summarization | Get a summary of details in a case record generated by Now Assist. |
| Incident summarization | Get a summary of details in an incident record generated by Now Assist. |
| Look up Case Records | Look up case records to enable summarizing them with the Case Summarization tool. |
| Look up Incident Records | Look up incident records to enable summarizing them with the Incident summarization tool. |

## Functionalities

### Tool categories

Tools define which functionality and data a server exposes to clients and the actions that can be performed on an instance by clients. Tools are based on capabilities, such as Knowledge Graph, Subflows, Action, REST APIs, and Now Assist skills (including custom skills created with Now Assist Skill Kit). Tools include inputs that correspond to the fields of the existing capability; inputs that are enabled for a tool are exposed to clients.

Tools can be created from these categories:

- **REST API**
- **Action**
- **Knowledge graph**
- **Subflow**
- **Now Assist skill**

> **Note:** The minimum version required is Zurich patch 9 and Australia patch 2.

> **Note:** If you add inputs to a Now Assist skill after a tool has been created, you must create another tool to include the additional inputs and replace the existing tool.

### Roles required

- **Create a server:** `sn_mcp_server.admin` or `admin`
- **Create a tool (all tool categories):** `sn_mcp_server.tools_admin`, `sn_mcp_server.admin`, or `admin`

### Server form

| Field | Description |
| --- | --- |
| Label | An internal name of your choice, for the server. |
| Name | An auto-created server name derived from the label you chose. This is used in the URL endpoint. The server URL is created accordingly. |
| Short description | A description of the capabilities and purpose of the server. |
| Tools | The tools that the server exposes. |

After creating the server, the **Server URL** field is populated with the endpoint that you use to integrate with clients:

```
https://<instance>.service-now.com/sncapps/mcp-server/mcp/<server-name>
```

### Tool form (common fields)

The following fields appear when creating a tool from any category (Subflow, Action, REST API, Knowledge Graph, or Now Assist skill). The category-specific source field is named for the category (for example, **Subflow**, **Action**, **REST API**, **Knowledge Graph**, or **Now Assist skill**).

| Field | Description |
| --- | --- |
| `<Category>` (Subflow / Action / REST API / Knowledge Graph / Now Assist skill) | Select a value of that type from the list. For Knowledge Graph, you can choose either Enterprise Graph or one of the custom graphs. |
| Label | An internal name for the tool. |
| Description | The description of what the tool intends to do. This input is exposed to AI clients and used to determine when to call this tool. **Note:** Admins must add a specific and action-oriented description as the AI clients access it to decide when to invoke the tool. |
| MCP Servers | One or more servers you want to add your tool to. |

> **Note:** The category is auto-populated if selected in the last modal.

In the **Tool inputs** section, the fields associated with the capability are added. The tool is then published on the MCP Server and discoverable by MCP clients.

### Knowledge Graph tool inputs

When creating a tool from Knowledge Graph, the Tool inputs section includes these inputs:

- **anchortables** — Use one or more table names to make your query more specific. Helpful when you know which tables are relevant to your question.
- **tags** — Include one or more tags to prioritize a specific group of tables (for example, `CSM`, `HR`). These tags are required for the Enterprise Graph.
- **description** — Your query or request that will use Knowledge Graph.
- **apioptions**

## Instructions / Procedures

### Create a Model Context Protocol server

**Role required:** `sn_mcp_server.admin` or `admin`

**About this task:** Depending on your requirements, you can create one or more servers that expose different tools for different use cases, such as for HR or IT workflows, or for different clients. You can also use the preconfigured Quickstart Server. This page also provides explore search, refresh, and many other page view options.

**Procedure:**

1. Navigate to **All > MCP Server Console** or **Admin > MCP Server Console**.
2. From the **Configuration** tab, select **Servers**.
3. Select **Create server**.
4. On the form, fill in the fields (see the Server form table above).
5. From the **Tools** section, add the tools that the server exposes.
   - a. Select **Add tools**.
   - b. Search for tools from the list and select the tools to expose.
     - If you need to create a tool, select **Create tool**. For more information, see *Create a tool for a Model Context Protocol server*.
   - c. Select **Add**.
6. Select **Create** to make the server active.

> **Note:** To save your changes but not make the server active yet, select **Save as draft** instead. To activate it later, you can select **Activate** from the server record or the Servers page.

After creating the server, the **Server URL** field is populated with the endpoint that you use to integrate with clients: `https://<instance>.service-now.com/sncapps/mcp-server/mcp/<server-name>`.

**Trouble?** If you need to turn off access to an existing server, select **Deactivate** from the server record or the Servers page.

**What to do next:** Configure clients to connect to the server. For more information, see *Connecting to an MCP server from an MCP client*.

### Create a tool for a Model Context Protocol server

**Before you begin:** If you aren't using the Quickstart Server, create a server to which you can add tools.

**Role required:** `sn_mcp_server.tools_admin`, `sn_mcp_server.admin`, or `admin`

**About this task:** Each server must include at least one tool. Tools are based on capabilities such as Knowledge Graph, Subflows, Action, REST APIs, and Now Assist skills (including custom skills created with Now Assist Skill Kit). For a list of Now Assist skills that can be used as tools, see *Now Assist skill support in MCP Server Console*.

**Procedure:**

1. Navigate to **All > MCP Server Console** or **Admin > MCP Server Console**.
2. From the **Configuration** tab, select **Tools**.
   - If you add inputs to a Now Assist skill after a tool has been created, you must create another tool to include the additional inputs and replace the existing tool.
3. View and manage the tools created from various categories, and their associated attributes.
4. Select **Create tool**.
5. Select a tool type you want to create from these categories: **REST API**, **Action**, **Knowledge graph**, **Subflow**, and **Now Assist skill**. The steps are similar for any category you use for tool creation.
6. See one of these to learn more:
   - Create a tool from Subflow
   - Create a tool from REST API
   - Create a tool from Knowledge Graph
7. **Optional:** Turn off inputs from the tool that you don't want to expose.
   - a. In the **Tool inputs** section, locate the tool input.
   - b. From the **Enabled** column, select the toggle to turn off the input.

   > **Note:** Some tool inputs are required and can't be turned off.
8. Select **Create**.

**What to do next:** Configure clients to connect to the server and use the tool. For more information, see *Connecting to an MCP server from an MCP client*.

**Related topics:** Now Assist skills; Now Assist Skill Kit.

### Create a tool from Subflow

Subflows and actions empower agents to complete tasks seamlessly — from submitting requests to routing for approval and confirming outcomes across workflows, without leaving the client interface.

**Role required:** `sn_mcp_server.tools_admin`, `sn_mcp_server.admin`, or `admin`

**Perform these steps before creating a tool:**

1. Create a subflow in Flow Designer using supported input and output data types. See *Create a subflow in Workflow Studio* to learn more.
2. Establish the requisite AI Access Control List (ACL) to facilitate external invocation of the component. See *Create AI ACL* to learn more.
3. Confirm the compatibility status in the staging table. See *Check compatibility of Subflow or Action* to learn more.
4. Register the component as a tool within the MCP Server.

**Procedure:**

1. Select **Subflow** from the categories.
2. On the form, fill in the fields (see Tool form above; the Subflow-specific field is **Subflow** — Select a Subflow type from the list).

   In the **Tool inputs** section, add the fields associated with the capability. The tool is now published on the MCP Server and discoverable by MCP clients.

**What to do next:** Invoke the tool via Claude or an alternative MCP client and verify that it functions as intended. With the tool you registered, open your MCP client to test end-to-end execution.
