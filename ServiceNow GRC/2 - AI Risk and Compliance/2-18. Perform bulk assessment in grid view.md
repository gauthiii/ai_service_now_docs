# Perform Bulk Assessment in Grid View

Perform assessments on multiple risks and controls simultaneously in a risk assessment project using the grid view in AI Risk and Compliance Workspace. You can assess inherent risks, effectiveness of controls, residual risks, and target risks, and define risk responses that enable you to manage and mitigate the risks identified during the risk assessment process.

## What Is the Grid View?

The grid view presents a spreadsheet-style interface for assessing risks and controls in a grid-based layout. It lets you work across many risks and controls at once within a single risk assessment project.

| Roles |

## About This Task

You can perform the bulk risk assessment either from the **Employee Center** or the **AI Control Tower Workspace**.

## Procedure

### 1. Open the project from the Employee Center

1. Navigate to **All > Self-Service > Employee Center**.
2. On the Employee Center landing page, select **GRC Tasks**.
3. Select **Open** and navigate to **Bulk risk assessment**.
4. Open the risk assessment project that you want to assess.

### 2. Or open the project from the AI Control Tower Workspace

1. Navigate to **All > AI Control Tower**.
2. Open the risk assessment project either from the **Tasks** list or from the AI asset record.

### 3. Open the assessment

3. Select **Grid view** to assess the risk assessment project.
4. Select **Open assessment**. A spreadsheet-style interface appears to assess risks and controls in a grid-based view.

> **Note:** If more than one assessor is defined in the Stakeholder section, the assessor who opens the assessment first becomes the current assessor of the risk assessment project.

### 4. (Optional) Reassign the project

5. Select the **more actions** icon, and select **Reassign**.

### 5. (Optional) Add risks to the project

6. Using the **more actions** icon, you can:
   - **Create from risk statements** — create risks from the risk statement.
   - **Create ad hoc risk** — add a risk that is not in the library.
   - **Add risk** — add any underlying risks that have been removed.

   Risks with an in-progress assessment can't be added as part of the risk assessment project.

### 6. (Optional) Remove risks from the project

Removed scoped risks remain part of the project but are marked as not applicable for reporting purposes. However, removed ad hoc risks are completely deleted.

7. Steps:
   1. Select **Actions**.
   2. Select **Remove risk**. The Remove risk dialog box appears.
   3. In the dialog box, provide justification and select **Remove**. The selected risk is deleted, along with all associated responses.

### 7. Complete the assessments

8. Respond to all the factors in **Inherent assessment** for each risk within the project. The application automatically saves your responses and calculates the overall risk score.
9. To assess controls, select **Yes** in the **Applicable** column under **Control assessment**, and then respond to the control effectiveness.
10. To assess **residual** and **target** assessment (if applicable), select **Yes** in the **Applicable** column. The application automatically calculates the final computed scores for individual assessments.
11. Select a risk response strategy and create a risk response task.

### 8. Review, validate, and submit

12. To view the complete summary of the assessment, select **View summary**.
13. To check for any errors before you submit, select the **more actions** icon, and select **Validate**. You can see the list of errors, if any, that must be resolved in the side panel for the selected risk.
14. Identify the errors and resolve them.
15. Select **Validate** to check if the errors have been resolved.
16. To submit the risk assessment project for approval, select **Submit**, and select **Confirm Submission**.

> Assessments are validated again at submission. Review the assessment summary before you submit — **you can't make any changes after submission.**

## Results

The application automatically saves responses, calculates the overall risk score, and computes final scores for individual (inherent, control, residual, and target) assessments. Once validated and submitted with confirmation, the risk assessment project is sent for approval and can no longer be changed.
