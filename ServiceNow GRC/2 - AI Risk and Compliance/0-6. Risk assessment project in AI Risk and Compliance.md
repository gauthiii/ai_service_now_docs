# Risk Assessment Project in AI Risk and Compliance

A risk assessment project lets you perform assessments on multiple risks and controls **simultaneously** by creating a single project for an AI asset. This feature enables assessors to review multiple risks together to understand their potential impact, likelihood, and associated mitigation strategies.

## What Is a Risk Assessment Project?

A Risk assessment project is a **structured process** designed to assess multiple AI asset risks and controls at the same time, particularly within **workshop-based RCSA (Risk and Control Self-Assessment)** environments.

It replaces manual data entry and fragmented workflows by enabling:

- Efficient data collection
- Collaborative input from stakeholders
- Automated reporting

A Risk assessment project simplifies coordination, speeds up processes, and improves data accuracy by providing a **unified workspace** for evaluating risks collectively and reliably.

## Benefits

The key benefits of a risk assessment project are:

- **Simplifies the RCSA process** by enabling multiple risks and controls to be assessed at the same time, which saves time on manual data collection.
- **Reduces errors and inconsistencies** by using automated error handling and providing comprehensive error summaries before finalizing assessments.
- **Reduces back-and-forth** among stakeholders by involving all required participants in the assessment project.
- **Enables navigation** between different stages of risk assessments and access to multiple risks and controls simultaneously, without switching screens.
- **Provides detailed summary reports** on the assessment results.
- **Simplifies sign-off** on all the risks together within the project.
- **Provides reference information** related to each risk in the sidebar panel.

## Personas Involved

| Persona | Responsibilities |
|---------|------------------|
| **Project Owner** | Creates and manages the risk assessment project. Sets up the project context, identifies relevant stakeholders, and defines the scope of risks that must be assessed. |
| **Assessor** | Performs assessment on multiple risks and controls within the project. Analyzes each risk, evaluates the effectiveness of existing controls, and creates risk response strategies. |
| **Approver** | Reviews the completed risk assessments and provides the final sign-off. Ensures all scoped risks have been assessed, then either approves or rejects the risk assessment. |

## Roles Required (Permissions)

The following roles are required for a risk assessment project. Each grants a different level of access.

| Role | Role Name (technical) | Permissions |
|------|-----------------------|-------------|
| **Reader** | `sn_risk_advanced.risk_asmt_project_reader` | **Read-only access** to the risk assessment projects. |
| **User** | `sn_risk_advanced.risk_asmt_project_user` | Ability to **create** risk assessment projects and **update or delete only the projects created by the user**. |
| **Manager** | `sn_risk_advanced.risk_asmt_project_manager` | Ability to **create, update, and delete any** risk assessment projects. |

> **Note:** You must assign these roles **manually** to users to get the appropriate permissions for managing risk assessment projects.

## Workflow / User Journey

The risk assessment project workflow is a structured process to assess multiple risks and controls of an AI asset simultaneously. It progresses through the following stages:

### 1. Define

The risk assessment project is created in this stage. Users with the `sn_risk_advanced.risk_asmt_project_user` role can create a project and complete these tasks:

- Define the related **assessable entity** and the **risk assessment methodology (RAM)**.
- Specify the **project name and description**.
- Identify and add relevant **stakeholders**, including the risk assessment project owner, assessors, and watchlist users.

### 2. Risk Scoping

The **project owner** defines the risks to be assessed. The following options are available to map risks for the selected composite or single entity:

| Option | Description |
|--------|-------------|
| **Create risk from risk statements** | Add risks from the risk statement. |
| **Create ad-hoc risk** | Add a risk that is not in the library. |
| **Add risk** | Create risks from the internal risk library. |

### 3. Assessment

The **assessor** evaluates each scoped risk using the defined RAM. During this stage:

- The assessor can access a **summary** of the assessment throughout the process.
- **Automated error handling** enables assessors to validate projects, reducing the likelihood of errors and inconsistencies.
- After completion, the assessor **submits the assessment for approval**.
- Assessors can also **add risks** during the assessment stage.

> **Note:** If any risk assessment has a **High inherent rating**, the approval is sent to the risk assessment project owner. If the project owner and assessor are the **same user**, the approval is **skipped**. To use this approval flow, you must activate the default approval configuration **'Bulk risk assessment approval config.'**

### 4. Approval

The **approvers** configured in the approval configurator review the assessment summary. Based on their satisfaction, the approver can do one of the following:

| Action | Result |
|--------|--------|
| **Approve** | The project moves to the **Completed** stage. |
| **Reject** | The project moves back to the **Assessment** stage for the assessor to address identified issues and make necessary revisions. |

> **Note:** After the project reaches the **Completed** state, you can create a new project with the same RAM and entity. When the new project reaches the Completed state, the old project moves to the **Archived** state.

## Project Lifecycle Summary

The overall lifecycle of states a project moves through:

```
Define → Risk Scoping → Assessment → Approval → Completed → (Archived)
```

- A **rejected** approval returns the project to **Assessment**.
- A **Completed** project can be superseded by a new project (same RAM + entity), which moves the old one to **Archived** once the new one completes.
