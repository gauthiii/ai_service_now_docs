# Create Bulk Risk Assessment

## Overview

A **bulk risk assessment project** lets you perform assessments on multiple risks and controls simultaneously using the AI Risk and Compliance workspace. When you create the project you can define its context, including:

- The assessable entity
- The Risk assessment methodology (RAM)
- The project name and description
- The stakeholders you identify and add

## Before You Begin

| Requirement | Detail |
|-------------|--------|
| Role required | `sn_risk_advanced.risk_asmt_project_user` |
| Application | You must install the **Advanced Risk** application to create a bulk risk assessment. |

## Procedure

1. Navigate to **All > AI Risk and Compliance Workspace**.
2. Select the **list** icon.
3. Navigate to the AI asset inventory and open the AI asset record.
4. To create a bulk risk assessment, use one of the following two methods:
   - Select the **AI assessment** drop-down, and select **Bulk risk assessment**.
   - Navigate to **Assessments > Bulk risk assessments** and select **New**.
5. On the **Create bulk risk assessment** dialog, fill in the **Context** fields.

   ### Context

   | Field | Description |
   |-------|-------------|
   | Related entity | Entity that needs to be assessed. **Note:** The Related entity field is grayed out if only one related entity is associated with the AI asset. |
   | Risk assessment methodology (RAM) | The RAM you use to assess risks within the project. The default RAM is **Risk assessment for AI inventory**. You can create your own RAM and perform the bulk risk assessments. |

6. Select **Create**.
7. In the Risk assessment project, do the following:

   a. In the **Details** section, fill in the fields.

   ### Details

   | Field | Description |
   |-------|-------------|
   | Name | Name of the risk assessment project. The name is automatically generated based on the selected assessable entity, followed by the current month and year, with a separator between them. You can modify the name if necessary. |
   | Description | Description about the risk assessment project. It helps the assessors to understand the project in detail. |

   b. Select **Next**.

   c. In the **Stakeholders** section, fill in the fields.

   ### Stakeholders

   | Field | Description |
   |-------|-------------|
   | Owner | Name of the project owner. Automatically set to the user who created the project, but you can modify it if needed. |
   | Assessor type | User or group responsible for assessing the project. Choices: **Users** (select users), **Group** (select a group of users), **Entity owner** (select the assessable entity owner as assessor), **Entity stakeholder** (select assessor from the assessable entity stakeholders list). |
   | Assessors | User responsible for assessing the project. Appears only when **Users** is selected from Assessor type. |
   | Assessor group | Group responsible for assessing the risk. Appears only when **Group** is selected from Assessor type. |
   | Assessors from entity | Assessors from the assessable entity stakeholders list. Appears only when **Entity stakeholder** is selected from Assessor type. |
   | Watchlist type | Users or group who can view and receive notifications about the project. Choices: **Users** (select users), **Group** (select a group of users), **Entity stakeholder** (select watchlist users from the assessable entity stakeholders list). |
   | Watchlist users | Users who can view and receive notifications. Appears only when **Users** is selected from Watchlist type. |
   | Watchlist group | Group who can view and receive notifications. Appears only when **Group** is selected from Watchlist type. |
   | Watchlist users from entity | Watchlist users from the assessable entity stakeholders list. Appears only when **Entity stakeholder** is selected from Watchlist type. |

   d. Select **Next**.

   > The project moves to the **Scope risk** stage and you can see a list of risks mapped from the assessable entities.

8. To add risks in the **Scope risk** stage, you can do the following:
   - **Optional:** To create risks from the risk statement, select **Create from risk statements**.
   - **Optional:** To add a risk that is not in the library, select **Create ad-hoc risk**.
   - **Optional:** To remove risks from the assessment without deleting them from the underlying entities, select the risks, and select **Remove**.
   - **Optional:** To add any risks that have been removed, select **Add risk**.

9. Select **Next**.

   > The project moves to the **Assess** stage, and a notification goes to the assessor.

## Result

The bulk risk assessment project is created and progresses through the **Scope risk** stage to the **Assess** stage. Once it reaches the Assess stage, a notification is sent to the assessor responsible for assessing the project.
