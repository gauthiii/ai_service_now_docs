Types of privacy assessments
Privacy assessments can be sent using various methods such as entities, entity types, and
processing activities.
Multiple types of privacy assessments such as privacy screening assessment, privacy impact
assessment, transfer impact assessment, and so on can be configured according to your
requirements.
Initiating privacy assessments for an entity or a processing activity
To discover whether a business process or an application is processing personal data, you
can use privacy screening assessments. To regularly assess how processing activities are
processing personal data, you can use privacy assessments such as privacy impact assessment
(PIA).
Privacy screening assessment
Conducting a privacy screening assessment helps you discover whether the entity is processing
personal information or not.
You can send the privacy screening assessments in the following ways:
•From the entity record only when a processing activity is not available.
•From the entities related list in entity types.
When you initiate privacy screening assessments for entities, processing activities are created
based on the responses submitted by the entity owners.
Note: The processing activity creation rules can be configured based on the screening
assessment responses.
After a processing activity is created, based on the assessment configurations, the following
events occur:

Privacy assessments
When processing activity records are identified, to understand how a processing activity is
processing personal information, various types of privacy assessments such as privacy impact
assessments (PIA) and transfer impact assessment (TIA) are sent. You can trigger PIAs from
a single processing activity record. For more information, see Send a privacy assessment
from a processing activity. Alternatively, you can send privacy impact assessments to multiple
processing activities, using the Entity type capability or from the processing activity list view.
For more information on how to send an assessment to multiple entities, see Send a privacy
assessment to multiple entities.

Before you begin
Role required: sn_privacy.manager or sn_privacy.analyst
About this task
You can send a privacy screening assessment from an entity if you do not have a processing
activity created for that entity. After a processing activity is created for an entity, any further
assessments can only be sent through the processing activity.
Procedure
1.Navigate to All > Privacy Management > Privacy Workspace > Scoping > My entities.
2. Select and open the entity from which you want to send the assessments to the entity owner.
3. Select Initiate privacy assessments.
The Send privacy assessment dialog box opens.
4.Provide the required information such as the template to use, the due date, and so on in the
dialog box.
The description of the assessment is automatically fetched from the description provided at
the time of the assessment template configuration. You can modify the description as required.
5. Select Send.
New privacy assessment creation form
Use the fields on the new privacy assessment creation form to create a new privacy assessment.

Before you begin
Role required: sn_privacy.manager
Procedure
1.Navigate to All > Privacy Management > Privacy Workspace > List > Scoping > Entity types.
2. Select the entity types that contain the entities for which you want to send the assessments.
3. Select the entities related list.
4.Select the check box against the required entities.
5. Click Send privacy assessments.
6.From the Classification field, select the type of assessment.
7.From the Assessment template field, select the template you want to use.
8.In the Due Date field, select the due date for completing the assessment.
9.Click Send.
Result
The assessment is sent to the entity owners.
Send a privacy assessment from a processing activity
Send a privacy assessment to a processing activity owner from a processing activity record to
collect more information on why and how the processing activity is using personal information.
Before you begin
Role required: sn_privacy.manager or sn_privacy.analyst
Procedure
1.Navigate to All > Privacy Management > Privacy Workspace > Processing activities > All
processing activities.
2. Select and open the processing activity from which you want to send a privacy assessment.
3. Click Initiate privacy assessments.
You are redirected to the Create new privacy assessment form.
4.On the form, fill in the fields.
For an explanation of the fields, refer to the New privacy assessment creation form
5. Select Update state.
a.In the State field, select Assigned.
b.In the Additional comments field, provide comments for the person taking the assessment.
c.Select Submit.
Result
The assessment is sent to the entity owner.
Send privacy assessments from multiple processing activities
Send multiple privacy assessments from multiple processing activities. This capability enables
you to filter the processing activities by business units, locations, data subjects, and data types.
