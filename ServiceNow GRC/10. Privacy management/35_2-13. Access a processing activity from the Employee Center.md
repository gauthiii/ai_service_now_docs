About this task
Assigning a processing activity to a key stakeholder gives a business owner direct access to
review and update it. This eliminates the need for relaying changes through the privacy team.
The owner updates the activity in the Employee Center and assigns it back to the privacy team
when complete.
You can assign a processing activity to a key stakeholder when it is in the Discover state.
Procedure
1.From the List view of the Privacy Workspace, navigate to Processing activities > Unassigned
processing activities.
2. Select the processing activity to be assigned to a key stakeholder.
3. Select the more actions icon , and select Assign to key stakeholder.
Note: If a stakeholder is already assigned to the processing activity, select Reassign
to key stakeholder to change the assignment to a different user. This removes the edit
access of the existing stakeholder.
4.In the Assign to key stakeholder window, select a stakeholder.
Note: Only stakeholders with the processing activity privilege, Edit processing activity
and respond to privacy assessments, are available for selection. To modify privileges,
see Add key stakeholders to a processing activity.
5. Select Assign.
Result
The assigned stakeholder receives an email notifying them of the processing activity
assignment. The stakeholder can navigate to Employee Center > GRC tasks > Tasks > My
pending tasks > Processing activities to view and edit the assigned processing activity record.

Role required: sn_privacy.business_user
About this task
Privacy analysts can add any user as a key stakeholder to a processing activity, even those
without privacy roles. To enable users without privacy roles to view and edit a processing activity,
they must be granted the sn_privacy.business_user role, and then have their processing activity
privilege modified by a privacy analyst. To understand processing activity privileges, see Add key
stakeholders to a processing activity.
