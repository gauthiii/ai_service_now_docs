Request a new model
Request a new model in the Employee Center to initiate the model intake process and capture all
required details for review and approval.
Before you begin
Role required: sn_model_risk_mgmt.model_owner or sn_grc_emp_user.grc_employee
Any employee [sn_grc_emp_user.grc_employee] can request a model, when the Employee User
application is installed.
Procedure
1.Navigate to All > Self-Service > Employee Center.
2. On the Employee Center landing page, select Risk and compliance.
3. Select the New Model Intake card.
4.On the form, fill in the fields.
For a description of the field values on the New Model Intake form, see New Model Intake form.
5. Select Submit.
A new model record is created and submitted to the Model Governance team to review.

Perform model validation
Perform model validation to assess and validate the model, collect required documents, report
any issues, and determine if the model is fit for use. This process confirms that the model meets
required standards and is ready for deployment.
Before you begin
Role required: sn_model_risk_mgmt.model_validator
About this task
A model risk validation task contains the following stages:

•Document collection
•Validation checklist
•Issues
•Validation outcome
You can perform the listed activities in any sequence.
Procedure
1.To perform model validation from the GRC tasks page, navigate to All > Self-Service >
Employee Center and do the following:
a.Under My active items, select GRC tasks.
b.Select the model validation from the My to-dos card.
2. To perform model validation from the Model Risk Workspace, navigate to All > Model Risk
Workspace and do the following:
a.Select the list icon .
b.From the list, navigate to Models > Pre-deployment.
c.Select the model for which you want to perform validation.
3. In the Pre-deployment state, select Open task.
The model risk validation task workflow opens.
4.To send an evidence collection request, select Create Request.
For more information, see Request evidence for model.
5. In the Validation Checklist stage, select Open to perform the validation.
6.Respond to all the factors or questions listed in the validation task.
7.Select Submit.
After you submit your responses, you’re no longer able to modify them.
8.Optional: In the Issues stage, add new or existing issues with the model.
When you add new or existing issues, they are associated with the model risk and the model
risk task.

About this task
This task guides you on how to enable or disable the option to copy responses from previous
assessment tasks for a model risk. Model risk governance and model risk coordinators can
modify this field at any stage of the model risk life-cycle. When enabled, responses from the
most recent completed assessment are copied into new assessment tasks.
Note: You can copy responses only from assessment tasks, not from validation tasks.
Procedure
1.Navigate to All > Model Risk Workspace.
2. Select the list icon .
3. From the list, navigate to Models > All.
4.Select the model and go to the Details tab.
5. In the Pre-fill assessment responses, select Copy responses from previous assessment.
When enabled, new assessments copy responses from the most recently completed
assessment, if available. Respondent can edit the pre-filled responses before submitting.
