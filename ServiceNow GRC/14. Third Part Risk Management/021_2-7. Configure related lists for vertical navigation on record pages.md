Configure related lists for vertical navigation on record pages
Configure the related lists that appear in the vertical navigation layout on record pages in the
Vendor Management Workspace.
Before you begin
Role required: admin
About this task
Beginning with version 21.1.x, record pages in the Vendor Management Workspace use a
structured vertical navigation layout instead of the legacy horizontal tab-based layout. The
vertical navigation panel groups related lists and pages into logical sections.

Configuring related lists for vertical navigation requires two steps. First, configure the related
list on the record form using the workspace record view used by the Vendor Management
Workspace. Second, explicitly add the related list to the vertical navigation using configuration
tables.
The second step is required because related lists configured on the record form do not
automatically appear in the vertical navigation layout.

2. Configure the related list on the record form.
a.In the Application Navigator (All), enter the target table name followed by .list (for
example, core_company.list).
b.Open an existing record from the list.
c.From the form header, select Configure > Related Lists.
d. Select Edit this view.
e. From the view selector, choose the workspace record view used by the Vendor Management
Workspace.
f.In the Available list, select the related list.
g.Move the related list to the Selected list.
h.Select Save.
i. Verify that the related list appears on the form in the selected view.
This step makes the related list available to the workspace but does not add it to the vertical
navigation.
3. Navigate to the vertical navigation configuration table using one of the following options.
◦▪In the Application Navigator (All), enter
sn_rec_pg_vertical_view_config.list.
▪Select VRM vertical nav configuration.
◦All > Third-party Risk Management > Administration > Vertical layout configuration.
This table defines which related lists appear in the vertical navigation for workspace record
pages.
4.Create or update a vertical navigation entry.
a.To create a new entry, select New.
b.To update an existing entry, open it from the list.
5. Configure the vertical navigation entry.
a.In the Table field, select the target table.
b.In the Group field, select the group where the related list should appear.
c.Set the Type field to Related list.
d. Set the display order.
e. Select Save.
6.Verify the related list appears in the vertical navigation.
a.Open a record page in the Vendor Management Workspace.
b.Refresh the page if it was already open.
c.Confirm that the related list appears under the selected group in the vertical navigation
panel.
Result
The related list appears in the vertical navigation layout on the configured record page in the
Vendor Management Workspace.
