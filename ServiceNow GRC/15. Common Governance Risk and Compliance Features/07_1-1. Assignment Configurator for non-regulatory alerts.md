Assignment Configurator for non-regulatory alerts
Assignment rules help in automating organization’s task records such as action tasks, and non-
task records such as issues, incidents, regulatory alerts, and so on.
The Assignment Configurator module allows you to automatically set a value in the assigned_to
and assignment_group fields when a set of conditions occurs and applies to Integrated Risk
Management record-based tables such as issues, incidents, compliance cases, risks, controls
and many more.
An assignment rule must also meet these additional criteria to run:
•The task record has been created or updated. Assignment rules do not apply to unsaved
changes on a form.
•The task record must be unassigned. The record cannot have an existing value for either the
assigned_to or assignment_group fields. Assignment rules cannot overwrite existing
assignments (including assignments set by a default value or a previously run assignment
rule).
•The assignment rule is the first rule that matches the table and conditions. If more than one
assignment rule matches the conditions, only the rule with the lowest order value runs.
