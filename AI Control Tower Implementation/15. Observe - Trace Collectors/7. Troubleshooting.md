## What Is This?

This subtopic provides a troubleshooting reference for configuring trace collectors (trace connections) in AI Control Tower. It lists common issues encountered when setting up trace collection for externally hosted AI agents, their likely causes, and resolutions. Implementers use it to diagnose problems such as missing credentials, MID Server availability, and absent trace data after the first collection interval.

---

## Troubleshooting Reference

| Issue | Likely Cause | Resolution |
| --- | --- | --- |
| Credential not appearing in dropdown | Credential has not been created in the ServiceNow Credentials module. | Create the credential in the Credentials module first. Then return to the trace configuration. |
| MID Server cannot be selected or is unavailable | MID Server is not Active or has not been validated. | Verify the MID Server status and run validation. See MID Server Installation reference documentation. |
| No trace data appearing after first collection | Wrong AWS region selected, or cloud credentials lack required permissions. | Confirm the AWS region with your cloud team. Review credential permissions for trace collection APIs. |
| Trace connection created but execution logs show errors | Credential alias not configured (Feb release) or observability service is unreachable. | Verify the Credential Alias field is populated (Feb release). Confirm observability service connectivity from the MID Server. |

---

## How to Use This in Implementation

When trace data does not populate as expected after configuring a trace connection, match the observed symptom against the Issue column and apply the corresponding resolution. Prerequisites surfaced here include: creating the credential in the ServiceNow Credentials module before configuring the trace connection, ensuring the MID Server is Active and validated, selecting the correct AWS region, and ensuring cloud credentials carry the permissions required by trace collection APIs. For the Feb release, confirm the Credential Alias field is populated and that the observability service is reachable from the MID Server. Use the connection's execution logs (reviewed after submitting the form) to confirm trace data begins populating in AI Evaluations after the first collection interval.
