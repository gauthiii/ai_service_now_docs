## What Is This?

This subtopic explains where trace data collected through the trace collector configuration ultimately flows within AI Control Tower (AICT). The trace data feeds three distinct AICT capabilities, each serving a different purpose and audience. Implementers should ensure that stakeholders in each affected area are informed during the rollout.

---

## Where Trace Data Flows

Trace data collected through this configuration flows into three distinct AICT capabilities. Implementers should ensure stakeholders in each area are informed during rollout.

| AICT Capability | How Traces Are Used | Primary Stakeholder |
| --- | --- | --- |
| AI Evaluations | Trace data is forwarded to the monitoring service to generate observability metrics—latency, performance, error rates—for externally hosted AI agents. | AI Program Lead / AI Operations |
| AI Discovery | Traces surface AI agent metadata (e.g., AWS AgentCore agent info) not captured through standard discovery connections, enabling comprehensive inventory management. | AI Asset Manager |
| Security & Privacy | Trace data is routed to Galileo for security metric generation, enabling risk and compliance teams to evaluate AI agent behavior against defined security policies. | Risk & Compliance Lead |

---

## How to Use This in Implementation

Once trace collection is configured and trace data begins populating after the first collection interval, an implementer should communicate the downstream effects to the relevant stakeholders. Specifically: coordinate with the AI Program Lead / AI Operations regarding the new observability metrics (latency, performance, error rates) generated in AI Evaluations for externally hosted AI agents; coordinate with the AI Asset Manager because AI Discovery will now surface AI agent metadata (such as AWS AgentCore agent info) not captured by standard discovery connections; and coordinate with the Risk & Compliance Lead because trace data is routed to Galileo to generate security metrics in Security & Privacy for evaluating AI agent behavior against defined security policies.
