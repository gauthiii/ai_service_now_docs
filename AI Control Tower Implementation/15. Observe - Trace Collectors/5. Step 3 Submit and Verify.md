## What Is This?

"Step 3 Submit and Verify" is the final step in the trace collector configuration procedure within AI Control Tower, following Step 1 (Access Trace Configuration) and Step 2 (Fill In Trace Configuration Fields). It is where an implementer submits the completed trace configuration form and verifies that the trace collector has been set up correctly. This step concludes the workflow for configuring a trace collector for a supported cloud provider (AWS, GCP, or Azure).

### Context From the Preceding Steps (same procedure, for reference only)

This step is the third part of the "Configuration Steps" procedure used to configure a trace collector in AI Control Tower, which applies to all supported cloud providers unless noted:

- **Step 1 Access Trace Configuration** — Navigate to the AI Control Tower workspace, go to Configurations > AI Evaluations > Traces, click Add, and select the cloud provider connector (AWS, GCP, or Azure).
- **Step 2 Fill In Trace Configuration Fields** — Complete the required trace configuration form fields (Name, Credential, Collection Frequency, MID Server, and conditionally Credential Alias and AWS Region).

## How to Use This in Implementation
