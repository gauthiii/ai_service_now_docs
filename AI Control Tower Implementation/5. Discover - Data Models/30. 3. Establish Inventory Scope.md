## What Is This?

"Establish Inventory Scope" is the third step in the structured, governance-driven approach to determining the scope of the AI Asset Inventory. It defines the boundaries of what the AI asset inventory will cover — the asset types, source systems, and lifecycle stages to be tracked. It fits within the broader "Determining the Scope of AI Asset Inventory" workflow during the Discover phase of AICT implementation.

---

## Establish Inventory Scope

This step is part of the structured, governance-driven approach required to define scope under "Determining the Scope of AI Asset Inventory."

Define:

- Asset types to be tracked
- Source systems and platforms
- Lifecycle coverage (intake → build → deploy → retire)

### Diagram/Screenshot: Lifecycle coverage flow

The lifecycle coverage is expressed as a simple linear flow with arrows connecting four stages:

```
intake → build → deploy → retire
```

This indicates the inventory should track AI assets across the full lifecycle, from initial intake through build and deployment, and ultimately to retirement.

---

## How to Use This in Implementation

After defining AI assets (Step 1) and classifying AI use cases as AI Systems (Step 2), an implementer uses this step to scope the inventory itself. Specify which asset types will be tracked, identify the source systems and platforms that feed the inventory, and ensure lifecycle coverage spans the full intake → build → deploy → retire sequence. This scoping feeds into the subsequent step, "Integrate with Enterprise Systems," where alignment with CRM, ITSM, HR, vendor/procurement, and software asset management tools is ensured.
