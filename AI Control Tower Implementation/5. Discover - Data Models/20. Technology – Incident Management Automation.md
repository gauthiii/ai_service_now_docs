## What Is This?

This subtopic presents a worked example of the AI Control Tower data model applied to a technology-industry scenario. It maps a Generative AI incident-management use case onto the standard AICT categories (Business Application, AI Type, AI System, AI Model, Dataset, Input, Prompt, Output). It illustrates how a real Now Assist for ITSM use case is described and inventoried within the AICT data model.

---

## Scenario

A technology provider uses Generative AI to summarize IT incidents and assign ownership.

## Data Model Mapping

| Category | Example |
|---|---|
| Business Application | Now Assist for ITSM |
| AI Type | Generative AI |
| AI System | Incident Summarization |
| AI Model | LLM (e.g., Mixtral, GPT-4 via Azure OpenAI) |
| Dataset | Ticket data (title, description, activity, resolution notes) |
| Input | Ticket ID, description, category, assignment history |
| Prompt | "Summarize this incident in under 250 words" |
| Output | Incident summary |

---

## How to Use This in Implementation

Use this example as a template when cataloging a Generative AI incident-management use case in the AI Control Tower. For each AICT data-model category, capture the concrete value: identify the business application (Now Assist for ITSM), classify the AI type (Generative AI), name the AI system and underlying model, and record the dataset, inputs, prompt, and expected output. Documenting these fields consistently lets the AICT inventory track, govern, and assess the use case alongside other AI systems.
