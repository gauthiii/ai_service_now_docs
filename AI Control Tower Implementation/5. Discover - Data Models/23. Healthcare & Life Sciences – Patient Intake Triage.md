## What Is This?

This subtopic presents a worked example of how a Generative AI use case maps onto the AICT AI asset data model. The scenario is a hospital that uses GenAI to summarize patient symptoms and determine urgency. It illustrates how each AICT category (Business Application, AI Type, AI System, AI Model, Dataset, Input, Prompt, Output) is populated for a healthcare and life sciences triage use case.

---

## Scenario

A hospital uses GenAI to summarize patient symptoms and determine urgency.

## Patient Intake Triage – AI Asset Mapping

| Category | Example |
|----------|---------|
| Business Application | Electronic Health Record (EHR) |
| AI Type | Generative AI |
| AI System | Symptom Summarization |
| AI Model | Medical LLM (fine-tuned) |
| Dataset | Patient intake notes, diagnosis history |
| Input | Patient symptoms, prior records, demographics |
| Prompt | "Summarize symptoms and recommend triage category" |
| Output | Clinical summary and urgency classification |

---

## How to Use This in Implementation

When registering a healthcare GenAI triage use case in AICT, use this mapping as a template for the AI asset record. Set the AI Type to Generative AI, associate the AI System (Symptom Summarization) with the fine-tuned Medical LLM model, and link the relevant datasets (patient intake notes, diagnosis history). Capture the input fields, the triage prompt, and the expected output (clinical summary and urgency classification) so the asset is fully tracked. Because this is a GenAI use case operating on sensitive patient data, ensure the prompt, datasets, and outputs are documented for governance, risk, and compliance review.
