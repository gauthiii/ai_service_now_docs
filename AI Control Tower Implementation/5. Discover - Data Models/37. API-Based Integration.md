## What Is This?

API-Based Integration is one of the methods used to discover and populate the AI Asset Inventory in ServiceNow AI Control Tower. It is intended for large-scale environments that require automation, enabling AI asset records to be created, updated, and retrieved programmatically and kept in sync with external systems. It complements other population methods such as Auto Discovery and Manual Intake within the recommended hybrid approach.

---

## API-Based Integration

Used for large-scale environments requiring automation.

### Capabilities

- Create, update, and retrieve AI asset records
- Integrate with external systems
- Maintain real-time inventory accuracy

### Requirements

- Appropriate roles (asset, model_manager)
- Asset Classes plugin enabled

---

## How to Use This in Implementation

Implementers choose API-Based Integration when populating the AI Asset Inventory at scale or when external systems must drive inventory data. Before using it, ensure the prerequisites are met: assign the appropriate roles (asset, model_manager) and enable the Asset Classes plugin. With these in place, the API can be used to create, update, and retrieve AI asset records, integrate with external systems, and maintain real-time inventory accuracy. This method fits within a hybrid approach alongside Auto Discovery (scheduled sync of ServiceNow assets) and Manual Intake.
