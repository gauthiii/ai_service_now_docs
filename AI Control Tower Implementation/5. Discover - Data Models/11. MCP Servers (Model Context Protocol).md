## What Is This?

MCP (Model Context Protocol) Servers are one of the data model entities tracked within the Discover capability of AI Control Tower. They act as standardized connectors between AI systems and external tools, data sources, or services. Tracking them in AICT gives governance visibility into how AI systems reach out to external resources and the associated dependencies and risks.

---

## What MCP Servers Enable

MCP servers act as standardized connectors between AI systems and external tools, data sources, or services.

They enable:

- Secure data access
- Tool invocation
- Cross-platform interoperability

## Governance Value

Tracking MCP servers provides insight into:

- External system dependencies
- Data access pathways
- Security and privacy risks

---

## How to Use This in Implementation

When mapping your AI landscape in AICT, register MCP servers as a distinct data model entity alongside models, datasets, and prompts. Cataloging each MCP server makes external system dependencies and data access pathways visible, which supports security and privacy risk assessment. Use this inventory to understand how AI systems connect to external tools and services and to govern those integration points.
