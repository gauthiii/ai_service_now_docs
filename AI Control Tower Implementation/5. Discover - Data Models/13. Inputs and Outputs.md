## What Is This?

Inputs and Outputs is a data model concept in AI Control Tower that describes the data flowing into an AI system and the actions or responses that result from it. Inputs represent the data provided to an AI system, while outputs represent the resulting actions or responses. Defining these clearly supports governance by enabling transparency, traceability, and risk analysis across AI capabilities.

---

## Inputs and Outputs

Inputs represent the data provided to an AI system, while outputs represent the resulting actions or responses.

Examples include:

- **Inputs:** text, images, sensor data
- **Outputs:** summaries, predictions, decisions, recommendations

**Governance Value** — Defining inputs and outputs supports:

- Behavioral transparency
- Decision traceability
- Risk and impact analysis

---

## How to Use This in Implementation

When inventorying an AI system in AICT, document both the inputs it consumes (e.g., text, images, sensor data) and the outputs it produces (e.g., summaries, predictions, decisions, recommendations). Capturing this mapping enables behavioral transparency, decision traceability, and risk and impact analysis, which are foundational for governing the AI system end to end.
