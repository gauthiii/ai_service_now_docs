## What Is This?

This subtopic presents a worked example of how a Manufacturing organization's predictive maintenance use case maps onto the AI Control Tower data model. It uses a Traditional AI scenario where a manufacturing organization predicts equipment failure using AI, and breaks the use case down into the standard AICT categories (Business Application, AI Type, AI System, AI Model, Dataset, Input, Prompt, Output). It serves as a reference pattern for cataloging non-generative, telemetry-driven AI systems in AICT.

---

## Scenario

A manufacturing organization predicts equipment failure using AI.

## Use Case Mapping

| Category | Example |
| --- | --- |
| Business Application | Asset Management |
| AI Type | Traditional AI |
| AI System | Predictive Maintenance Engine |
| AI Model | Time-series forecasting model |
| Dataset | Sensor logs, maintenance records, machine usage |
| Input | Machine telemetry data |
| Prompt | Not applicable |
| Output | Predictive alert (e.g., failure within 48 hours) |

---

## How to Use This in Implementation

When registering a manufacturing predictive maintenance use case in AICT, use this mapping as a template: classify it under the Asset Management business application as a Traditional AI system named "Predictive Maintenance Engine," backed by a time-series forecasting model. Because it is non-generative, the Prompt category is not applicable. Populate the Dataset with sensor logs, maintenance records, and machine usage; set the Input to machine telemetry data; and define the Output as a predictive alert (for example, failure within 48 hours). This pattern shows implementers how to model AI systems that operate on streaming telemetry rather than prompts.
