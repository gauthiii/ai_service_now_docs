## What Is This?

This subtopic identifies where indicator data comes from when configuring indicators for continuous monitoring in the Risk and Compliance section of AICT. It establishes that there are two distinct data sources, and that the applicable source must be identified before an indicator is configured. It fits into the broader implementation work of setting up indicators to automate governance oversight at scale.

---

## Indicator Data Sources

Indicator data is sourced from two sources:

1. Metrics generated natively within ServiceNow.
2. Data imported from external integrations.

Before configuring indicators, identify which data source applies.

---

## How to Use This in Implementation

When setting up an indicator, an implementer must first determine which of the two data sources applies — native ServiceNow metrics or data brought in through an external integration. Making this determination up front is a prerequisite to configuring the indicator correctly, since the configuration approach depends on the source of the data.
