## What Is This?

API Integration describes how the AI Assets API exposes out-of-the-box (OOTB) endpoints for CRUD operations against AI asset records in the AI Control Tower. It is the integration pattern for organizations that maintain a live AI registry in an external system and need to keep ServiceNow synchronized in real time. It fits into the technical and functional considerations for connecting AICT to external tooling.

---

## AI Assets API

The AI Assets API provides OOTB endpoints for CRUD operations (Create, Read, Update, Delete) across AI asset types. This integration pattern is best suited for organizations that maintain a live AI registry in another system and require real-time synchronization.

### Important

The AI Assets API requires the Asset Classes (`sn_ent`) plugin to be installed. Confirm this dependency is satisfied before configuring API-based integrations.

## How to Use This in Implementation

To use the AI Assets API, first confirm the Asset Classes (`sn_ent`) plugin is installed, since it is a hard dependency for the API. Once the plugin is satisfied, implementers can configure the OOTB CRUD endpoints to create, read, update, and delete AI asset records. This pattern is recommended when an external system acts as the source-of-truth AI registry and must stay in real-time sync with ServiceNow.
