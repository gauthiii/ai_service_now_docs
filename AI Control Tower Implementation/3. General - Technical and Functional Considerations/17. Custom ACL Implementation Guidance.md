## What Is This?

This subtopic provides guidance on when and how to modify Access Control Lists (ACLs) in AI Control Tower (AICT). It addresses situations where the out-of-the-box (OOTB) roles do not fully satisfy an organization's access requirements, and advises restraint when introducing custom ACLs. It fits into the access and security design portion of an AICT implementation, immediately following the standard role definitions.

---

## Custom ACL Implementation Guidance

When out-of-the-box roles do not fully address an organization's access requirements, ACLs can be modified to change the baseline permission set. Implementation teams should limit the overall number of custom ACLs to maintain upgradability and ensure organizations can leverage the latest AICT features with minimal rework.

---

## How to Use This in Implementation

When the standard AICT and AI Risk and Compliance roles do not meet an organization's specific access needs, implementers can adjust ACLs to alter the baseline permission set. To preserve upgradability and minimize future rework, keep the number of custom ACLs as low as possible so the organization can continue to adopt the newest AICT features.
