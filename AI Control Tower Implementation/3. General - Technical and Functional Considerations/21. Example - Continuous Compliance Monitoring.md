## What Is This?

This subtopic provides a worked example showing how indicators function in practice for continuous compliance monitoring within AI Control Tower. It demonstrates how an indicator tied to a regulatory control automatically evaluates compliance on a schedule and how out-of-compliance results surface to the AI Steward for corrective action. It fits into the broader discussion of indicators and data sources used to monitor AI controls.

---

## How Indicators Enable Continuous Compliance Monitoring

The following example illustrates how indicators function in practice. If an indicator is configured for a control linked to the NIST AI Risk Management Framework, it will automatically run at an established interval and analyze data to determine whether the control remains in compliance.

If the control falls out of compliance, it is automatically set to a Non-Compliant state and surfaces within the Risk and Compliance Compliance by Authority Documents and Policies section of the workspace. This is immediately visible to the AI Steward, who can monitor the latest developments for control and ensure corrective action is on track.

### Tip

Indicator templates are the most efficient way to monitor similar controls across multiple AI assets. Define a single template and generate asset-specific indicators from it to reduce configuration overhead.

---

## How to Use This in Implementation

To apply this in an implementation, configure an indicator for a control that is linked to an authority document or framework (for example, the NIST AI Risk Management Framework). Set the indicator to run on an established interval so it automatically analyzes the relevant data and evaluates whether the control remains compliant. When a control is flagged Non-Compliant, ensure the AI Steward role has access to the Risk and Compliance "Compliance by Authority Documents and Policies" section of the workspace, where non-compliant controls surface for monitoring and corrective action. To scale across many AI assets efficiently, build indicator templates and generate asset-specific indicators from them rather than configuring each indicator individually.
