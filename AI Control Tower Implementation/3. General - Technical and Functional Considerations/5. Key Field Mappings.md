## What Is This?

Key Field Mappings define how source data from imported AI assets should be aligned to out-of-the-box (OOB) fields in the AI Control Tower. Applying these mappings during data import ensures consistency across asset records and establishes lifecycle accountability and governance coverage from the start of an implementation.

---

## Field Mapping Guidance

When importing AI assets, organizations should map source data to out-of-the-box (OOB) fields to ensure consistency. The following fields are critical to establish lifecycle accountability and governance coverage from day one.

| Field | OOB Field Name | Purpose |
| --- | --- | --- |
| Model Category | model_category | Distinguishes between AI systems, models, and datasets. |
| Provider/Manufacturer | manufacturer | Enables vendor oversight and third-party risk tracking. |
| Managed By | managed_by | Assigns lifecycle accountability to a named owner. |

---

## How to Use This in Implementation

During the Data Import phase, map your source asset data to these OOB fields as part of the field mapping configuration (especially relevant for the Data Source Imports pattern, which uses import sets with transform maps). Populating `model_category` ensures AI systems, models, and datasets are correctly classified; `manufacturer` supports vendor oversight and third-party risk tracking; and `managed_by` establishes a named owner for lifecycle accountability. Consistently applying these mappings from day one provides governance coverage and accountability across the AI inventory.
