## What Is This?

The "Key Value Benefits" subsections describe the governance value delivered by registering AI Models and Datasets in the AI Control Tower. They explain why cataloging models and datasets matters for transparency, traceability, privacy, and quality management as part of the Govern – Risk and Controls process.

---

## AI Model — Key Value Benefits

The AI Model Registry is a catalog of the AI models that support one or more AI systems or use cases.

It enables technical transparency, version tracking, and model-level governance, ensuring changes to models are controlled and traceable.

Example: A machine-learning classification model used to predict loan default risk, version 2.3, deployed in production.

---

## Dataset — Key Value Benefits

The Dataset Registry is an inventory of datasets used to train, test, or operate AI models.

It supports data transparency, privacy, and quality management by documenting where data comes from and how it is used.

---

## How to Use This in Implementation

When implementing the Govern – Risk and Controls process, register each AI model in the AI Model Registry and each dataset in the Dataset Registry to realize these benefits. For AI Models, use the registry to track versions and control changes so that model-level governance is traceable. For Datasets, document the source and usage of each dataset to maintain data transparency, privacy, and quality. For both, requests are submitted via the "Request an AI Model" and "Request a Dataset" links, and configuration is handled by referring to the Govern – Lifecycle topic.
