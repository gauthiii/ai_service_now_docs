## What Is This?

The "Procedure" subsections describe the action an implementer takes to create or request registry records for governance entities such as AI Models and Datasets within the Govern – Risk and Controls area. Each procedure points to the request action used to add the entity to its registry, with configuration handled through the Govern – Lifecycle topic.

---

## Procedure — AI Model

The AI Model Registry is a catalog of the AI models that support one or more AI systems or use cases.

**Procedure:**

- Request an AI Model (link)

**Configuration:** Refer to Govern – Lifecycle topic.

---

## Procedure — Dataset

The Dataset Registry is an inventory of datasets used to train, test, or operate AI models.

**Procedure:**

- Request a Dataset (link)

**Configuration:** Refer to Govern – Lifecycle topic.

---

## How to Use This in Implementation

To register a governance entity, use the corresponding request action: "Request an AI Model" to add a model to the AI Model Registry, or "Request a Dataset" to add a dataset to the Dataset Registry. In both cases, the underlying configuration is performed through the Govern – Lifecycle topic referenced in this document, which the procedure depends on for setup.
