## What Is This?

This subtopic provides the overview descriptions for the registry entities used in the Govern – Risk and Controls area of AI Control Tower. It explains what the AI Model Registry and the Dataset Registry are, what they catalog, and the governance value they provide. These registries support technical transparency and traceability for the AI systems and use cases being governed.

---

## AI Model — Overview

The AI Model Registry is a catalog of the AI models that support one or more AI systems or use cases.

### Key Value Benefits

It enables technical transparency, version tracking, and model-level governance, ensuring changes to models are controlled and traceable.

Example: A machine-learning classification model used to predict loan default risk, version 2.3, deployed in production.

## Dataset — Overview

The Dataset Registry is an inventory of datasets used to train, test, or operate AI models.

### Key Value Benefits

It supports data transparency, privacy, and quality management by documenting where data comes from and how it is used.

## How to Use This in Implementation

Implementers use the AI Model Registry to catalog every AI model backing an AI system or use case, capturing version information (e.g., version 2.3) and deployment state so model changes remain controlled and traceable. The Dataset Registry should be populated with each dataset used to train, test, or operate those models, documenting data provenance and usage to meet data transparency, privacy, and quality requirements. To request entries, use the "Request an AI Model" and "Request a Dataset" procedures; for configuration of both registries, refer to the Govern – Lifecycle topic in this document.
