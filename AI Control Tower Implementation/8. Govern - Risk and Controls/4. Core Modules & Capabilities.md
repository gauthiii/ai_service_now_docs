## What Is This?

This subtopic introduces the foundational set of core modules that make up an AI risk, compliance, and accountability solution. These modules act as the building blocks for the practice's key governance activities. While each module delivers specific capabilities on its own, the section emphasizes that real value comes only when the modules are configured and integrated together as part of a cohesive operating model.

---

## Overview

The core modules outlined in this section represent the foundational building blocks for managing AI risk, compliance, and accountability. They support key governance activities such as:

- Entity scoping
- AI inventory management
- Risk identification and assessment
- Control implementation
- Compliance monitoring
- Workflow automation
- Reporting

Each module contributes specific capabilities, but value is realized when they are configured and integrated as part of a cohesive operating model.

---

## How to Use This in Implementation

Treat this section as the entry point to the detailed module-by-module guidance that follows. When implementing, do not deploy modules in isolation: plan how entity scoping, AI inventory, risk assessment, controls, compliance monitoring, workflow automation, and reporting connect to one another. Configure and integrate them as a single, cohesive operating model so the governance activities they enable reinforce each other rather than functioning as standalone capabilities.
