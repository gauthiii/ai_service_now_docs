## What Is This?

This is the "Configuration" subsection under the **AI System / Use case** record within the **AI Artifacts Informational Records** of the AI Risk & Compliance solution. It is the part of the implementation guide that addresses how an AI System / Use Case record is configured after it has been requested. It fits alongside the Overview, Key Value Benefits, and Procedure subsections that describe the AI System / Use Case Registry.

---

## Configuration

On this page (p.84), **Configuration** appears only as a heading at the bottom of the page, immediately following the **Procedure** subsection of the **AI System / Use case** record. No body content for the Configuration subsection is present on this page; its detailed content continues on the following page.

For context, the preceding subsections of the AI System / Use case record on this page state:

- **Overview:** The AI System / Use Case Registry is a central inventory of all AI-enabled business use cases within the organization.
- **Key Value Benefits:** It provides visibility, ownership, and accountability for each AI system and serves as the starting point for governance, risk assessment, and compliance activities. Example: An "Automated Loan Approval System" used by the Retail Banking entity to support customer credit decisions.
- **Procedure:** Request an AI use case / system (link).
