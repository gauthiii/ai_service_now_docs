## What Is This?

This subtopic describes the procedure for working with the AI System / Use Case informational record in the AI Risk & Compliance solution. The AI System / Use Case Registry is a central inventory of all AI-enabled business use cases within the organization, and the procedure is the action an implementer takes to register an entry in it.

---

## Procedure: AI System / Use Case

The procedure for the AI System / Use Case informational record is:

- Request an AI use case / system (link)

This procedure relates to the AI System / Use Case Registry, which provides visibility, ownership, and accountability for each AI system and serves as the starting point for governance, risk assessment, and compliance activities.

Example given on the page: An "Automated Loan Approval System" used by the Retail Banking entity to support customer credit decisions.

---

## How to Use This in Implementation

To populate the AI System / Use Case Registry, an implementer follows the procedure to request an AI use case / system via the provided link. This creates the central inventory entry that becomes the starting point for subsequent governance, risk assessment, and compliance activities. After completing the procedure, the implementer continues to the Configuration step for the AI System / Use case record.
