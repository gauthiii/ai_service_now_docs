## What Is This?

This subtopic covers the "Procedure" steps for working with the AI Model Registry and the Dataset Registry within the Govern – Risk and Controls area of AI Control Tower. These procedures describe the action a user takes to request a new AI Model or Dataset entry. They sit alongside each registry's Overview, Key Value Benefits, and Configuration guidance in the governance lifecycle.

---

## AI Model — Procedure

Request an AI Model (link)

Related context for this registry on the page:

- **Overview:** The AI Model Registry is a catalog of the AI models that support one or more AI systems or use cases.
- **Key Value Benefits:** It enables technical transparency, version tracking, and model-level governance, ensuring changes to models are controlled and traceable.
  - Example: A machine-learning classification model used to predict loan default risk, version 2.3, deployed in production.
- **Configuration:** Refer to Govern – Lifecycle topic.

---

## Dataset — Procedure

Request a Dataset (link)

Related context for this registry on the page:

- **Overview:** The Dataset Registry is an inventory of datasets used to train, test, or operate AI models.
- **Key Value Benefits:** It supports data transparency, privacy, and quality management by documenting where data comes from and how it is used.
- **Configuration:** Refer to Govern – Lifecycle topic.

---

## How to Use This in Implementation

To register an AI model, use the "Request an AI Model" procedure (link) to create a catalog entry that supports one or more AI systems or use cases. To register a dataset, use the "Request a Dataset" procedure (link) to add it to the inventory of datasets used to train, test, or operate AI models. For configuration of either registry, refer to the Govern – Lifecycle topic in this document.
