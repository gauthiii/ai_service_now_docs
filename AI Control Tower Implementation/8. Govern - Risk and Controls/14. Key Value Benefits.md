## What Is This?

In the Govern – Risk and Controls area, several registry components (such as the AI Model Registry and the Dataset Registry) document the "Key Value Benefits" they provide to governance. These benefits explain why each registry exists and the governance value it delivers, helping implementers understand the purpose of cataloging models and datasets that support AI systems and use cases.

---

## AI Model — Key Value Benefits

The AI Model Registry is a catalog of the AI models that support one or more AI systems or use cases.

Key Value Benefits:

- It enables technical transparency, version tracking, and model-level governance, ensuring changes to models are controlled and traceable.

Example: A machine-learning classification model used to predict loan default risk, version 2.3, deployed in production.

---

## Dataset — Key Value Benefits

The Dataset Registry is an inventory of datasets used to train, test, or operate AI models.

Key Value Benefits:

- It supports data transparency, privacy, and quality management by documenting where data comes from and how it is used.

---

## How to Use This in Implementation

When configuring the AI Model Registry and Dataset Registry, implementers should record the governance value each entry provides. For AI models, capture version information and deployment status to support version tracking and traceability (for example, a loan-default classification model at version 2.3 in production). For datasets, document data provenance and usage to support transparency, privacy, and quality management. Procedures for requesting an AI Model or a Dataset and their configuration both refer to the Govern – Lifecycle topic in this document.
