## What Is This?

This subtopic explains exactly what the Real-World Time Savings (RWTS) metric measures within AI Control Tower (AICT) value reporting. RWTS estimates human effort avoided through AI-assisted work by converting AI input and output token volumes into time-equivalent estimates. It defines the calculation method and the directional approximation used for assist-based skills, positioning RWTS as a standardized proxy metric for portfolio-level value reporting rather than a precise stopwatch measurement.

---

## What RWTS Measures

RWTS estimates time savings by converting AI input and output token volumes into time-equivalent estimates using standardized reading and writing speed assumptions.

The RWTS calculation is based on:

$$RWTS = \left(\frac{\text{Total Read Tokens}}{\text{Reading Speed}}\right) + \left(\frac{\text{Total Write Tokens}}{\text{Writing Speed}}\right)$$

This produces an estimated average time savings per skill execution, which is why the metric is often presented as an "average RWTS per skill."

For assist-based or non-generative skills, AICT uses a directional guideline of:

$$1\ \text{Assist} \approx 1\ \text{Minute Saved}$$

This approximation is intentionally simplified to support scalable value reporting across large AI portfolios.

---

## How to Use This in Implementation

When configuring value reporting in AICT, implementers should understand that RWTS is derived from token volumes (read and write tokens) divided by standardized reading and writing speeds, producing an average time-saving estimate per skill execution. For assist-based or non-generative skills, apply the directional rule of roughly 1 assist equals 1 minute saved. Treat the resulting figures as standardized, directional proxy estimates for portfolio-level trending — not as exact, per-user productivity measurements — when communicating value to stakeholders.
