## What Is This?

This subtopic defines the setup steps for the Multi-Instance Framework (MIF) Configuration within the Measure – Value practice of AI Control Tower. MIF enables value aggregation across production and sub-production environments, so that value metrics can be consolidated across multiple ServiceNow instances. These steps describe how to register and link instances so AICT can aggregate value data.

---

## Multi-Instance Framework (MIF) Setup Steps

The Multi-Instance Framework (MIF) Configuration enables value aggregation across production and sub-production environments.

Setup Steps:

1. Register production instance in sub-production
2. Wait for auto-approval
3. Configure instance relationships in AICT workspace
4. Add sub-production instances under Multi-instance setup

---

## How to Use This in Implementation

To aggregate AI value data across environments, first register the production instance from within the sub-production environment, then wait for the registration to be auto-approved. Once approved, configure the instance relationships in the AICT workspace and add the sub-production instances under the Multi-instance setup. Completing these steps in order establishes the instance relationships required for MIF to aggregate value across production and sub-production environments.
