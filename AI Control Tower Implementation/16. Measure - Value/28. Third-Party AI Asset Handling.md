## What Is This?

This subtopic describes how AI Control Tower tracks and measures value for third-party AI assets. Usage is captured through a dedicated tracking table, and value is derived from a default calculation model based on invocation activity. It fits within the value measurement workflow, supporting accuracy of value reporting for assets that are not native to the platform.

---

## Tracking and Calculation

- Usage tracked via `sn_ai_disc_ai_usage`
- Default calculation model:

  Invocation Count × 15 minutes × 50% acceptance

> **Note**
> Default assumptions are configurable via custom templates

---

## How to Use This in Implementation

To measure value for third-party AI assets, rely on the `sn_ai_disc_ai_usage` table to track usage. Value is computed using the default calculation model (Invocation Count × 15 minutes × 50% acceptance). If the default assumptions for invocation duration or acceptance rate do not match your organization's expectations, override them by configuring custom templates.
