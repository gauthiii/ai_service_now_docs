## What Is This?

This subtopic provides guidance on how to correctly position and apply the RWTS (Recommended Work Time Saved) value metric during implementation. It clarifies that RWTS is an estimated, comparative metric rather than a precise productivity audit, and advises how to use it appropriately in reporting and executive discussions within AI Control Tower value measurement.

---

## Implementation Considerations for RWTS

- Ensure stakeholders understand that RWTS is an estimated value metric, not a direct productivity audit
- Use RWTS primarily for trend analysis, adoption reporting, and portfolio-level value measurement
- Align executive reporting with the directional nature of the metric
- Avoid positioning RWTS as a contractual or guaranteed time-savings claim
- Supplement RWTS with customer-specific KPIs where higher precision is required

---

## How to Use This in Implementation

When implementing AICT value measurement, treat RWTS as a standardized, directional metric for comparing AI value across skills and use cases. Apply it for trend analysis, adoption reporting, and portfolio-level value measurement rather than as a guaranteed or contractual time-savings figure. Set stakeholder expectations that the metric estimates human effort avoided, and supplement it with customer-specific KPIs whenever higher precision is needed.
