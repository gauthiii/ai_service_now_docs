## What Is This?

This subtopic describes the underlying data architecture that powers AI Control Tower's value and usage measurement. It identifies the key tables that store AI usage and assist metrics, the system behavior and automation (jobs and indicator dependencies) that generate those metrics, and the default value assumptions used to establish baseline value calculations. It sits within the Measure - Value area of implementation, supporting value dashboards.

---

## Data Architecture Tables

| Table | Purpose |
| --- | --- |
| `sys_gen_ai_usage_log` | Stores AI usage and assist metrics |
| `sn_ai_disc_ai_usage` | Tracks third-party AI usage |

---

## System Behavior and Automation

- Daily Jobs
  - Calculate usage and productivity metrics
- Historical Data Job
  - Generates 30 days of baseline data for new instances
- Indicator Dependency
  - Value dashboards depend on successful PA indicator execution

## How to Use This in Implementation

When implementing value measurement, rely on `sys_gen_ai_usage_log` for AI usage and assist metrics and `sn_ai_disc_ai_usage` for third-party AI usage tracking. Be aware that value dashboards depend on successful PA indicator execution, so verify PA indicators run correctly. Daily jobs compute usage and productivity metrics, and for new instances a historical data job seeds 30 days of baseline data. Start by establishing baseline logic with the default value assumptions (15-minute time saved, 50% acceptance rate); these are OOB template assumptions and can be overridden via Value Templates as needed.
