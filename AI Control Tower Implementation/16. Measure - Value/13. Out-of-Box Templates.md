## What Is This?

Value Templates define the calculation logic used to measure AI-driven productivity and value realization in AI Control Tower. AICT ships with a set of out-of-box Value Templates covering the main AI asset types, and also lets implementers add, edit, and map new templates. This subtopic covers the provided out-of-box templates and the workflow for creating, editing, and mapping value templates.

## Adding a New Template

1. Go to: Workspaces > AI Control Tower -> AI Asset -> Productivity -> Value templates
2. Click: New Template
3. Fill in required fields and click next
4. Add mapping: select the AI system type and AI system to be mapped.
5. Preview the value estimate using the test feature for all mapped AI systems.
6. Select the AI system for preview and click Validate and calculate
7. If the result are satisfactory, Click Publish

Note: Upon Publish, Value daily job will trigger and calculate productivity and usage metrics surfacing in the Value dashboard.

## Edit an Existing Template

1. Navigate to: Workspaces > AI Control Tower -> AI Asset -> Productivity -> Value templates
2. Select a template to edit, AI Agent change
   - Behaviour depends on the template current state
3. Edit fields as needed
4. Save as Draft → Validate → Publish

Permitted actions depend on the template state:

| Template State | Permitted Actions |
|---|---|
| Published | Metric fields cannot be edited. Template details (Department, Description) can be updated. Mapping records can be added or removed. Use the Duplicate action to clone the template (with or without mapping records). Cloned templates start in Draft state. |
| Draft | All fields (metrics, template details, mapping records) are editable. Draft templates can be deleted or published. |

## Template Mapping

1. Navigate to: Workspaces > AI Control Tower -> AI Asset -> Productivity -> Template mapping
2. Click New record,
   a. Select the template, the persona filed is auto-populated from selected template
   b. Select the AI system type

## How to Use This in Implementation

Implementers can start from the four out-of-box templates (Third-party Assets, ServiceNow Skills, ServiceNow Agents, Agentic Workflows) or build custom ones via the Value templates workspace. Creating a template follows a 3-step wizard — Formula (build the calculation from persona, usage, time, and acceptance), Map (select AI system type and assets), and Test (validate and calculate the value estimate) — before publishing. Publishing triggers the Value daily job, which calculates productivity and usage metrics surfacing in the Value dashboard. Note the state-based editing constraints: once a template is Published, metric fields are locked (only template details and mappings change, and you must Duplicate to clone into a new Draft), whereas Draft templates are fully editable and can be deleted. Use Template mapping to associate templates with AI system types, where the persona field auto-populates from the selected template.
