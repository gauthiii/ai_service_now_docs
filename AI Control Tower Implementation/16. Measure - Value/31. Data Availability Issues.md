## What Is This?

This subtopic addresses troubleshooting steps for when Value Management dashboards display no data. It explains the corrective actions to take and the sequential dependency chain that must succeed for value dashboards to populate, helping implementers diagnose and resolve missing or incomplete data visibility.

---

## Troubleshooting: If Dashboards Show No Data

- Verify PA indicators have executed
- Trigger historical data job manually
- Confirm scheduled jobs are active

---

## Key Takeaways

- Value Management is template-driven and indicator-dependent
- Accurate measurement requires:
  - Proper mapping
  - Valid PA indicators
  - Completed daily jobs
- Governance is enforced through:
  - Role-based ownership
  - Template lifecycle controls
  - Standardized calculation models

---

## Implementation Note

Value dashboard population depends on a sequential dependency chain:

1. PA indicators execute
2. Historical data is generated
3. Value calculations are applied
4. Dashboards are populated

Failure at any stage will result in missing or incomplete data visibility.

### Diagram/Screenshot: Sequential Dependency Chain

The implementation note describes a linear, four-stage dependency flow:

```
PA indicators execute
        ↓
Historical data is generated
        ↓
Value calculations are applied
        ↓
Dashboards are populated
```

Each stage depends on the successful completion of the prior stage; a failure anywhere along the chain results in missing or incomplete data visibility.

---

## How to Use This in Implementation

When Value Management dashboards show no data, work through the dependency chain in order. First, verify that PA indicators have executed (the foundation of the chain). If historical data is missing, trigger the historical data job manually. Confirm that scheduled jobs remain active so that calculations and dashboard population continue on schedule. Because measurement is template-driven and indicator-dependent, ensure proper mapping, valid PA indicators, and completed daily jobs before troubleshooting downstream. Diagnosing failures stage-by-stage isolates where the chain broke.
