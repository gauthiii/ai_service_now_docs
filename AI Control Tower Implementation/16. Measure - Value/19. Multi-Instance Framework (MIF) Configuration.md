# Multi-Instance Framework (MIF) Configuration

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Measure - Value → Multi-Instance Framework (MIF) Configuration
> Source: AI-Control-Tower.pdf, p.207

---

## What Is This?

The Multi-Instance Framework (MIF) Configuration enables value aggregation across production and sub-production environments in AI Control Tower. It allows a production instance and its associated sub-production instances to be linked so that value metrics can be measured and rolled up across the full set of instances. This is part of the Measure - Value workstream of an AICT implementation.

---

## Purpose

Enables value aggregation across production and sub-production environments.

---

## Setup Steps

1. Register production instance in sub-production
2. Wait for auto-approval
3. Configure instance relationships in AICT workspace
4. Add sub-production instances under Multi-instance setup

---

## How to Use This in Implementation

To aggregate AICT value metrics across multiple environments, first register the production instance from within the sub-production instance, then wait for the registration to be auto-approved. Once approved, configure the relationships between the instances in the AICT workspace, and finally add the sub-production instances under the Multi-instance setup so that value data flows together. This depends on having both production and sub-production environments available and on the registration completing approval before relationships can be configured.
