## What Is This?

This subtopic covers how AI Control Tower value data is initialized and how historical data is generated to populate value dashboards. It describes the behavior of the historical data generation job, its execution cadence, its dependencies, and when it automatically deactivates. This is where the bulk of the data-generation content belongs in the implementation.

---

## Data Initialization and Historical Data Generation

Include:

- AIValue Generate Historical Data job behavior
- Hourly execution logic
- Dependency on PA indicators
- Auto-deactivation after 30-day backfill

---

## How to Use This in Implementation

When initializing AI Control Tower value reporting, ensure the AIValue Generate Historical Data job is configured according to its expected behavior. Account for its hourly execution logic and its dependency on PA indicators, which must be available for the job to function. Note that the job auto-deactivates after the 30-day backfill is complete, so plan initialization timing accordingly.
