# Process Flow Summary

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Cross Product Integration - CMDB → Process Flow Summary
> Source: AI-Control-Tower.pdf, p.240

---

## What Is This?

The Process Flow Summary maps the end-to-end lifecycle of an AI asset across AICT and CMDB integration, from initial ideation through production. It aligns each stage with its corresponding CSDM (Common Service Data Model) domain, the key actions performed, and the AICT artifacts produced. This serves as a reference for understanding how governance, discovery, and CMDB relationships progress through the deployment journey.

---

## Process Flow Summary Table

| Stage | CSDM Domain | Key Actions | AICT Artifacts |
|-------|-------------|-------------|----------------|
| Ideation | Ideation & Strategy | AI demand captured in SPM | SPM demand record created |
| ARB (Gate 1) | Design & Planning | Business application registered. AI system and models created. Dependencies documented in EA. | CMDB CIs created. AICT intake initiated. |
| Build & Test | Build & Integration | Development, testing, risk assessment, and compliance review. | AICT lifecycle: Assess, Build & Test stages. |
| Release (Gate 2) | Service Delivery | Final governance fields validated. Discovery confirmed. Deployment approved. | AICT records finalized. Discovery sources confirmed. |
| Production | Service Delivery | Live. Monitoring, performance measurement, and value tracking active. | Deploy stage. Value metrics active. |

---

## How to Use This in Implementation

Implementers use this summary to coordinate AICT governance with CMDB/CSDM stage progression. Each stage has clear entry conditions and produced artifacts: capture demand in SPM at ideation, register the business application and create CMDB CIs at the ARB (Gate 1) review, run risk assessment and compliance review during Build & Test, validate governance fields and confirm discovery at the Release (Gate 2) review, and activate monitoring and value metrics in Production. Use the CSDM domain column to ensure each artifact lands in the correct data model domain as the asset advances.
