# Blending Discovery Sources

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Cross Product Integration - CMDB → Blending Discovery Sources
> Source: AI-Control-Tower.pdf, p.242

---

## What Is This?

This subtopic addresses how to reconcile data when multiple discovery mechanisms — AICT discovery connectors and ITOM/Service Graph connectors — both return information about the same infrastructure. It explains the need to establish authoritative sources for attributes to maintain CMDB data quality during cross-product CMDB integration.

---

## Reconciling Multiple Discovery Sources

AICT discovery connectors and ITOM/Service Graph connectors serve different purposes and may both return data about the same infrastructure. When both are active, define which source is authoritative for each attribute class to prevent reconciliation conflicts and data quality degradation.

---

## How to Use This in Implementation

When both AICT discovery connectors and ITOM/Service Graph connectors are enabled in an environment, an implementer must explicitly designate, for each attribute class, which discovery source is authoritative. Failing to do so leads to reconciliation conflicts and degraded CMDB data quality, since both sources may populate overlapping data about the same infrastructure.
