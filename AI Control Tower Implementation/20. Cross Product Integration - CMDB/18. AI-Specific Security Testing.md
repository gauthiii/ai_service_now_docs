## What Is This?

AI-Specific Security Testing addresses threat vectors that are unique to AI systems and that must be folded into an organization's existing security operations and vulnerability management processes. In the AICT implementation, evidence of this testing is captured against AI system records so it can be referenced during governance and release validation. It ensures AI-specific risks are not overlooked by traditional security testing alone.

---

## AI-Specific Security Testing

AI-specific threat vectors—including prompt injection, data leakage, and model poisoning—require integration into existing SecOps and vulnerability management processes. Evidence of security testing should be attached to AI system records in AICT and included in release gate validation.

## How to Use This in Implementation

When implementing AICT, integrate AI-specific threat testing (prompt injection, data leakage, model poisoning) into your existing SecOps and vulnerability management workflows rather than treating them as separate efforts. Attach the resulting security testing evidence to the relevant AI system records in AICT, and incorporate that evidence into release gate validation so that AI systems cannot pass through release gates without demonstrated AI-specific security testing. Use the reference resources above for product, AI inventory, and governance workspace documentation.
