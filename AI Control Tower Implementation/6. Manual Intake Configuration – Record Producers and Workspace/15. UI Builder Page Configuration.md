## What Is This?

The UI Builder Page Configuration defines the reusable UI Builder page that drives the manual intake experience in AI Control Tower. A single parameterized page (`Intake-form`) is used to render the appropriate intake form based on the type of AI asset being created. It fits into the manual intake configuration by providing the front-end form rendered when users add AI systems, models, datasets, or prompts.

---

## UI Builder Page Configuration

The intake experience is driven by a reusable UI Builder page.

| Attribute | Configuration |
|---|---|
| Page Name | Intake-form |
| Variant | Default |
| Application Scope | AI Asset Management |
| URL Path | `/intake-form/{table}/{sys_id}` |

---

## Page Parameters and Dynamic Behavior

The Intake-form page is parameterized to dynamically render forms based on the AI asset type being created.

- Parameters:
  - `table` → Defines the asset type
  - `sys_id` → Set to `-1` for new records

---

## Asset-Specific Parameter Mapping

| Action | Parameter Value (table) | Purpose |
|---|---|---|
| Add AI System | `alm_ai_system_digital_asset` | Captures AI System (use case) details |
| Add AI Model | `alm_ai_model_digital_asset` | Captures AI Model details |
| Add Dataset | `alm_ai_dataset_digital_asset` | Captures dataset details |
| Add Prompt | `alm_ai_prompt_digital_asset` | Captures prompt details |

---

## How to Use This in Implementation

To configure the manual intake experience, implementers reference the reusable `Intake-form` UI Builder page (Default variant) in the `AI Asset Management` application scope, accessed via the URL path `/intake-form/{table}/{sys_id}`. The page is parameterized so a single page handles all AI asset types: pass the `table` parameter to select the asset type and set `sys_id` to `-1` when creating a new record. Map each intake action to its corresponding table value — `alm_ai_system_digital_asset` for AI Systems, `alm_ai_model_digital_asset` for AI Models, `alm_ai_dataset_digital_asset` for Datasets, and `alm_ai_prompt_digital_asset` for Prompts — so the correct form is rendered for the asset being captured.
