## What Is This?

This subtopic defines the strategy for how AI assets are brought into (intake) and onboarded onto the AI Control Tower platform. It recommends a hybrid intake model combining Record Producers and Workspace/API-based automation, and specifies the governance metadata and lifecycle defaults that intake must enforce. It fits into manual intake configuration as the guiding strategy that shapes how record producers and workspace entry points are designed.

---

## Intake and Onboarding Strategy

- Use a hybrid intake model:
  - Record Producers → structured, auditable intake
- Workspace → operational efficiency
  - APIs → large-scale automation
- Standardize intake forms with minimum required governance attributes
- Ensure mandatory metadata capture:
  - Ownership
  - Business purpose
  - Risk classification
  - Data sources
- Default lifecycle states appropriately:
  - Draft → new assets
  - Deployed → existing assets
- Prevent bypass of intake via governed entry points only

---

## How to Use This in Implementation

When configuring manual intake, apply a hybrid model: use Record Producers where structured, auditable intake is needed; use Workspace for operational efficiency; and use APIs for large-scale automation. Standardize all intake forms around a minimum set of governance attributes, and enforce mandatory capture of ownership, business purpose, risk classification, and data sources. Set lifecycle defaults so new assets begin in Draft and existing assets default to Deployed. Lock down entry so that all intake flows through governed entry points only, preventing any bypass of the intake process.
