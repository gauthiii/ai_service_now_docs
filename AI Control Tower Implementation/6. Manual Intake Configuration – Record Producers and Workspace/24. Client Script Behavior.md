## What Is This?

This subtopic describes the role that client scripts play in the manual intake forms used to onboard AI assets in AI Control Tower. Client scripts drive dynamic form behavior, govern stepper progression, and—most importantly—handle the model table assignment logic that switches the intake experience between different AI asset types. They are a core part of how the record producers and workspace render and behave at runtime.

---

## Client Script Behavior

Client scripts play a critical role in:

- Dynamically setting the model table
- Controlling form behavior based on asset type
- Managing stepper progression
- Updating client state parameters

### Note

The model table assignment logic is central to switching between AI System, Model, Dataset, and Prompt intake experiences.

---

## How to Use This in Implementation

When configuring or customizing the manual intake forms, implementers should account for the client scripts that dynamically determine and assign the model table, since this logic drives which intake experience (AI System, Model, Dataset, or Prompt) is presented. Client scripts also manage form behavior by asset type, stepper progression, and client state parameters—so any customization to form flow or asset-type handling must preserve or extend this scripting logic rather than bypass it.
