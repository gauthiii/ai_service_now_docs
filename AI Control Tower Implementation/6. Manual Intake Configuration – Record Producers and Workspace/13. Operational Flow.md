## What Is This?

The Operational Flow describes the end-to-end sequence a user follows to manually add an asset to the AI Asset Inventory from the AI Control Tower Workspace. It captures the step-by-step path from navigating the workspace through to the asset entering the governance lifecycle. This sits within the manual intake configuration and frames how the workspace-based intake actually plays out for an end user.

---

## Operational Flow Steps

1. User navigates to AI Control Tower Workspace
2. Selects asset type (e.g., AI System)
3. Initiates "Add" action
4. Completes structured form
5. Asset is created and enters governance lifecycle

## Workspace Framework Context

AI Control Tower leverages the Now Experience Framework (UI Builder) to deliver a modern, role-based workspace for managing AI Asset Inventory. This context introduces the core UI Builder configurations that enable asset intake and management from the workspace perspective, which lead into the subsequent UI Builder Experience Configuration.

---

## How to Use This in Implementation

To support this flow, ensure the AI Control Tower Workspace is available to users and that the AI asset inventory lists (AI systems, AI models, Prompts, Datasets, MCP servers) and their "Add" actions are configured. The workspace is built on the Now Experience Framework (UI Builder), so the intake experience depends on UI Builder configuration rather than the Service Catalog. Once a user completes the structured form via the "Add" action, the created asset automatically enters the governance lifecycle (e.g., Assess phase, Draft state, In review status), so confirm lifecycle and risk-classification defaults are in place.
