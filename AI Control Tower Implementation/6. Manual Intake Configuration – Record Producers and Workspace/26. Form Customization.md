---

## What Is This?

Form Customization is one of the Configuration Recommendations for the manual intake forms used in AI Control Tower. It provides guidance on how implementers should safely tailor the intake form to capture additional information while preserving the out-of-the-box governance baseline. It fits within the broader Configuration Recommendations alongside State Management.

---

## Configuration Recommendations: Form Customization

- Modify the "Intake" view to add new attributes
- Avoid removing out-of-the-box (OOTB) fields, as they represent minimum governance requirements

---

## How to Use This in Implementation

When extending the intake experience, implementers should add any new attributes through the "Intake" view rather than altering the underlying form structure directly. The out-of-the-box fields must be retained because they represent the minimum governance requirements; removing them would compromise the governance baseline of the intake process.
