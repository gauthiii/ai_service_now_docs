## What Is This?

The AI Control Tower Workspace is built on the Now Experience Framework (UI Builder) to deliver a modern, role-based workspace for managing the AI Asset Inventory. This subtopic defines the core UI Builder experience attributes that establish the workspace shell, its scope, URL, and the role required to access it. It is the foundation on which asset intake and management from the workspace perspective are configured.

---

## Workspace Experience Overview

AI Control Tower leverages the Now Experience Framework (UI Builder) to deliver a modern, role-based workspace for managing AI Asset Inventory. This section outlines the core UI Builder configurations that enable asset intake and management from the workspace perspective.

The AI Control Tower Workspace is configured as a UI Builder experience with the following attributes:

| Attribute | Configuration |
|---|---|
| Workspace | AI Control Tower |
| Application Scope | AI Control Tower Core |
| App Shell | Workspace App Shell |
| Experience Name | AI Control Tower |
| URL Path | `ai-governance-workspace` |
| Role Required | `sn_ai_governance.workspace_user` |

## How to Use This in Implementation

To stand up the AI Control Tower Workspace, implementers configure the UI Builder experience using the attributes above: set the workspace to "AI Control Tower" within the "AI Control Tower Core" application scope, base it on the Workspace App Shell, and expose it at the URL path `ai-governance-workspace`. Access is gated by the `sn_ai_governance.workspace_user` role, so users must hold that role to open the workspace. Once the experience is in place, the workspace surfaces the AI asset inventory lists (AI systems, models, prompts, datasets, MCP servers) and the "Add" actions used for manual asset intake.
