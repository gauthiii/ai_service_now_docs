## What Is This?

This subtopic describes the Key System Actions triggered by the "Record Producer: Request a Dataset" intake form, which captures datasets used in training and evaluation. It explains which records the system automatically creates when the dataset intake form is submitted, and how those records are linked to enable lifecycle governance and traceability.

---

## Key System Actions for "Request a Dataset"

The dataset intake form performs the following system actions:

- **Creates** AI Dataset Digital Asset
  - `[alm_ai_dataset_digital_asset]`
  - Defaults category to AI Dataset
  - Captures dataset relationships
- **Creates** AI Dataset Product Model
  - `[cmdb_ai_dataset_product_model]`
  - Establishes dataset attributes and lifecycle state
  - Links product model to digital asset
- **Creates** AI Asset Governance Details
  - Enables lifecycle governance and traceability

> **Note:** No Configuration Item (CI) is created during intake.

---

## How to Use This in Implementation

When an implementer configures the "Request a Dataset" record producer, submission of the form automatically generates the AI Dataset Digital Asset (`alm_ai_dataset_digital_asset`), the AI Dataset Product Model (`cmdb_ai_dataset_product_model`), and AI Asset Governance Details. The digital asset defaults its category to AI Dataset and captures dataset relationships, while the product model establishes dataset attributes, sets lifecycle state, and links back to the digital asset to support governance and traceability. Note that no Configuration Item (CI) is created during this intake, so any CI-based tracking must be handled separately.
