# State Management

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Manual Intake Configuration – Record Producers and Workspace → State Management
> Source: AI-Control-Tower.pdf, p.59

---

## What Is This?

State Management is a configuration recommendation within the manual intake setup for AI assets. It guides implementers on which lifecycle state to assign to an AI asset when onboarding it through the intake experience, based on whether the asset is newly created or already exists in the environment.

---

## State Management Guidance

- Use **Draft** for new (undeployed) AI assets
- Use **Deployed** for existing assets being onboarded

---

## How to Use This in Implementation

When onboarding AI assets through manual intake, set the asset state appropriately: assign the Draft state to assets that are new and not yet deployed, and assign the Deployed state to existing assets that are being onboarded into AI Control Tower. This recommendation accompanies form customization guidance (modifying the "Intake" view to add attributes while avoiding removal of out-of-the-box governance fields).
