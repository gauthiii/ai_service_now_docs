## What Is This?

This subtopic defines the access level and roles required for users to use the manual intake record producers in AI Control Tower. It establishes that record producers are available to users with AI Request User access, and explains how this access maps to roles so AI Asset Owners can manage assets across their lifecycle within governed workflows.

---

## Access and Role Requirements

All record producers are available to users with **AI Request User** access, which includes:

- **Role:** `sn_grc_ai_gov.ai_risk_and_compliance_business_user`
- **Inherited by:** `sn_ai_asset_mgmt.ai_asset_owner`

This ensures that AI Asset Owners can manage assets across their lifecycle—from intake through retirement—within governed workflows.

---

## How to Use This in Implementation

When configuring manual intake, grant users the AI Request User access so they can access all record producers. Assign the `sn_grc_ai_gov.ai_risk_and_compliance_business_user` role; note that this role is inherited by `sn_ai_asset_mgmt.ai_asset_owner`, so AI Asset Owners automatically gain the ability to manage assets throughout their lifecycle within governed workflows. Verify role assignments before expecting users to submit intake requests through the Service Catalog or AI Control Tower Workspace.
