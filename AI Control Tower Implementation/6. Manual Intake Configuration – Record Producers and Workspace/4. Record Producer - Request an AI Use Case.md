## What Is This?

"Request an AI Use Case" is one of the manual-intake record producers delivered with AI Control Tower. This intake form captures AI use case details and creates the foundational AI System Digital Asset, which serves as the anchor record for governing an AI system through its lifecycle. It is delivered through the Service Catalog intake channel and is available to users with AI Request User access.

---

## Record Producer Overview

This intake form captures AI use case details and creates the foundational AI System Digital Asset.

From the delivered record producer catalog, this producer is defined as:

| Name | Table | Intake Channel |
|------|-------|----------------|
| Request an AI Use Case | AI System Digital Asset `[alm_ai_system_digital_asset]` | Service Catalog |

---

## Key System Actions

When this record producer is submitted, it performs the following key system actions:

- **Creates** AI System Digital Asset
  - `[alm_ai_system_digital_asset]`
  - Sets class, state, ownership, and model category
  - Stores associated models and datasets
- **Creates** AI System Component Product Model
  - `[cmdb_ai_system_component_product_model]`

> Note: On page 53 the description of the second created record (AI System Component Product Model) continues beyond the page boundary; only the content shown above belongs to this page.

---

## How to Use This in Implementation

To use this record producer, an implementer publishes it through the Service Catalog so that users with AI Request User access can request a new AI use case. Submission of the form creates the foundational AI System Digital Asset record (`alm_ai_system_digital_asset`) — setting its class, state, ownership, and model category, and storing associated models and datasets — and also creates an AI System Component Product Model record (`cmdb_ai_system_component_product_model`). Because this asset is the foundational anchor for the AI system, ensure ownership and model category values are captured accurately at intake so downstream governance and lifecycle workflows operate on correct data.
