## What Is This?

Stepper Configuration covers how the multi-step (stepper) flow of the manual intake record producer is controlled and modified. The stepper is driven by client state parameters, and any change to it must respect the dependencies enforced through client scripts. This fits into the Manual Intake Configuration work where record producers and workspace experiences are tailored.

---

## Stepper Configuration

- Controlled via client state parameters
- Validate dependencies with client scripts before making changes

---

## How to Use This in Implementation

When customizing the intake stepper, implementers manage its behavior through client state parameters rather than direct edits to the flow. Before making any change, validate the dependencies enforced by client scripts to avoid breaking the step sequence or downstream logic.
