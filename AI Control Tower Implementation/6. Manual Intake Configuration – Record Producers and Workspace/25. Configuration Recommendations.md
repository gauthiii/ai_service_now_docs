## What Is This?

This subtopic provides practical recommendations for configuring the manual intake experience for AI assets in AI Control Tower. It covers how to safely customize the intake form and how to set the lifecycle state of AI assets during onboarding. These guidelines help implementers extend intake while preserving the minimum governance requirements baked into the out-of-the-box configuration.

---

## Form Customization

- Modify the "Intake" view to add new attributes
- Avoid removing out-of-box (OOTB) fields, as they represent minimum governance requirements

---

## State Management

- Use Draft for new (undeployed) AI assets
- Use Deployed for existing assets being onboarded

---

## How to Use This in Implementation

When extending the manual intake experience, customize through the "Intake" view to add any new attributes your organization requires, rather than altering the underlying form structure. Leave the out-of-box fields in place, since they enforce the minimum governance requirements expected by AI Control Tower. When onboarding assets, set the state appropriately: use the Draft state for new AI assets that have not yet been deployed, and use the Deployed state for existing assets that are already in use and being brought under governance.
