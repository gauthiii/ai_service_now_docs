## What Is This?

This subtopic describes how the Form Controller dynamically renders intake forms in AI Control Tower based on the target table, layout, and view configuration. It explains the role of client-side scripts in driving form behavior, and provides configuration recommendations for customizing intake forms and managing record state. It fits into the manual intake configuration of AI assets (AI System, Model, Dataset, and Prompt).

---

## Form Controller

The Form Controller dynamically determines:

- Target table
- Form layout
  - View configuration

Form components are driven by:

- Table parameter (`table`)
- Configured views (e.g., *Intake view*)
- Client-side scripts

---

## Client Script Behavior

Client scripts play a critical role in:

- Dynamically setting the model table
- Controlling form behavior based on asset type
- Managing stepper progression
- Updating client state parameters

> **Note**
> The model table assignment logic is central to switching between AI System, Model, Dataset, and Prompt intake experiences.

---

## Configuration Recommendations

### Form Customization

- Modify the "Intake" view to add new attributes
- Avoid removing out-of-box (OOTB) fields, as they represent minimum governance requirements

### State Management

- Use Draft for new (undeployed) AI assets
- Use Deployed for existing assets being onboarded

---

## How to Use This in Implementation

When configuring manual intake, rely on the Form Controller to render the correct form by passing the `table` parameter and the appropriate configured view (such as the Intake view). Use client scripts to dynamically set the model table, control form behavior by asset type, manage stepper progression, and update client state parameters — this logic is what switches the experience between AI System, Model, Dataset, and Prompt. To customize, extend the Intake view with new attributes rather than removing OOTB fields (which enforce minimum governance requirements), and set asset state to Draft for new undeployed assets or Deployed for assets being onboarded.
