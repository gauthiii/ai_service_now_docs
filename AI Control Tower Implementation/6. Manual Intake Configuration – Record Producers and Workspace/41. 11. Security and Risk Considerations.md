## What Is This?

This subtopic covers the security and risk considerations to apply when configuring manual intake for AI assets in AI Control Tower (AICT). It focuses on tracking data access, maintaining visibility into sensitive data and impactful model outputs, and applying controls for responsible AI. It ensures the intake process aligns with enterprise security and data governance policies.

---

## Security and Risk Considerations

- Track data access via:
  - MCP Servers
  - External integrations
- Ensure visibility into:
  - Sensitive datasets
  - Model outputs impacting decisions
- Apply controls for:
  - Bias and fairness
  - Data privacy
  - Model explainability
- Align with enterprise security and data governance policies

---

## How to Use This in Implementation

When configuring manual intake, implementers should ensure that data access paths — including MCP Servers and external integrations — are tracked, and that AICT provides visibility into sensitive datasets and into model outputs that impact decisions. Apply governance controls covering bias and fairness, data privacy, and model explainability, and confirm that the overall configuration aligns with the organization's existing enterprise security and data governance policies.
