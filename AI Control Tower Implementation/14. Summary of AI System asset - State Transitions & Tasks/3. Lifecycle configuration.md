## What Is This?

Lifecycle configuration is part of the AICT Customization Guide. It explains how implementers can add, delete, or modify the lifecycle phases that govern an AI Asset as it moves through its governance lifecycle. This customization is performed by an AI steward against the `sn_ai_governance_lifecycle` table and the associated playbook.

---

## Add/Delete/Modify Lifecycle Phases

Rough notes:

- Deletion of an OOB phase is not recommended at all
- It is ideal to not update the Order of the OOB phases. I.e. do not place Assess after Build. For more information on editing existing phases please refer to the "Playbook: AI Asset Lifecycle" section
- To add new lifecycle phases, an AI steward can add more entries to the `sn_ai_governance_lifecycle` table, then update the playbook

## How to Use This in Implementation

To customize lifecycle phases, an AI steward adds entries to the `sn_ai_governance_lifecycle` (AI Asset Lifecycles) table and then updates the corresponding playbook so the new phase is reflected in the governance flow. Implementers should avoid deleting out-of-the-box (OOB) phases and should preserve the order of OOB phases (for example, do not place Assess after Build). Refer to the "Playbook: AI Asset Lifecycle" section for guidance on editing existing phases.
