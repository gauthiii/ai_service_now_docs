## What Is This?

This subtopic lists the new tasks that an implementer can add to a particular phase of the AI Asset Onboarding lifecycle within Workflow Studio. It identifies the lifecycle tasks, jobs, and mapping activities that extend a phase, indicating which are demonstration (Demo), out-of-the-box (OOTB), or system-defined. It fits into the configuration and extension of AI System asset state transitions and their associated tasks.

---

## New Tasks to Add to a Phase

The following tasks can be added to a particular phase:

1. Onboarding and Offboarding Lifecycle Tasks (Demo vs OOTB / System-defined) (MS1)
2. Scheduler Jobs, if any
3. Synchronization Jobs, if any
4. Map AI Risk Statements to AI System,
5. Map AI Control objectives to AI System
6. Map Use and Purpose to AI system **(MS2)**

## How to Use This in Implementation

To extend a phase, an implementer opens the AI Asset Onboarding Playbook in Workflow Studio, selects the target phase node, and adds the relevant tasks listed above. This includes adding onboarding/offboarding lifecycle tasks, any scheduler or synchronization jobs, and the mapping tasks that relate AI Risk Statements, AI Control objectives, and Use and Purpose to the AI System. Each task node's UI Layout can be configured through the properties panel (table, title, associated record/table, empty-state labels, flow count, and list query) before saving and activating the workflow.
