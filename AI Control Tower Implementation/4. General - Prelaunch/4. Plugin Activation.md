# Plugin Activation

---

## What Is This?

Plugin Activation is a prelaunch step that ensures all required ServiceNow plugins for AI Control Tower are installed and active before configuration proceeds. The plugins must be activated in the order listed, and each one should be confirmed active before moving on to the next. This establishes the foundational platform capabilities (control tower, governance, asset management, risk and compliance, and case management) needed for the rest of the implementation.

---

## Plugin Activation

Activate required plugins in the order listed. Confirm each plugin is active before proceeding to the next.

---

## How to Use This in Implementation

During prelaunch, an implementer activates each plugin in the exact sequence shown, starting with AI Control Tower (com.sn_aict) and ending with AI Risk & Compliance Integration with Control Tower (com.sn_grc_ai_irm_intg). All six plugins are Required. Confirm each plugin reports as active before activating the next, since later plugins may depend on earlier ones. Complete this step before Core Role Provisioning and other configuration tasks.
