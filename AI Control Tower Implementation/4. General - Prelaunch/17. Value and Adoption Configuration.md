## What Is This?

Value and Adoption Configuration covers setting up the Value and Adoption dashboards in AI Control Tower to track AI impact and usage metrics. It ensures the data pipeline (daily value calculation job and historical backfill) is running so the dashboards display meaningful results. This is a prelaunch step that surfaces productivity, ROI, and adoption insights for deployed AI systems.

---

## Overview

Configure the Value and Adoption dashboards to track AI impact and usage metrics. The daily value calculation job must be active and the historical backfill script must have run before meaningful data will display.

---

## Value and Adoption Checklist

| ✓ | Value and Adoption |
| --- | --- |
| ☐ | Verify the daily value calculation scheduled job is active (runs daily, 1–3pm) |
| ☐ | Confirm the historical data backfill script has run on installation (populates past 30 days) |
| ☐ | Review and configure value templates for each AI system (defaults: 15 min time saved per invocation, 50% acceptance rate for third-party agents) |
| ☐ | Configure custom value metrics if the organization requires goals beyond the four out-of-the-box categories (ROI, Productivity, Cost Avoidance, Risk Reduction) |
| ☐ | If using multi-instance deployment, configure the Multi-Instance Framework (MIF) for centralized value rollup |
| ☐ | Verify the Value tab displays total productivity gained, average AI users, and top AI systems by value |
| ☐ | Verify the Adoption tab displays usage data (daily AI actions, daily unique users, department-level usage) |

---

## How to Use This in Implementation

Before configuring dashboards, ensure the daily value calculation scheduled job is active (runs daily between 1–3pm) and confirm the historical data backfill script ran on installation (populates the past 30 days) — these are prerequisites for meaningful data. Review and adjust the value templates per AI system, noting the defaults of 15 minutes time saved per invocation and a 50% acceptance rate for third-party agents. The four out-of-the-box value categories are ROI, Productivity, Cost Avoidance, and Risk Reduction; configure custom value metrics only if goals fall outside these. For multi-instance deployments, set up the Multi-Instance Framework (MIF) for centralized value rollup. Finally, validate that the Value tab (total productivity gained, average AI users, top AI systems by value) and the Adoption tab (daily AI actions, daily unique users, department-level usage) render correctly.
