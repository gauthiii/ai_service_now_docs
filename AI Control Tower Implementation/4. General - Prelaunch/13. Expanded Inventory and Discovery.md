## What Is This?

Expanded Inventory and Discovery is a prelaunch activity that grows the AI asset inventory beyond its initial population to include external AI platforms, API integrations, and additional data sources. It ensures the foundational inventory is stable before scope is expanded. This step prepares AICT to discover, import, and govern AI assets across the broader enterprise landscape.

---

## Overview

Expand the AI asset inventory beyond the initial population to include external AI platforms, API integrations, and additional data sources. Confirm foundational inventory is stable before expanding scope.

| ☐ | Validate that all discovered and imported assets appear correctly in the inventory with proper metadata and relationships |

## How to Use This in Implementation

Before expanding inventory scope, confirm the foundational AI asset inventory is stable. Begin by defining guardrails that distinguish in-scope from out-of-scope assets (excluded vendor tools, prohibited AI systems, and in-production systems requiring governance). Identify integration points with existing enterprise systems (CRM, ITSM, HRIS, SAM, and vendor/procurement management). Configure and test hyperscaler discovery connections for Amazon Bedrock, Azure AI Foundry, Copilot Studio, and GCP Vertex AI, verifying discovered assets populate the CMDB. For programmatic integration via the AI Assets API, activate the Asset Classes plugin (sn_ent), and configure custom Scripted REST API endpoints for advanced needs. When importing from external sources (JDBC, OIDC, LDAP, REST, file), configure data sources, import set tables, transform maps, and scheduled imports as applicable. Finally, validate that all discovered and imported assets appear correctly with proper metadata and relationships.
