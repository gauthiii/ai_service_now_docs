# Initial Inventory Scope

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: General - Prelaunch → Initial Inventory Scope
> Source: AI-Control-Tower.pdf, p.30

---

## What Is This?

Initial Inventory Scope defines the initial boundaries for the AI asset inventory before populating records. It is a prelaunch planning step that establishes what qualifies as an AI asset, which assets to populate first, and how they will be intake into the inventory. A scoped initial deployment reduces complexity and can be expanded during the Full Functionality phase.

---

## Inventory Scope Checklist

The following checklist defines the scope decisions for the initial inventory.

| ✓ | Inventory Scope |
|---|---|
| ☐ | Define what qualifies as an AI asset for the initial deployment (e.g., start with AI systems and models; expand to datasets and prompts in a later phase) |
| ☐ | Identify the initial set of AI assets to populate in the inventory |
| ☐ | Determine the primary intake method for initial population: manual intake via workspace, Record Producers, AI Discovery connectors, or automated Now Assist sync |
| ☐ | Confirm whether Now Assist auto-discovery should be active (enabled by default; runs hourly) |

---

## How to Use This in Implementation

An implementer uses this step before populating any inventory records. First, decide which asset types are in scope for the initial deployment—the guidance suggests starting with AI systems and models and deferring datasets and prompts to a later phase. Next, enumerate the specific assets to load. Then select the primary intake method (manual workspace intake, Record Producers, AI Discovery connectors, or automated Now Assist sync). Finally, confirm whether Now Assist auto-discovery should remain active; it is enabled by default and runs hourly. Keeping the initial scope narrow reduces complexity and leaves room to expand during the Full Functionality phase.
