## What Is This?

Initial Inventory Population is a prelaunch step in which you load the AI asset inventory using at least one intake method and confirm that the resulting records are created correctly in the CMDB. It validates that AI assets flow into AI Control Tower (via Now Assist sync, manual workspace intake, or Record Producers) and appear with the expected metadata and governance records. This step follows workspace verification and confirms the inventory is operational before further lifecycle work.

---

## Overview

Populate the AI asset inventory using at least one intake method and verify that records are created correctly in the CMDB.

---

## Initial Inventory Checklist

| ✓ | Initial Inventory |
| --- | --- |
| ☐ | If using Now Assist sync: confirm the Sync Now Assist AI Assets scheduled job is active and has run successfully |
| ☐ | If using manual intake via workspace: test adding an AI System, AI Model, Dataset, and Prompt from the AI Control Tower Workspace list pages |
| ☐ | If using Record Producers: verify that Request an AI Use Case, Request an AI Model, and Request a Dataset are available in the Service Catalog |
| ☐ | Verify that submitted assets appear in the AI Asset Inventory tab with correct metadata |
| ☐ | Verify that an AI Asset Governance Details record is created for each asset |

---

## How to Use This in Implementation

After completing workspace verification, choose at least one intake method to seed the AI asset inventory:

- **Now Assist sync** — ensure the Sync Now Assist AI Assets scheduled job is active and has run successfully.
- **Manual workspace intake** — test adding an AI System, AI Model, Dataset, and Prompt from the AI Control Tower Workspace list pages.
- **Record Producers** — confirm the Request an AI Use Case, Request an AI Model, and Request a Dataset catalog items are available in the Service Catalog.

Then validate the outcome: submitted assets must appear in the AI Asset Inventory tab with correct metadata, and an AI Asset Governance Details record must be created for each asset. This confirms records are correctly created in the CMDB before proceeding to lifecycle validation.
