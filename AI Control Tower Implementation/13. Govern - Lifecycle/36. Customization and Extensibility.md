## What Is This?

This subtopic describes how customers can extend and tailor the AI Asset Lifecycle Playbook to support their own AI asset types and lifecycle tasks. It centers on using Flow Designer to clone and adapt the out-of-the-box (OOB) Flows and Subflows that drive lifecycle execution. It fits into the Govern - Lifecycle area as the guidance for safely customizing playbook behavior while preserving upgrade compatibility.

---

## Customization and Extensibility

Customers can extend and tailor the playbook through Flow Designer:

- **Recommended approach:**
  - Clone existing Flows and Subflows
  - Modify triggers and conditions to support additional AI asset types
- **Task customization options:**
  - Define new lifecycle tasks using Flow Designer conditions
  - Adjust task sequencing where needed
- **Best practice guidance:**
  - Avoid deactivating out-of-the-box (OOB) tasks
  - Maintain the standard lifecycle stage order (Assess → Build & Test → Deploy)
  - Use cloning rather than direct modification to preserve upgrade compatibility

---

## How to Use This in Implementation

When extending the lifecycle playbook, implementers should work in Flow Designer and clone the existing OOB Flows and Subflows rather than editing them directly — this preserves upgrade compatibility. Modify the triggers and conditions on the cloned artifacts to support additional AI asset types, and define new lifecycle tasks using Flow Designer conditions, adjusting sequencing as needed. Do not deactivate OOB tasks, and always keep the standard lifecycle stage order of Assess → Build & Test → Deploy intact.
