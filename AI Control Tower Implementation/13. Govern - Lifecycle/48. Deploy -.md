## What Is This?

The Deploy subtopic describes the flow that runs when an AI asset reaches the Deploy phase of the AI governance lifecycle. It defines the "Deploy AI Asset Task Flow," which triggers automatically on the lifecycle phase change and creates a governance task to deploy the AI asset into the customer environment. It fits at the deployment stage of the AICT lifecycle governance process.

---

## Flow: Deploy AI Asset Task Flow

| Step | Action | Details / Task Type |
|------|--------|---------------------|
| 1 | Flow initiation | Triggers when AI Asset phase changes to *Deploy* |
| 1.1 | Create *Deploy Asset* task | AI asset approval task of type *AI Governance Task* |

**Calls:** Deploy AI Asset Task – to deploy AI asset in the customer environment

---

## How to Use This in Implementation

When an AI asset's lifecycle phase transitions to *Deploy*, the Deploy AI Asset Task Flow is automatically initiated. It creates a *Deploy Asset* task of type *AI Governance Task*, which calls the Deploy AI Asset Task to deploy the AI asset into the customer environment. Implementers should ensure the lifecycle phase is correctly set to Deploy to trigger this flow and that the associated governance task is completed to finalize deployment.
