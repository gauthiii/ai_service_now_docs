---

## What Is This?

The Lifecycle Task Model describes how lifecycle tasks are provisioned for AI assets within the AI Asset Lifecycle Playbook. It defines which asset types come preconfigured with lifecycle tasks out of the box and how tasks are added for other asset types. It fits into the Govern - Lifecycle implementation as part of how the playbook drives lifecycle execution across stages.

---

## Lifecycle Task Model

- Out-of-the-box (OOB), AI System assets are preconfigured with lifecycle tasks
- For other AI asset types (e.g., models, datasets, prompts):
  - Tasks may be manually created by the AI Steward
  - Or programmatically enabled through configuration

---

## How to Use This in Implementation

For AI System assets, implementers can rely on the preconfigured OOB lifecycle tasks with no additional setup. For other asset types such as models, datasets, and prompts, an AI Steward must either manually create the lifecycle tasks or enable them programmatically through configuration. This determines how lifecycle tasks become available within the AI Asset Lifecycle Playbook for each asset type.
