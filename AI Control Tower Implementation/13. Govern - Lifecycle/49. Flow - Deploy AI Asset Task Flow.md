## What Is This?

The Deploy AI Asset Task Flow is part of the **Deploy** stage of the AI System asset lifecycle in AI Control Tower. It is a flow that triggers when an AI Asset moves into the Deploy phase and creates the governance task needed to deploy the AI asset into the customer environment. It fits into the broader Govern - Lifecycle process that orchestrates approval and deployment tasks across the AI asset state transitions.

---

## Deploy: Flow Steps

The flow triggers when the AI Asset phase changes to *Deploy*.

| Step | Action | Details / Task Type |
|------|--------|---------------------|
| 1 | Flow initiation | Triggers when AI Asset phase changes to *Deploy* |
| 1.1 | Create *Deploy Asset* task | AI asset approval task of type *AI Governance Task* |

**Calls:** Deploy AI Asset Task – to deploy AI asset in the customer environment

---

## How to Use This in Implementation

To apply this flow, ensure the AI System asset lifecycle is configured so that an asset can transition into the *Deploy* phase. When that phase change occurs, the flow automatically initiates and creates a *Deploy Asset* task (an AI asset approval task of type *AI Governance Task*). This task invokes the **Deploy AI Asset Task**, which deploys the AI asset into the customer environment. The prerequisite is that prior lifecycle stages (such as the Ready for Deployment review and conformity assessment tasks) have been completed before the asset reaches the Deploy phase.
