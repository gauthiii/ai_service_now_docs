# Flow: Assign analyst to AI System

---

## What Is This?

This subtopic documents the "Assign analyst to AI System" flow within the AI Control Tower governance lifecycle. The flow automatically assigns a risk analyst to an AI system once the AI system's impact assessment has been completed. It fits into the Govern lifecycle as a follow-on automation after the impact assessment task is closed, ensuring the AI system record reflects the analyst responsible for it.

---

## Flow Steps

The flow is triggered when an AI system impact assessment is closed, and it updates the AI system record to assign the analyst.

| Step | Action / Task | Details / Task Type |
|------|---------------|---------------------|
| 1 | Flow initiation | When AI system impact assessment is closed |
| 1.1 | Update AI system record (i.e assign analyst field to the user who closed the task) | Assign risk analyst on AI system to the user who closed the task |

---

## How to Use This in Implementation

To apply this flow, ensure the AI system impact assessment process is configured so that closing an impact assessment task triggers this flow. The flow keys off the closure event of the AI system impact assessment, then updates the AI system record by setting its analyst field to the user who closed the task—effectively assigning that user as the risk analyst for the AI system. This depends on the upstream impact assessment task being completed and closed; no analyst is assigned until that closure occurs.
