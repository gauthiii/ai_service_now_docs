## What Is This?

This subtopic covers the configuration considerations for the Build & Test phase when implementing controls in the AI lifecycle. It points implementers to the Govern - Lifecycle topic for modifying the Flow and associated Sub-Flows used in this phase, and provides notes on enabling smart assessments and safely modifying system-defined flows.

---

## Configuration

For more details on modifying Flow and associated Sub-Flows belonging to Build & test phase, refer to the Govern - Lifecycle topic.

### Note

- Enable Smart Assessments: Set the system property `sn_grc.enable_smart_assessment_on_control` to true to utilize advanced assessment functionality
- To make change or corrections to system-defined flows/ sub-flows, create a copy version for upgrade safety measures

## How to Use This in Implementation

When implementing controls in the Build & test phase, refer to the Govern - Lifecycle topic for the details of modifying the phase's Flow and Sub-Flows. To enable advanced (smart) assessment functionality on controls, set the system property `sn_grc.enable_smart_assessment_on_control` to true. When changes or corrections to system-defined flows or sub-flows are required, create a copy version rather than editing the original, to preserve upgrade safety.
