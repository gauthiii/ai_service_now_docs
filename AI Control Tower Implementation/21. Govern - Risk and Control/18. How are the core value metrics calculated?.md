# How are the core value metrics calculated?

---

## What Is This?

This subtopic explains how AI Control Tower (AICT) quantifies the value delivered by AI assets. AICT calculates value using three primary metrics: Usage, Time Saved, and Acceptance Rate. These metrics feed the Value dashboards and underpin AICT's value-tracking capabilities for governed AI assets.

---

## Core Value Metrics

AICT calculates value using three primary metrics:

### Usage

- Represents the number of AI invocations within a defined daily window.
- Derived from event records in the `sys_generative_ai_usage_log` table (or equivalent).

### Time Saved

- Estimates the reduction in manual effort per invocation.
- This can be configured as:
  - A fixed value (e.g., 15 minutes), or
  - A Performance Analytics (PA) indicator

### Acceptance Rate

- Measures the proportion of AI-generated outputs accepted by users.
- Calculated as:
  - *(Accepted AI outputs ÷ Total AI outputs) × 100*
- Can be defined as a constant or a PA indicator

---

## How to Use This in Implementation

When configuring value tracking in AICT, implementers should ensure usage event records are flowing into the `sys_generative_ai_usage_log` table (or equivalent) so the Usage metric can be derived. For Time Saved and Acceptance Rate, decide whether to use a fixed constant value or bind the metric to a Performance Analytics (PA) indicator. The Acceptance Rate formula *(Accepted AI outputs ÷ Total AI outputs) × 100* should be confirmed against the available output-acceptance data. These three metrics together drive the Value dashboards, so accurate data dependencies are required for value calculations to display correctly.
