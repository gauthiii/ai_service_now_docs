# Classic Attestation vs Smart Attestation

## What Is This?

ServiceNow supports two attestation approaches for control attestation, and choosing between them is an important configuration decision during the Build and Test phase task story. This subtopic compares the Classic Attestation approach (driven by the Platform Assessment Engine) against the Smart Attestation approach (driven by the Smart Assessment Engine), helping implementers decide which engine to configure for their AI governance attestations.

---

## Comparison: Classic vs Smart Attestation

ServiceNow supports two attestation approaches. This is an important configuration decision for the Build and Test phase task story.

| | Classic Attestation (Platform Assessment Engine) | Smart Attestation (Smart Assessment Engine) |
|---|---|---|
| Record prefix | AINST | ASMT |
| Link visibility | Records appear only after assessment completion | Attestation link visible before and after completion |
| Advanced features | Standard assessment engine Group | Auto-save, reassignment triggers, question-level instructions, domain separation Combine |
| Recommendation | Legacy approach | Recommended for all new implementations |

---

## How to Use This in Implementation

During the Build and Test phase, implementers must choose which attestation engine drives control attestation tasks:

- Use **Smart Attestation (Smart Assessment Engine)**, which produces records with the **ASMT** prefix, for all new implementations. It is the recommended approach and offers advanced capabilities including auto-save, reassignment triggers, question-level instructions, and domain separation. Attestation links are visible both before and after completion.
- **Classic Attestation (Platform Assessment Engine)**, which produces records with the **AINST** prefix, is the legacy approach. Its attestation records appear only after assessment completion, and it uses the standard assessment engine grouping.

Make this configuration decision early in the Build and Test task story since it determines record prefixes, link visibility behavior, and which advanced features are available.
