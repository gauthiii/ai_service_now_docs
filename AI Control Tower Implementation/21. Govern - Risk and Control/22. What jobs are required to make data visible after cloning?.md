# What jobs are required to make data visible after cloning?

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Govern - Risk and Control → What jobs are required to make data visible after cloning?
> Source: AI-Control-Tower.pdf, p.254

---

## What Is This?

After cloning an instance, AICT Value dashboards may not populate until certain jobs are run. This subtopic identifies the jobs and scheduled scripts required to make Value dashboard data visible following a clone. It fits into the operational maintenance of AICT Value reporting.

---

## Jobs Required to Make Data Visible After Cloning

To ensure Value dashboards populate correctly:

- Run all OOB Performance Analytics indicators at least once
- If data is still not visible:
  - Manually trigger: `aivalue.valuedashboard.HistoricalJob`
- Verify that the following scheduled script is active:
  - AIValue Generate Historical Data

---

## How to Use This in Implementation

After a clone, an implementer should first run all out-of-the-box Performance Analytics indicators at least once so the Value dashboards have source data. If dashboards remain empty, manually trigger the `aivalue.valuedashboard.HistoricalJob` and confirm that the `AIValue Generate Historical Data` scheduled script is active. These steps reestablish the historical data generation that drives Value dashboard population.
