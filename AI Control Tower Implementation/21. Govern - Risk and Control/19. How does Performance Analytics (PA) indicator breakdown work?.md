# How does Performance Analytics (PA) indicator breakdown work?

> Section: Govern - Risk and Control → How does Performance Analytics (PA) indicator breakdown work?

---

## What Is This?

This subtopic explains how Performance Analytics (PA) indicators behave when they are reused across multiple AI assets in AI Control Tower value calculations. Because a single PA indicator can be shared, AICT relies on an indicator *breakdown by asset* to retrieve asset-specific scores. This ensures that value metrics (such as Time Saved or Acceptance Rate that are defined as PA indicators) are calculated accurately per asset rather than from aggregated data.

---

## PA Indicator Breakdown Behavior

PA indicators may be shared across multiple AI assets. To ensure accurate value calculations:

- Indicators must include a breakdown by asset
- This enables the value calculation engine to retrieve asset-specific scores

**Important:** If an indicator is used without a breakdown:

- The system retrieves aggregated data
- In this case, the indicator must be scoped to a single asset to avoid inaccurate calculations

---

## How to Use This in Implementation

When defining value metrics such as Time Saved or Acceptance Rate as PA indicators (rather than fixed values), configure the PA indicator with a breakdown by asset so the value calculation engine can pull the correct asset-specific score. If you cannot add a breakdown, restrict that indicator to a single AI asset only—using a breakdown-less indicator across multiple assets returns aggregated data and produces inaccurate per-asset value calculations.
