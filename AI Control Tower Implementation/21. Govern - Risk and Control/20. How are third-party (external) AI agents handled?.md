## What Is This?

This subtopic explains how AI Control Tower handles third-party (external) AI agents — agents that are not native to the platform. It covers how such agents are registered and tracked, what metrics are captured for them, and how their value (time saved) is calculated by default. It fits into value tracking and inventory management for externally sourced AI agents.

---

## Third-Party Agent Registration and Tracking

Third-party agents are:

- Registered within the AI Inventory
- Tracked using the `sn_ai_disc_ai_usage` table

## Current Capability

- Only usage (invocation count) is captured for external agents

## Default Value Calculation

The default value (time saved) is calculated using the following formula:

```
EnterpriseAgents.InvocationCount.DailyLink × 15 mins × 50%
```

Where:

- 15 minutes = default time saved per invocation
- 50% = default acceptance rate

These are out-of-box assumptions and can be overridden using custom Value Templates.

---

## How to Use This in Implementation

To handle external AI agents, register them in the AI Inventory so their usage is tracked through the `sn_ai_disc_ai_usage` table. Be aware that only invocation count is captured for these agents, so value is derived from a default formula assuming 15 minutes saved per invocation and a 50% acceptance rate. If these out-of-box assumptions do not match your organization's reality, override them by configuring custom Value Templates.
