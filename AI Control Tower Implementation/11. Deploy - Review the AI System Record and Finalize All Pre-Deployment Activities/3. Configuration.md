## What Is This?

This subtopic covers the configuration of the Flow and associated Sub-Flows that belong to the Deploy phase of the AI Asset Onboarding lifecycle. It directs implementers to the Govern – Lifecycle topic for detailed guidance on modifying these flows, and stresses safe handling of system-defined flows during the deployment of an AI system record.

---

## Configuration of Deploy Phase Flows

For more details on modifying Flow and associated Sub-Flows belonging to the Deploy phase, refer to the Govern – Lifecycle topic.

### Note

To make change or corrections to system-defined flows / sub-flows, create a copy version for upgrade safety measures.

## How to Use This in Implementation

When configuring the Deploy phase, do not edit the system-defined Flow or Sub-Flows directly. Instead, create a copy version before making any changes or corrections, so that platform upgrades remain safe. For the detailed procedure on modifying the Deploy-phase Flow and its Sub-Flows, consult the Govern – Lifecycle topic. The Deploy phase appears as a stage in the AI Asset Onboarding lifecycle, with its own task (e.g., "Deploy Asset") that must be completed to advance the record.
