## What Is This?

A state model defines the workflow of a record by specifying the states a case or inquiry moves through, along with the conditions that govern each transition. Within AI Case Management, each case type or inquiry type follows the workflow states and transition conditions assigned to it. State models are how implementers structure and control the lifecycle of AI cases and inquiries.

---

## State Model Components

A state model defines the workflow of a record by specifying the states a case or inquiry moves through, along with the conditions that govern each transition. Each case type or inquiry type follows the workflow states and transition conditions assigned to it.

There are three components to configure:

- **State Model** — The state model establishes the overall framework of states and transitions for a given case or inquiry type. (Configuration steps available via "click here" link.)
- **Workflow State** — Workflow states define the lifecycle stages of a case or inquiry within AI Case Management. To add new states, administrators create them directly in the State field of the AI Case table. (Configuration steps available via "click here" link.)
- **State Transitions** — State transitions define the conditions that control how a case moves between workflow states. Using the condition builder in the State Transitions table, administrators can specify the required conditions for entering or exiting each state. (Configuration steps available via "click here" link.)

## How to Use This in Implementation

To implement a workflow for AI Case Management, configure the three components in sequence: define the overall **State Model** for the case or inquiry type, create the required **Workflow States** directly in the State field of the AI Case table, and define the **State Transitions** in the State Transitions table using the condition builder to specify entry/exit conditions for each state. Use the per-industry examples (Financial, Public Sector, Technology, HealthCare) as reference patterns — note that different case types may require additional intermediate states such as Investigate and Post Case rather than a simple New → In Progress → Complete flow.
