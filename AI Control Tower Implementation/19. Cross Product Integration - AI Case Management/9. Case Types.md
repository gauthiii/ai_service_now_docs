## What Is This?

Case Types is a configuration within AI Case Management that lets you define AI case types and sub-types to create a structured taxonomy for AI-related cases. It organizes cases consistently, supports automated routing and prioritization, and underpins governance and regulatory compliance. It fits into the Configurations area of the Cross Product Integration / AI Case Management implementation.

---

## Defining AI Case Types and Sub-Types

Defining AI case types and sub-types creates a structured taxonomy that organizes cases consistently, improves reporting accuracy, and enables trend analysis across AI-related issues. Proper categorization supports automated routing and prioritization, leading to faster resolution. It also ensures governance and regulatory compliance by establishing consistent handling for similar cases — and scales as AI use cases and risk management needs evolve.

Common examples include:

- Accuracy Degradation
- Unexpected Behavior
- Model Drift
- Data Quality

For configuration steps, see the referenced "click here" link in the source document.

## How to Use This in Implementation

When implementing AI Case Management, define a consistent set of case types and sub-types up front so that cases are categorized uniformly. Use the common examples (Accuracy Degradation, Unexpected Behavior, Model Drift, Data Quality) as a starting taxonomy, and expand as AI use cases and risk management needs evolve. A well-defined taxonomy is a prerequisite for automated routing, prioritization, accurate reporting, trend analysis, and governance/regulatory compliance. Follow the configuration steps linked in the documentation to set up these types.
