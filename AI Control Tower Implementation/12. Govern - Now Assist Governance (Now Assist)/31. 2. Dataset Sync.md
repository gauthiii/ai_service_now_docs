## What Is This?

Dataset Sync is one of the Now Assist governance sync operations in AI Control Tower (AICT). It iterates OneExtend Test Datasets with run type `eval_run` and maps them to AI Dataset Digital Assets. It ensures that both the dataset and its parent Base Dataset are represented in AICT for governance visibility.

---

## How Dataset Sync Works

Iterates OneExtend Test Datasets with run type `eval_run` and maps them to AI Dataset Digital Assets. If a Base Dataset (parent) exists, it is also synced; if not, it is created.

| Record Updated | Fields Synced |
|----------------|---------------|
| AI Dataset Product Model | Name, Description, Model Category |
| Base Dataset Product Model (if applicable) | Same fields as above; created if Base Dataset does not yet exist in AICT |

---

## How to Use This in Implementation

When configuring Now Assist governance in AICT, Dataset Sync automatically brings evaluation datasets into the governance model. Implementers should ensure that OneExtend Test Datasets are tagged with run type `eval_run` so they are picked up by the sync. The sync handles parent Base Datasets automatically—creating them in AICT if they do not already exist—so no manual creation of Base Dataset records is required. Synced fields are limited to Name, Description, and Model Category on the AI Dataset Product Model (and the Base Dataset Product Model when applicable).
