## What Is This?

Sync AI Agents is a synchronization process within Now Assist governance that maps AI agent records into the AI System data model. It identifies the associated AI System, syncs the agent with the AI System Component Product Model, and maps agent metadata onto the AI system. This ensures AI agents are represented in the governance and CMDB structure for tracking and oversight.

---

## Source

Source: `$sn_aia_agent$`

---

## Process

The synchronization process performs the following steps:

- Identify associated AI System
- Sync with AI System Component Product Model `$cmdb_ai_system_component_product_model$`
- Map agent metadata to the AI system

---

## Mapped Attributes

The following attributes are mapped during the sync:

- Name
- Description
- Model Category
- Documentation
- Manufacturer

---

## Note

> Prompt product models are also synchronized for each associated AI Prompt Digital Asset.

---

## How to Use This in Implementation

When implementing Now Assist governance, configure the Sync AI Agents process to draw from the `sn_aia_agent` source. The process requires an associated AI System to exist so that the agent can be linked and its metadata (Name, Description, Model Category, Documentation, Manufacturer) mapped onto the AI system via the AI System Component Product Model (`cmdb_ai_system_component_product_model`). Be aware that prompt product models are synchronized as well for each associated AI Prompt Digital Asset, so related prompt assets should also be in place to ensure complete synchronization.
