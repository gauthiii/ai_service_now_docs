## What Is This?

This subtopic defines the data source used by the "Sync Dataset Assets" step in Now Assist governance. It identifies the specific OneExtend test dataset that drives the synchronization of AI dataset assets into the governance model. It is the starting input for the dataset sync process that subsequently creates or updates AI Dataset Digital Assets and Product Models.

---

## Source Definition

The source for syncing dataset assets is:

- *OneExtend Test Dataset* `$sys_one_extend_test_dataset$` (run type = *eval_run*)

---

## How to Use This in Implementation

When performing the "Sync Dataset Assets" step, use the OneExtend Test Dataset (`sys_one_extend_test_dataset`) records filtered to a run type of `eval_run` as the source input. This dataset feeds the downstream Process step, which iterates through dataset records to identify or create AI Dataset Digital Assets (`alm_ai_dataset_digital_asset`) and their associated AI Dataset Product Models (`cmdb_ai_dataset_product_model`).
