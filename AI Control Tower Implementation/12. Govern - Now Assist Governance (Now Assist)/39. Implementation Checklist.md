# Implementation Checklist

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Govern - Now Assist Governance (Now Assist) → Implementation Checklist

---

## What Is This?

This is a verification checklist used to confirm that Now Assist AI Governance auto-discovery is correctly configured and active. It lists the discrete tasks an implementation team must complete and assigns an owner to each, ensuring deployed Now Assist AI assets are properly synced into the AICT inventory and ready for governance use.

---

## Implementation Checklist

Use this checklist to confirm your Now Assist AI Governance auto-discovery is correctly configured and active.

| Task | Owner |
|------|-------|
| Confirm Sync Now Assist AI Assets scheduled job is Active | Platform Admin |
| Verify Now Assist skills are deployed and in active use | Implementation Lead |
| Run the scheduled job manually and confirm assets appear in AICT inventory | Platform Admin |
| Confirm AI Models, Datasets, Prompts, and Agentic Assets are all represented | AI Governance Owner |
| Review any customization requirements; if needed, extend NowAssistAIAssetsUtil | Platform Admin |
| Brief Risk & Compliance Lead on inventory and how to use it for AI risk assessments | AI Governance Owner |
| Document any deviations from default configuration in your governance runbook | Implementation Lead |

---

## How to Use This in Implementation

Work through each task in order, assigning the named owner role for accountability. Begin by confirming the "Sync Now Assist AI Assets" scheduled job is Active (Platform Admin) and that Now Assist skills are actually deployed and in use (Implementation Lead). Run the scheduled job manually to validate that assets surface in the AICT inventory, then confirm that AI Models, Datasets, Prompts, and Agentic Assets are all represented (AI Governance Owner). If the default discovery does not capture everything needed, extend the `NowAssistAIAssetsUtil` script include. Finally, brief the Risk & Compliance Lead on using the inventory for AI risk assessments and record any deviations from the default configuration in your governance runbook.
