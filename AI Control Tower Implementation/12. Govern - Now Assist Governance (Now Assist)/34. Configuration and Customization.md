## What Is This?

This subtopic covers the configuration and customization options for the **Sync Now Assist AI Assets** scheduled job, which maintains AICT's inventory of deployed Now Assist assets. It explains the default behavior, how to disable auto-discovery, how to customize sync logic, and the governance implications of these choices. It fits into the Now Assist governance workflow as the operational layer that keeps the AI inventory current.

---

## Default Configuration

The **Sync Now Assist AI Assets** scheduled job is enabled by default. No additional setup is required for standard Now Assist deployments. The job runs automatically and maintains continuous inventory sync.

---

## Disabling Auto-Discovery

You may deactivate the scheduled job if your organization does not use Now Assist or has a separate inventory management process. To disable:

- Navigate to **System Definition > Scheduled Jobs**
- Locate **Sync Now Assist AI Assets**
- Set the job status to **Inactive**

**Recommendation:** Keep the scheduled job active. Disabling it means your AI inventory will no longer reflect deployed Now Assist assets, creating governance gaps and potential compliance risk.

---

## Customizing Sync Behavior

If your implementation requires non-standard sync logic — for example, filtering which assets are synced or adding custom field mappings — customize using the standard ServiceNow Script Include inheritance pattern:

| Step | Action |
|------|--------|
| 1 | Create a new Script Include that extends `NowAssistAIAssetsUtil` |
| 2 | Override only the methods you need to change (e.g., `syncAIModels`, `syncAIPrompts`) |
| 3 | Do not modify the base `NowAssistAIAssetsUtil` directly — preserve base functionality |
| 4 | Test in a sub-production environment before deploying changes |

## How to Use This in Implementation

For standard Now Assist deployments, no action is required — the **Sync Now Assist AI Assets** scheduled job is active by default and keeps the AI inventory current. Only disable it (via **System Definition > Scheduled Jobs**, setting **Sync Now Assist AI Assets** to **Inactive**) if Now Assist is not used or a separate inventory process exists, and be aware this creates governance/compliance gaps. To customize sync behavior, follow the Script Include inheritance pattern: extend `NowAssistAIAssetsUtil`, override only the needed methods (such as `syncAIModels` or `syncAIPrompts`), never edit the base class directly, and validate in a sub-production environment before deploying.
