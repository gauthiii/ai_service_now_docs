## What Is This?

This subtopic explains how to extend or alter the Now Assist AI asset sync logic when an implementation requires non-standard behavior. It is part of the Configuration and Customization area for Now Assist governance, where the default scheduled job keeps the AI inventory in sync automatically. Customization is achieved through the standard ServiceNow Script Include inheritance pattern rather than by editing base code.

---

## Customizing Sync Behavior

If your implementation requires non-standard sync logic — for example, filtering which assets are synced or adding custom field mappings — customize using the standard ServiceNow Script Include inheritance pattern:

| Step | Action |
|------|--------|
| 1 | Create a new Script Include that extends NowAssistAIAssetsUtil |
| 2 | Override only the methods you need to change (e.g., syncAIModels, syncAIPrompts) |
| 3 | Do not modify the base NowAssistAIAssetsUtil directly — preserve base functionality |
| 4 | Test in a sub-production environment before deploying changes |

---

## How to Use This in Implementation

To customize sync behavior, create a new Script Include that extends the base `NowAssistAIAssetsUtil` and override only the specific methods you need to change, such as `syncAIModels` or `syncAIPrompts`. Leave the base `NowAssistAIAssetsUtil` unmodified so that base functionality is preserved and upgrades remain safe. Always validate changes in a sub-production environment before deploying to production. This work fits alongside the default configuration (the Sync Now Assist AI Assets scheduled job, enabled by default) and the option to disable auto-discovery, but customization is the recommended path when filtering synced assets or adding custom field mappings is required.
