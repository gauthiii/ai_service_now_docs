## What Is This?

"Product Model Attributes" defines the set of fields captured on an AI Product Model record when assets are synchronized into the AI Control Tower data model. These attributes describe the model so that synced AI assets (such as AI Model Digital Assets) map to a meaningful, identifiable product model entry during the Now Assist governance synchronization process.

---

## Product Model Attributes

The Product Model Attributes captured for a model asset are:

- Name
- Description
- Model Category

## How to Use This in Implementation

When configuring or validating the synchronization of AI model assets into the AI Control Tower, ensure each synced AI Product Model record is populated with the three attributes: **Name**, **Description**, and **Model Category**. These fields identify and classify the model so that associated digital assets and AI systems can be correctly linked during governance synchronization. Note that other sync sections (for example, prompt assets) define their own product model attribute sets, so confirm you are mapping the correct attribute list for the asset type being synced.
