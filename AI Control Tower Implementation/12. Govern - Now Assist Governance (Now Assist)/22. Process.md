## What Is This?

This subtopic describes the synchronization "Process" steps used by the Now Assist governance sync routines for AI Agents and AI Use Cases. Each process identifies an associated AI System, syncs it with the relevant CMDB AI System Component Product Model, and maps metadata attributes to the AI system. It fits into implementation as the mechanism that populates and aligns AI agent and use case records with the AI system inventory.

---

## Process — Sync AI Agents

Source: `$sn_aia_agent$`

Process steps:

- Identify associated AI System
- Sync with AI System Component Product Model `$cmdb_ai_system_component_product_model$`
- Map agent metadata to the AI system

Mapped Attributes:

- Name
- Description
- Model Category
- Documentation
- Manufacturer

> **Note:** Prompt product models are also synchronized for each associated AI Prompt Digital Asset.

## Process — Sync AI Use Cases

Process steps:

- Sync with AI System Component Product Model
- Map use case metadata to the AI system

Mapped Attributes:

- Name
- Description

- Documentation
- Manufacturer

> **Note:** Prompt product models are also synchronized for associated AI Prompt Digital Assets.

---

## How to Use This in Implementation

When configuring Now Assist governance synchronization, ensure each AI Agent and AI Use Case has an associated AI System so the process can identify it. The sync then aligns the record with the corresponding AI System Component Product Model and maps the listed metadata attributes (Name, Description, Model Category, Documentation, Manufacturer) onto the AI system. Note that prompt product models are synchronized automatically for each associated AI Prompt Digital Asset, so no separate prompt-model sync step is required.
