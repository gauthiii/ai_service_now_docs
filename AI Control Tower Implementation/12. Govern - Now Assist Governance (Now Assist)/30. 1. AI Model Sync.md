## What Is This?

AI Model Sync is one of the asset sync methods in AI Control Tower (AICT). It iterates all Generative AI Model Configurations and maps them to AI Model Digital Assets in AICT. Both the digital asset record and its associated AI Model Product Model are updated during the sync, keeping the governance inventory aligned with deployed AI models.

---

## Sync Behavior

Each sync method follows the same create-or-update logic: if an asset already exists in AICT, it is updated; if it does not exist, it is created.

AI Model Sync specifically iterates all Generative AI Model Configurations and maps them to AI Model Digital Assets in AICT. Both the digital asset record and its associated AI Model Product Model are updated.

## Fields Synced

| Record Updated | Fields Synced |
| --- | --- |
| AI Model Digital Asset | Model Card, Required Infrastructure, Evaluation Metrics Report |
| AI Model Product Model | Name, Model Category, Supported Languages, Context Window, Documentation, Manufacturer, Parameters, Model Size (MB), Source, Deployment Guideline, Training Procedure |

---

## How to Use This in Implementation

This sync runs as part of the scheduled asset discovery job. Implementers should keep the scheduled job enabled so that Generative AI Model Configurations are continuously reflected as AI Model Digital Assets and AI Model Product Models in AICT. Because the sync is idempotent (create-or-update), it is safe to run repeatedly without duplicating records. If automatic discovery is not desired, the job can be deactivated, but manual asset management would then be required to keep the model inventory complete.
