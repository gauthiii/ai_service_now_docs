## What Is This?

Base Dataset Handling is a step within the "Sync Dataset Assets" process of Now Assist governance. It defines the create/update logic applied when a dataset has a parent (base dataset). It ensures that base datasets and their associated product models are correctly identified, created, or updated during dataset asset synchronization.

---

## Base Dataset Handling

This step is performed while iterating through dataset records during the Sync Dataset Assets process.

- If a parent (base dataset) exists:
  - Apply the same create/update logic
- If not:
  - Create the base dataset and associated product model

### Context: Sync Dataset Assets (preceding steps)

Base Dataset Handling follows the broader dataset sync logic on the same page:

- **Source:** *OneExtend Test Dataset* `$sys_one_extend_test_dataset$` (run type = `eval_run`)
- **Process:**
  - Iterate through dataset records
  - For each dataset:
    - Identify or create an AI Dataset Digital Asset `$alm_ai_dataset_digital_asset$`
    - Update or create the associated AI Dataset Product Model `$cmdb_ai_dataset_product_model$`

---

## How to Use This in Implementation

When synchronizing dataset assets, the implementer iterates through dataset records and, for each one, manages both the digital asset (`alm_ai_dataset_digital_asset`) and the product model (`cmdb_ai_dataset_product_model`). For base dataset handling specifically: check whether a parent (base dataset) exists. If it does, apply the same create/update logic used for the dataset; if it does not, create the base dataset and its associated product model. This guarantees that the base dataset hierarchy is fully represented before downstream governance processing.
