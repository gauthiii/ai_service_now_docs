## What Is This?

This subtopic describes the synchronization processes that map Now Assist governance entities into the AI system records of the CMDB. Specifically, it covers the "Process" steps and mapped attributes for syncing AI Agents and AI Use Cases. These processes ensure that agent and use case metadata are reflected against the corresponding AI System Component Product Model so that governance and inventory data stay aligned.

---

## Sync AI Agents — Process

Source: `$sn_aia_agent$`

### Process

- Identify associated AI System
- Sync with AI System Component Product Model `$cmdb_ai_system_component_product_model$`
- Map agent metadata to the AI system

### Mapped Attributes

- Name
- Description
- Model Category
- Documentation
- Manufacturer

> **Note:** Prompt product models are also synchronized for each associated AI Prompt Digital Asset.

## How to Use This in Implementation

When implementing Now Assist governance synchronization, each AI Agent (sourced from `$sn_aia_agent$`) and each AI Use Case is mapped to its associated AI System by syncing against the AI System Component Product Model (`$cmdb_ai_system_component_product_model$`). For both entity types, the synchronization carries the same set of mapped attributes — Name, Description, Model Category, Documentation, and Manufacturer. Implementers should also be aware that prompt product models are synchronized alongside, for each associated AI Prompt Digital Asset.
