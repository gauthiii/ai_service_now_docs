## What Is This?

This is the **Process** step within the **Sync Dataset Assets** routine of Now Assist governance. It defines the logic used to iterate through dataset records and ensure that, for each dataset, the corresponding AI Dataset Digital Asset and AI Dataset Product Model records exist (creating or updating them as needed). It fits into the governance flow that keeps dataset digital assets synchronized in the configuration management data layer.

---

## Process (Sync Dataset Assets)

- Iterate through dataset records
- For each dataset:
  - Identify or create an AI Dataset Digital Asset `$alm_ai_dataset_digital_asset$`
  - Update or create the associated AI Dataset Product Model `$cmdb_ai_dataset_product_model$`

---

## How to Use This in Implementation

When implementing the Sync Dataset Assets routine, loop over every dataset record. For each one, ensure an AI Dataset Digital Asset record (`alm_ai_dataset_digital_asset`) is identified or created, and ensure the associated AI Dataset Product Model record (`cmdb_ai_dataset_product_model`) is created or updated. This keeps each dataset represented as a synchronized digital asset and product model pair. Base (parent) datasets are handled separately by the Base Dataset Handling logic.
