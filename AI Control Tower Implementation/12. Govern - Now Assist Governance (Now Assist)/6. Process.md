## What Is This?

This subtopic describes the **Process** step within "1. Sync Model Assets," part of the Automated AI Asset Synchronization performed by the AICT scheduled job "Sync Now Assist AI Assets." It defines what the synchronization job does for each Generative AI model configuration when discovering and registering AI model assets used by Now Assist skills. It fits into the broader Now Assist governance flow where AICT continuously syncs AI Models, Datasets, Prompts, and Agentic AI assets from the live Now Assist configuration.

---

## Process — Sync Model Assets

The synchronization process for model assets performs the following:

- Iterate through all model configurations
- For each configuration:
  - Identify or create an AI Model Digital Asset `$alm_ai_model_digital_asset$`
  - Prepare model metadata using *NAAssetsStaticContent*
  - Update or create the associated AI Model Product Model `$cmdb_ai_model_product_model$`

### Related context (Source for Sync Model Assets)

The source for this step is:

- *Generative AI Model Configurations* `$sys_generative_ai_model_config$`

---

## How to Use This in Implementation

When implementing Now Assist governance, ensure the scheduled job "Sync Now Assist AI Assets" is active so model assets are synchronized automatically. For model asset synchronization, AICT reads from the Generative AI Model Configurations table (`sys_generative_ai_model_config`), iterates every configuration, and for each one creates or updates an AI Model Digital Asset (`alm_ai_model_digital_asset`), prepares metadata via *NAAssetsStaticContent*, and creates or updates the associated AI Model Product Model (`cmdb_ai_model_product_model`). This removes the need for manual model asset entry and keeps the AI inventory aligned with deployed Now Assist functionality.
