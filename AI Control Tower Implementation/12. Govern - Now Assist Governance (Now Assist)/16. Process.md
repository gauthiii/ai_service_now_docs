## What Is This?

This subtopic defines the **Process** step within **Sync Prompt Assets** (step 3 of the Now Assist governance synchronization flow). It describes how prompt configurations are iterated and turned into governed AI Prompt assets and product models. It fits into implementation as the procedural logic that drives prompt asset creation and association with AI systems and models.

---

## Process

The process for synchronizing prompt assets is:

- Iterate through prompt configurations
- For each prompt:
  - Identify or create an AI Prompt Digital Asset `$alm_ai_prompt_digital_asset$`
  - Update or create the associated AI Prompt Product Model `$cmdb_ai_prompt_product_model$`
  - Associate prompts with relevant AI systems and models

---

## How to Use This in Implementation

When implementing prompt asset synchronization, an implementer loops over the source Generative AI Configurations (`$sys_generative_ai_config$`, Prompt Templates). For each prompt, ensure an AI Prompt Digital Asset (`$alm_ai_prompt_digital_asset$`) exists or is created, ensure the corresponding AI Prompt Product Model (`$cmdb_ai_prompt_product_model$`) is created or updated, and establish the associations linking each prompt to its relevant AI systems and models. This process depends on the prompt configuration source being available and on the digital asset and product model tables being in place.
