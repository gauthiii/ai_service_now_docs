## What Is This?

This subtopic explains how AI Gateway relates to the other products in the broader ServiceNow AI platform. AI Gateway does not operate independently — understanding its relationships is essential to governing it effectively and positioning it correctly within an enterprise AI strategy. It clarifies how AI Gateway connects to AI Control Tower, AI Agent Fabric, and the MCP Server Console.

---

## AI Gateway Relationships Across ServiceNow AI Products

AI Gateway does not operate independently. Understanding its relationships within the broader ServiceNow AI platform is essential to governing it effectively and positioning it correctly within an enterprise AI strategy.

| Product | Role | Relationship to AI Gateway |
| --- | --- | --- |
| AI Control Tower | Parent governance platform | AI Gateway is a feature set within AICT. MCP Servers are governed as AI assets using AICT's playbooks, CMDB integration, and AI Steward role. |
| AI Agent Fabric | MCP connectivity layer | Gives agents the capability to use MCP (protocol layer). AI Gateway governs whether specific MCP Servers are permitted and enforces runtime security. Both are needed for enterprise-safe connectivity. |
| MCP Server Console | ServiceNow capability exposure | Defines what ServiceNow exposes to external agents as MCP tools. AI Gateway governs what gets called — controlling and observing those invocations. Together they form the end-to-end MCP story. |

### Warning

An enterprise can have AI Agent Fabric enabled without AI Gateway — agents can connect to MCP Servers directly, but with no centralized governance. AI Gateway is the layer that makes that connectivity enterprise-safe.

---

## How to Use This in Implementation

When positioning AI Gateway within an enterprise AI strategy, treat it as a feature set within AI Control Tower rather than a standalone product — MCP Servers are governed as AI assets through AICT's playbooks, CMDB integration, and the AI Steward role. Pair AI Gateway with AI Agent Fabric (the protocol/connectivity layer) so that agent connectivity is both enabled and governed; enabling AI Agent Fabric alone leaves connections without centralized governance. Combine AI Gateway with the MCP Server Console so that what ServiceNow exposes as MCP tools is matched by control and observability over what gets invoked, forming the end-to-end MCP story.
