---

## What Is This?

Release Compatibility documents how AI Gateway is delivered and which ServiceNow Store app versions, platform releases, plugins, and dependencies are required to run it. AI Gateway is installed automatically with the AI Control Tower Core/Pro Plus SKU, so no separate activation is required. This subtopic falls under the Technical and Functional Considerations for Cross Product Integration with AI Gateway.

---

## Installation and Activation

AI Gateway is installed automatically with the AI Control Tower Core/Pro Plus SKU (`sn_awh_config`). No separate activation is required.

---

## Store App Version and Platform Compatibility

| Store App Version | Platform Compatibility |
|---|---|
| AWH for AI Control Tower 1.0.5 (AI Control Tower Core 5.0.0) | Zurich Patch 4+, Yokohama Patch 11+ |
| AWH for AI Control Tower 2.0.0 — Q1 2026 (AI Control Tower Core 6.1.0) | Australia release family, Zurich Patch 7+ |

---

## Required Plugins and Dependencies

- AI Control Tower Core/Pro Plus SKU — auto-installs `sn_awh_config`
- `sn_ai_governance` version 6.0.1 or higher (Q1 2026)
- `sn_aict_nowassist` version 3.0.0 or higher (Q1 2026)

---

## How to Use This in Implementation

Before enabling AI Gateway, confirm your instance is on a supported platform release (Zurich Patch 4+ / Yokohama Patch 11+ for AICT Core 5.0.0, or the Australia release family / Zurich Patch 7+ for AICT Core 6.1.0). Because AI Gateway ships automatically with the AI Control Tower Core/Pro Plus SKU via the `sn_awh_config` app, there is no separate activation step. Ensure the dependent plugins `sn_ai_governance` (6.0.1+) and `sn_aict_nowassist` (3.0.0+) are installed at the required versions before using AI Gateway functionality.
