## What Is This?

Every agent platform connecting to an MCP Server through AI Gateway requires a registered client. This subtopic compares the two registration approaches — Manual Registration and CIMD — whose differences carry significant operational implications as the MCP Server inventory grows.

---

## Registration Approaches

Every agent platform connecting to an MCP Server through AI Gateway requires a registered client. The registration approach has significant operational implications as the MCP Server inventory grows.

| Method | Manual Registration | CIMD |
|---|---|---|
| Scope | Per server, per platform | Single instance-level configuration; all new MCP Servers inherit automatically |
| Credential Sprawl | Increases with each new server | Significantly reduced — no per-server credential configuration |
| Supported Platforms | All platforms | Copilot Studio, VS Code (additional platforms in roadmap) |
| Recommendation | Use for platforms not yet supporting CIMD | Configure first for supported platforms before registering any MCP Servers |

### Important

Switching from manual to CIMD registration for existing servers requires re-registration.

---

## How to Use This in Implementation

For platforms that support CIMD (Copilot Studio, VS Code), configure CIMD first — before registering any MCP Servers — so that all new MCP Servers inherit the instance-level configuration automatically and credential sprawl is minimized. Use manual registration only for platforms that do not yet support CIMD, accepting that credentials must be configured per server, per platform. Plan registration approach up front: switching existing servers from manual to CIMD registration requires re-registering them.
