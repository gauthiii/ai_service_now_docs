> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Cross Product Integration - AI Gateway → Enforcement: Implementation Sequence

---

## What Is This?

This subtopic explains how to safely turn on enforcement for MCP Servers accessed through AI Gateway. Enforcement blocks unapproved MCP Servers from use in AI Agent Studio, and because the activation order matters, it defines the required sequence for enabling it in production without breaking existing agent functionality.

---

## Enforcement Behavior and Default State

Enforcement — blocking unapproved MCP Servers from use in AI Agent Studio — is off by default in Q1 2026. The activation sequence matters. Enabling enforcement before existing connections are registered will break agent functionality for any server that has not been approved in AI Control Tower.

## Required Sequence Before Activating Enforcement in Production

- Register all existing MCP Server connections through AI Gateway.
- Complete approvals for servers that meet governance standards.
- Activate enforcement to gate new connections going forward.

---

## How to Use This in Implementation

Before enabling enforcement in a production environment, an implementer must work through the sequence in order: first register every existing MCP Server connection through AI Gateway, then complete approvals in AI Control Tower for the servers that meet governance standards, and only then activate enforcement. Following this order ensures that approved servers continue functioning while enforcement gates new connections going forward. Activating enforcement before existing connections are registered and approved will break agent functionality for any unapproved server.
