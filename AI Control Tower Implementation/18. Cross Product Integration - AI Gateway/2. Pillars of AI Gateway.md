## What Is This?

This subtopic introduces the structural model of AI Gateway. AI Gateway is structured around four pillars that reflect how enterprises need to manage MCP (Model Context Protocol) Server connections at scale. It frames how the governance capabilities of AI Gateway operate within the broader AI Control Tower context.

---

## Pillars of AI Gateway

AI Gateway is structured around four pillars that reflect how enterprises need to manage MCP Server connections at scale. These pillars operate concurrently — not as sequential setup steps, but as ongoing governance capabilities that are active throughout the lifecycle of every MCP Server connection.

---

## How to Use This in Implementation

When implementing AI Gateway, treat its four pillars as concurrent, always-on governance capabilities rather than a one-time setup sequence. Each pillar remains active throughout the entire lifecycle of every MCP Server connection, so implementers should ensure all four are accounted for continuously as MCP Server connections are scaled across the enterprise.
