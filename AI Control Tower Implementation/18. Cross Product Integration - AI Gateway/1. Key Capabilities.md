## What Is This?

The Key Capabilities of AI Gateway describe the core governance and control functions it provides as ServiceNow's governance and control plane for cross-platform AI agent connections. AI Gateway is a feature set within AI Control Tower (not a standalone product), installed automatically with the AI Control Tower Core/Pro Plus SKU. It provides a centralized registry, policy enforcement, and runtime observability for all MCP (Model Context Protocol) Server connections, regardless of which platform built the agent.

---

## AI Gateway Overview

AI Gateway is ServiceNow's governance and control plane for cross-platform AI agent connections. As enterprises build AI agents across ServiceNow AI Agent Studio, Microsoft Copilot Studio, Google Gemini, and other platforms, every agent-to-tool connection becomes a potential governance gap. AI Gateway closes that gap by providing a centralized registry, policy enforcement, and runtime observability for all MCP (Model Context Protocol) Server connections — regardless of which platform built the agent.

AI Gateway is not a standalone product. It is a feature set within AI Control Tower, installed automatically with the AI Control Tower Core/Pro Plus SKU. It inherits the AI asset lifecycle, playbook workflows, and CMDB integration that AI Control Tower provides, extending that governance framework specifically to cross-platform agent interoperability.

AI Gateway is platform-agnostic. It governs MCP Server connections from ServiceNow AI Agent Studio and from external platforms including Microsoft Copilot Studio, Google Gemini, Anthropic Claude, VS Code, and any platform implementing the MCP specification. The AI Steward has visibility into all connections, maintaining oversight across the enterprise regardless of where agents were built.

## Key Capabilities

| Capability | What It Does |
| --- | --- |
| Centralized MCP Registry | Discover, register, and manage all MCP Server connections through a single governed asset inventory — sourced from AI Agent Studio sync, the Anthropic MCP Catalog, or manual registration. |
| AI Steward Approvals | MCP Servers are reviewed using Asset Approval Playbooks in AI Control Tower before agents can use them. All decisions are documented and auditable. |
| Runtime Security Controls | AI Stewards can pause MCP traffic globally or per-server without code changes. PII Vault Service integration blocks payloads containing sensitive data before they reach the MCP Server. |
| OAuth 2.1 Authentication | All MCP Server connections are authenticated centrally through AI Gateway using OAuth 2.1. Agents never connect to MCP Servers directly. |
| Observability | Usage, latency, and connection health metrics are available at the server level (AI Gateway dashboard) and tool level (KPIs & Metrics tab on individual MCP Server records). |

## Pillars of AI Gateway

AI Gateway is structured around four pillars that reflect how enterprises need to manage MCP Server connections at scale. These pillars operate concurrently — not as sequential setup steps, but as ongoing governance capabilities that are active throughout the lifecycle of every MCP Server connection.

---

## How to Use This in Implementation

Implementers should treat AI Gateway as a feature set within AI Control Tower rather than a separate install — it is provisioned automatically with the AI Control Tower Core/Pro Plus SKU and reuses the existing AI asset lifecycle, playbook workflows, and CMDB integration. To govern cross-platform agents, register MCP Server connections in the centralized registry (via AI Agent Studio sync, the Anthropic MCP Catalog, or manual registration), then route them through Asset Approval Playbooks for AI Steward sign-off before agents can use them. Configure OAuth 2.1 so all connections authenticate centrally through the Gateway (agents never connect to MCP Servers directly), enable PII Vault Service integration and global/per-server traffic pause for runtime security, and monitor usage, latency, and health via the AI Gateway dashboard (server level) and the KPIs & Metrics tab (tool level). These capabilities are concurrent governance functions, not one-time setup steps.
