## What Is This?

This subtopic covers the **technical and functional prerequisites** for the AI Gateway integration — specifically **release compatibility** and the **required plugins and dependencies**. Use it to confirm an environment is on a supported ServiceNow release/SKU before configuring AI Gateway / MCP governance.

---

## Release Compatibility

AI Gateway is **installed automatically with the AI Control Tower Core/Pro Plus SKU (`sn_awh_config`)**. **No separate activation is required.**

| Store App Version | Platform Compatibility |
| --- | --- |
| AWH for AI Control Tower 1.0.5 (AI Control Tower Core 5.0.0) | Zurich Patch 4+, Yokohama Patch 11+ |
| AWH for AI Control Tower 2.0.0 — Q1 2026 (AI Control Tower Core 6.1.0) | Australia release family, Zurich Patch 7+ |

---

## Required Plugins and Dependencies

- **AI Control Tower Core/Pro Plus SKU** — auto-installs `sn_awh_config`
- `sn_ai_governance` version **6.0.1 or higher** (Q1 2026)
- `sn_aict_nowassist` version **3.0.0 or higher** (Q1 2026)

---

## How to Use This in Implementation

Before configuring the AI Gateway:

1. Confirm the instance is on a **supported platform release** — Zurich Patch 4+/Yokohama Patch 11+ for AWH 1.0.5, or the Australia release family / Zurich Patch 7+ for the Q1 2026 AWH 2.0.0 build.
2. Verify the **AI Control Tower Core/Pro Plus SKU** is licensed — it auto-installs `sn_awh_config`, so AI Gateway needs no separate activation.
3. Ensure the dependency plugins meet the minimum versions: `sn_ai_governance` ≥ 6.0.1 and `sn_aict_nowassist` ≥ 3.0.0 (both Q1 2026).

If any version is below the minimum, upgrade before attempting MCP Server approval/routing configuration, since the governance and Now Assist sync features depend on these plugin versions.
