## What Is This?

Amazon Data Mapping defines how AI asset and usage data discovered from Amazon (AWS) sources is routed within ServiceNow AI Control Tower. It maps each data source utility to its staging table and final target table, so that discovered Bedrock, AgentCore, and SageMaker assets land in the correct AI Control Tower records. This is part of the Discover - Discovery flow for the AI Service Graph Connector for Amazon.

## How to Use This in Implementation

When configuring the AI Service Graph Connector for Amazon, use this mapping to understand where discovered data flows. Each data source utility extracts a specific asset type (Bedrock AI assets, systems, models, tools, prompts, subcomponents, usage; AgentCore systems, tools, usage; and SageMaker models and model cards) into an intermediate staging table prefixed `sn_ai_disc_aws_sgc_`. The staged records are then transformed into the corresponding target tables — AI systems, models, prompts, tools, subcomponent relationships, and usage records. Use the staging table names to troubleshoot discovery results and verify that records are populated before they reach the target AI Control Tower tables.
