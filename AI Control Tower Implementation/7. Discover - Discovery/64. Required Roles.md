# Required Roles

> Section: Discover - Discovery → Required Roles

---

## What Is This?

This subtopic lists the roles required to use the AI Service Graph Connector for Salesforce, which discovers AI assets and usage data from Salesforce into ServiceNow AI Control Tower. These roles authorize a user to configure and run the Salesforce connector discovery. They are a prerequisite for setting up the Salesforce connection during the Discover phase of an AICT implementation.

---

## Required Roles for the AI Service Graph Connector for Salesforce

The following roles are required:

- `sn_ai_disc.discovery_admin`
- `sn_cmdb_int_util.sgc_admin`

---

## How to Use This in Implementation

Before configuring the AI Service Graph Connector for Salesforce, grant the implementing user both the `sn_ai_disc.discovery_admin` and `sn_cmdb_int_util.sgc_admin` roles. With these roles assigned, the user can create the Salesforce connection (using the required Salesforce connection details and OAuth credentials) so that AI Control Tower can discover Salesforce AI agents, Einstein models, tools, prompts, and usage information.
