## What Is This?

AWS Prerequisites defines what your organization must have in place before creating an Amazon AI connection with the AI Service Graph Connector for Amazon. It covers the required AWS account, IAM credentials, API access, and the specific IAM permissions needed for each supported AWS service. This step is a precondition for discovering AI assets and usage data from AWS services into ServiceNow AI Control Tower.

---

## AWS Prerequisites

Before creating an Amazon AI connection, confirm that your organization has:

- An active AWS account.
- IAM credentials with the required read permissions.
- API access enabled for the AWS services being connected, including Amazon Bedrock, Amazon SageMaker, Amazon CloudWatch, and Amazon Bedrock AgentCore, as applicable.

---

## Required IAM Permissions

| AWS Service | Required Permissions |
| --- | --- |
| Amazon Bedrock | `bedrock:List*`, `bedrock:Get*` |
| Amazon SageMaker | `sagemaker:List*`, `sagemaker:Describe*` |
| Amazon CloudWatch | `logs:DescribeLogGroups`, `logs:DescribeLogStreams`, `cloudwatch:GetMetricData` |
| Amazon Bedrock AgentCore | `bedrock:ListAgents`, `bedrock:GetAgent` |

---

## How to Use This in Implementation

Before configuring the AI Service Graph Connector for Amazon, an implementer must verify that an active AWS account exists and that IAM credentials with the listed read permissions are available. Enable API access for each AWS service in scope (Amazon Bedrock, Amazon SageMaker, Amazon CloudWatch, and Amazon Bedrock AgentCore as applicable), and grant the corresponding IAM permissions from the table for each service to be discovered. Meeting these prerequisites is required prior to creating the Amazon AI connection.
