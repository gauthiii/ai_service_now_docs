## What Is This?

This subtopic documents the Microsoft APIs used by AI Control Tower to discover and collect usage data from Azure AI Foundry. It lists each API call, the Azure endpoint it targets, and the purpose it serves during discovery and usage collection. These APIs enable AICT to inventory Foundry projects, deployments, agents, and conversations as part of the discovery process.

---

## Microsoft APIs Used — Azure Foundry (Discovery and Usage)

| API | Endpoint | Purpose |
|-----|----------|---------|
| List Projects | Azure Management API | Lists Azure AI Foundry projects in a resource group |
| List Deployments | Azure AI Services project deployment endpoint | Lists model deployments within a Foundry project |
| List Agents | Azure AI Services agents endpoint | Lists agents in a Foundry project |
| List Agent Versions | Azure AI Services agent versions endpoint | Lists versions of a specific agent |
| List Conversations | Azure AI Services conversations endpoint | Lists conversation threads in a Foundry project |
| List Conversation Items | Azure AI Services conversation items endpoint | Lists messages or items within a conversation |

---

## How to Use This in Implementation

When configuring AICT discovery against Azure AI Foundry, ensure the integration has access to the Azure Management API and the relevant Azure AI Services endpoints listed above. The discovery flow proceeds hierarchically: list projects in a resource group, then deployments, agents (and their versions), and finally conversations and conversation items. This sequence builds the Foundry inventory and usage data that AICT records. Note that "AI Services Classic" is a separate (neighboring) API set used for the classic Azure AI Services experience and is not part of this Foundry subtopic.
