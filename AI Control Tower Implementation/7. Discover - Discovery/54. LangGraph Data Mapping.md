---

## What Is This?

LangGraph Data Mapping describes how the AI Service Graph Connector for LangGraph maps discovered LangGraph and LangSmith records into ServiceNow AI Control Tower tables. After a LangGraph connection is created, AI Control Tower discovers LangGraph agents and usage data, then maps those records into AI digital asset, usage, and CMDB tables. This mapping defines which data source flows through which import set table into which target table.

---

## LangGraph Data Mapping

The following table maps each LangGraph data source to its import set table and target table(s).

| Data Source | Import Set Table | Target Table |
|---|---|---|
| SG-LangGraph Agents | `sn_langgraph_integ_agents` | `alm_ai_system_digital_asset`; `alm_ai_prompt_digital_asset`; `cmdb_ci_function_ai` |
| SG-LangGraph Usage | `sn_langgraph_integ_usage` | `alm_ai_model_digital_asset`; `sn_ai_disc_ai_usage` |

---

## LangGraph Target Tables

The following table lists each target table and its purpose.

| Table | Purpose |
|---|---|
| `alm_ai_system_digital_asset` | Stores AI system digital assets |
| `alm_ai_prompt_digital_asset` | Stores AI prompt digital assets |
| `alm_ai_model_digital_asset` | Stores AI model digital assets |
| `sn_ai_disc_ai_usage` | Stores AI usage and execution data |
| `cmdb_ci_function_ai` | Stores AI function or agent CI records |

---

## How to Use This in Implementation

When implementing the AI Service Graph Connector for LangGraph, use this mapping to understand where discovered records land. Agent records from the `sn_langgraph_integ_agents` import set populate the AI system digital asset, AI prompt digital asset, and AI function/agent CI (CMDB) tables. Usage records from the `sn_langgraph_integ_usage` import set populate the AI model digital asset and AI usage/execution tables. The usage information mapped into these tables is consumed by the AI Control Tower value dashboard. This mapping follows connection creation and depends on the prerequisites being met (a LangSmith API key with workspace access, and for self-hosted environments, the appropriate LangSmith API and host API URLs).
