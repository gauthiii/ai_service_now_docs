## What Is This?

After the Amazon AI connection is created, AI Control Tower can import AWS AI asset metadata and usage data into the AI inventory and related usage tables. This subtopic lists the AWS APIs that AI Control Tower calls during discovery, mapped to the ServiceNow action names that invoke them. It documents which Amazon Bedrock services are queried to populate the inventory.

## How to Use This in Implementation

These action-to-API mappings define exactly which AWS Bedrock APIs AI Control Tower invokes after the Amazon AI connection is established, so the IAM permissions granted to the connection's credentials must include each listed API (for example, GetAgent, ListAgents, ListAgentAliases, ListAgentVersions, GetAgentActionGroup, ListAgentActionGroups, ListAgentCollaborators, GetInferenceProfile, and GetFoundationModel). The `sgawsbedrock_test_connection` action (which calls ListAgents) can be used to validate connectivity. Ensure the connection's role has permission for every API in these tables so that agent, foundation model, and inference profile metadata import successfully into the AI inventory and usage tables.
