## What Is This?

This subtopic lists the LangSmith REST APIs that the AI Service Graph Connector for LangGraph invokes to discover AI assets and usage data into ServiceNow AI Control Tower. These endpoints retrieve workspaces, agent deployments, tracer sessions, and run statistics from LangSmith. It is part of the LangGraph connector configuration within the Discover - Discovery flow.

---

## LangSmith APIs Used

| API | Endpoint Example | Purpose |
|-----|------------------|---------|
| List Workspaces | `https://api.smith.langchain.com/api/v1/workspaces/` | Lists workspaces available to the API key or user |
| List Deployments | `https://api.host.langchain.com/v2/deployments` | Lists agent deployments for a workspace |
| List Tracer Sessions | `https://api.smith.langchain.com/api/v1/sessions` | Lists tracer sessions for a workspace |
| Get Run Stats | `https://api.smith.langchain.com/api/v1/runs/stats` | Retrieves run statistics, including LLM invocations |

---

## How to Use This in Implementation

When configuring the AI Service Graph Connector for LangGraph, the LangSmith API key must provide access to the workspaces where AI assets and usage data should be discovered. These endpoints are called by the connector to enumerate workspaces, deployments, tracer sessions, and run statistics. For self-hosted LangSmith environments, confirm the appropriate LangSmith API and host API URLs, since the endpoint hosts (for example, `api.smith.langchain.com` and `api.host.langchain.com`) may differ from the SaaS defaults.
