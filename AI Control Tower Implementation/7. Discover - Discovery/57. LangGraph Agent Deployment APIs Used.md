## What Is This?

This subtopic lists the LangGraph deployment-level APIs that AI Control Tower uses to discover agent assets within each LangGraph deployment. These APIs enumerate the assistants and threads associated with a given deployment endpoint, supporting the discovery of LangGraph agent data into AI Control Tower.

---

## LangGraph Agent Deployment APIs

| API | Endpoint Example | Purpose |
| --- | --- | --- |
| Search Assistants | `{{endpoint_url}}/assistants/search` | Lists assistants in each deployment |
| Search Threads | `{{endpoint_url}}/threads/search` | Lists threads for each deployment |

The endpoint examples use a `{{endpoint_url}}` placeholder, indicating that the deployment-specific endpoint URL is substituted at runtime for each LangGraph deployment.

---

## How to Use This in Implementation

When configuring LangGraph discovery in AI Control Tower, these per-deployment APIs are invoked against each deployment's resolved `{{endpoint_url}}`. After deployments are enumerated, the Search Assistants API retrieves the assistants within a deployment, and the Search Threads API retrieves the threads within that deployment. Ensure the configured API key or connection has access to each deployment so these endpoints return the assistant and thread records for import.
