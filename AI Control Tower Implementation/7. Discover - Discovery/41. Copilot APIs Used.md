## What Is This?

This subtopic lists the Microsoft Copilot (Copilot Studio / Dataverse) APIs that the AI Service Graph Connector uses to discover and import Copilot AI assets into AI Control Tower. These APIs retrieve Copilot Studio bot metadata, their authoring components, and conversation transcripts. They form part of the discovery layer that feeds the AI Control Tower inventory and dashboards.

---

## Copilot APIs Used

| API | Endpoint | Purpose |
|-----|----------|---------|
| List Agents | Dataverse bots endpoint | Returns metadata for Copilot Studio bots registered in a Dataverse environment |
| List Components per Agent | Dataverse botcomponents endpoint | Returns authoring components such as topics, entities, variables, and trigger phrases |
| List Conversations | Dataverse conversationtranscripts endpoint | Retrieves conversation transcripts between users and Copilot Studio bots |

---

## How to Use This in Implementation

When configuring the AI Service Graph Connector to discover Microsoft Copilot assets, these three Dataverse endpoints are the integration points used to pull data. Implementers should ensure the connector has access to the target Dataverse environment so it can call the bots endpoint (to inventory registered Copilot Studio bots), the botcomponents endpoint (to capture authoring components per agent), and the conversationtranscripts endpoint (to retrieve usage/conversation records). The imported data follows the standard AI Service Graph Connector flow — landing in staging tables, then transform maps, then target tables — to populate the AI Control Tower inventory and dashboards.
