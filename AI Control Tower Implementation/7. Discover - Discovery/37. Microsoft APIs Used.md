## What Is This?

This subtopic documents the Microsoft (Azure) APIs that AI Control Tower Discovery uses to discover and collect usage data from Azure AI services. It lists the specific APIs, their endpoints, and their purpose, grouped by Azure AI Foundry and AI Services Classic. Implementers use this to understand which external Azure endpoints Discovery calls when inventorying Microsoft AI assets and their usage.

---

## Azure Foundry — Discovery and Usage

| API | Endpoint | Purpose |
|-----|----------|---------|
| List Projects | Azure Management API | Lists Azure AI Foundry projects in a resource group |
| List Deployments | Azure AI Services project deployment endpoint | Lists model deployments within a Foundry project |
| List Agents | Azure AI Services agents endpoint | Lists agents in a Foundry project |
| List Agent Versions | Azure AI Services agent versions endpoint | Lists versions of a specific agent |
| List Conversations | Azure AI Services conversations endpoint | Lists conversation threads in a Foundry project |
| List Conversation Items | Azure AI Services conversation items endpoint | Lists messages or items within a conversation |

---

## AI Services Classic

| API | Endpoint | Purpose |
|-----|----------|---------|
| List Projects | Azure Management API | Lists AI Services projects in a resource group |
| List Deployments | Azure AI Services deployment endpoint | Lists model deployments within a project |
| List Assistants | Azure AI Services assistants endpoint | Lists assistants or agents in a project |
| List Threads | Azure AI Services threads endpoint | Lists execution threads in a project |

---

## How to Use This in Implementation

When configuring AI Control Tower Discovery against Microsoft Azure, ensure the integration has access to these Azure endpoints. Discovery uses the Azure Management API to enumerate projects (both Foundry and AI Services Classic), then drills into each project via the Azure AI Services endpoints to list deployments, agents/assistants, and conversation/execution threads. Use the Azure Foundry set for Foundry-based projects and the AI Services Classic set for classic AI Services projects; the corresponding permissions for these endpoints must be granted so Discovery can collect both inventory and usage/execution data.
