## What Is This?

This subtopic lists the Azure Machine Learning Services / AI Hub APIs that the AI Service Graph Connector uses to discover Microsoft Azure AI assets. Each API is paired with its Azure endpoint and the purpose it serves during discovery. These calls let AI Control Tower inventory Azure subscriptions, resource groups, ML workspaces, deployments, assistants, threads, and thread runs.

---

## ML Services / AI Hub APIs

| API | Endpoint | Purpose |
|-----|----------|---------|
| List Subscriptions | Azure Management API | Lists available Azure subscriptions |
| List Resource Groups | Azure Management API | Lists resource groups within a subscription |
| List ML Workspaces | Azure Machine Learning Services API | Lists Azure ML workspaces in a resource group |
| List Deployments | Azure AI Services deployment endpoint | Lists model deployments within an ML project |
| List Assistants | Azure ML agents endpoint | Lists agents or assistants from an Azure ML workspace |
| List Threads | Azure ML threads endpoint | Lists threads from an Azure ML workspace |
| List Thread Runs | Azure ML thread runs endpoint | Lists runs within a specific thread |

---

## How to Use This in Implementation

When configuring the AI Service Graph Connector for Microsoft Azure, these APIs define the discovery scope and the order of traversal: start at the subscription and resource group level (Azure Management API), then enumerate ML workspaces, and finally drill into deployments, assistants, threads, and thread runs. Ensure the connector has credentials with access to the Azure Management API, the Azure Machine Learning Services API, and the Azure AI Services deployment endpoints so each of these calls can return its inventory into AI Control Tower.
