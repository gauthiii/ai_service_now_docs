## What Is This?

AI Services Classic is one of the Microsoft API sets that AI Control Tower uses to discover and gather usage data from Microsoft Azure AI Services. It defines the specific Azure APIs and endpoints that AICT calls to enumerate projects, deployments, assistants, and execution threads within an Azure AI Services environment. This fits into the Discovery process as the integration surface for pulling Classic AI Services entities into AICT.

---

## Microsoft APIs Used — AI Services Classic

The following Microsoft APIs are used for AI Services Classic discovery and usage.

| API | Endpoint | Purpose |
|-----|----------|---------|
| List Projects | Azure Management API | Lists AI Services projects in a resource group |
| List Deployments | Azure AI Services deployment endpoint | Lists model deployments within a project |
| List Assistants | Azure AI Services assistants endpoint | Lists assistants or agents in a project |
| List Threads | Azure AI Services threads endpoint | Lists execution threads in a project |

---

## How to Use This in Implementation

When configuring AI Control Tower discovery for an Azure AI Services Classic environment, ensure connectivity to the listed endpoints (Azure Management API and the AI Services deployment, assistants, and threads endpoints). AICT calls these APIs in sequence — projects first, then deployments, assistants, and threads — to build out the discovered entity hierarchy. This API set is distinct from the Azure Foundry discovery and usage APIs documented on the same page; select the set that matches the Azure AI platform in use.
