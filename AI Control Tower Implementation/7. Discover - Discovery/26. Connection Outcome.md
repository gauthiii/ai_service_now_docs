## What Is This?

The Connection Outcome describes what AI Control Tower can do once the Azure Foundry connection has been successfully created. It defines the assets and data that become discoverable through the connection and provides the data mapping between Azure Foundry data sources and their corresponding ServiceNow staging and target tables.

---

## Connection Outcome

After the Azure Foundry connection is created, AI Control Tower can discover Azure Foundry agents, models, prompts, tools, subcomponents, and execution data.

---

| Data Source | Staging Table | Target Table |
|---|---|---|

---

## How to Use This in Implementation

Use this mapping to confirm that the Azure Foundry connection is populating the correct ServiceNow tables after discovery runs. Each Azure Foundry data source first lands in its staging table (prefixed `sn_azure_sg_foundry_*`) and is then mapped into a target table. The "SG-Azure Foundry AI Discovery" source acts as the parent data source for the other discovery records. When validating a connection, check that agents, models, prompts, tools, subcomponents, and execution/usage data appear in their respective target tables (`alm_ai_system_digital_asset`, `alm_ai_model_digital_asset`, `alm_ai_prompt_digital_asset`, `sn_ent_ai_tool`, `sn_ent_ai_system_subcomponent_m2m`, and `sn_ai_disc_ai_usage`). This requires the Azure Foundry prerequisites to be met and both Discovery and Execution import schedules to be enabled.
