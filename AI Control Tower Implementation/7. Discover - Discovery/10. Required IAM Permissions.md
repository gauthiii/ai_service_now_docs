# Required IAM Permissions

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Discover - Discovery → Required IAM Permissions
> Source: AI-Control-Tower.pdf, p.67

---

## What Is This?

This subtopic defines the AWS IAM permissions that the AI Service Graph Connector for Amazon needs in order to discover AI assets and usage data from AWS services into ServiceNow AI Control Tower. These read-level permissions are granted to the IAM credentials used by the Amazon AI connection. They are a prerequisite for establishing a working Amazon connection during discovery.

---

## Required IAM Permissions

The IAM credentials used for the Amazon AI connection must include the following permissions per AWS service:

| AWS Service | Required Permissions |
|---|---|
| Amazon Bedrock | `bedrock:List*`, `bedrock:Get*` |
| Amazon SageMaker | `sagemaker:List*`, `sagemaker:Describe*` |
| Amazon CloudWatch | `logs:DescribeLogGroups`, `logs:DescribeLogStreams`, `cloudwatch:GetMetricData` |
| Amazon Bedrock AgentCore | `bedrock:ListAgents`, `bedrock:GetAgent` |

---

## How to Use This in Implementation

When configuring IAM credentials for the AI Service Graph Connector for Amazon, attach a policy that grants the listed read permissions for each AWS service you intend to discover (Amazon Bedrock, Amazon SageMaker, Amazon CloudWatch, and Amazon Bedrock AgentCore, as applicable). These permissions align with the AWS prerequisites of having an active AWS account, IAM credentials with the required read permissions, and API access enabled for the connected services. Grant only the permissions for the services in scope for your discovery to follow least-privilege practices.
