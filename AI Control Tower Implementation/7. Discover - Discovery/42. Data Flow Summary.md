---

## What Is This?

The Data Flow Summary describes the standard, end-to-end path that data follows when the AI Service Graph Connector imports AI asset information from external platforms (such as AWS or Microsoft) into AI Control Tower. It clarifies how raw imported data moves through ServiceNow's import architecture before becoming inventory and dashboard records. This sits at the discovery stage of implementation, explaining where ingested AI data lands and how it is transformed for use.

---

## Standard Data Flow

AI Service Graph Connector data follows a standard flow:

```
External AI Platform → Data Source → Staging Table → Transform Map → Target Table → AI Control Tower Inventory and Dashboards
```

- **Staging tables** temporarily hold imported data from AWS or Microsoft sources.
- **Target tables** store the transformed AI asset, relationship, and usage records used across AI Control Tower.

---

## How to Use This in Implementation

When configuring AI Service Graph Connector discovery, expect imported data to first land in staging tables as raw records from the external AI platform's data source. A transform map then maps and converts these staging records into target tables. The target tables are the authoritative store for AI asset, relationship, and usage data that AI Control Tower's inventory and dashboards consume. Implementers should verify each stage (data source connectivity, staging population, transform map execution, and target table results) when troubleshooting discovery gaps.
