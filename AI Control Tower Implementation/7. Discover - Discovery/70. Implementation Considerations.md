## What Is This?

This subtopic lists the prerequisites and validation checks that organizations should complete before enabling connector-based discovery in AI Control Tower. It ensures roles, credentials, permissions, schedules, and data mappings are in place so that discovered AI assets are imported accurately into the AI inventory. It sits in the Discovery phase of the implementation, just before connector-based discovery is activated.

---

## Validation Before Enabling Connector-Based Discovery

Organizations should validate the following before enabling connector-based discovery:

- Required roles are assigned in ServiceNow.
- External platform credentials are available and securely managed.
- Required API permissions are granted.
- Discovery and execution schedules are reviewed and activated where needed.
- Usage data is available in the source platform.
- Imported records are validated in the AI inventory after the first scheduled import.
- Data mappings align with the expected AI asset lifecycle model in AI Control Tower.

## How to Use This in Implementation

Before turning on connector-based discovery, an implementer should work through the validation checklist: assign the required ServiceNow roles, securely manage external platform credentials, grant the necessary API permissions, and review/activate discovery and execution schedules. Confirm that usage data exists in the source platform and that data mappings match AI Control Tower's AI asset lifecycle model. After the first scheduled import, validate the imported records in the AI inventory. Use the platform-specific configuration specifications (AWS, Microsoft Azure Foundry, Microsoft Copilot) for the detailed connector requirements relevant to each source system.
