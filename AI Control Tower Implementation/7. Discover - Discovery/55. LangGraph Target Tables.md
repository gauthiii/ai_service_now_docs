## What Is This?

LangGraph Target Tables are the ServiceNow tables where data discovered through the AI Service Graph Connector for LangGraph is ultimately stored. When the LangGraph connection discovers agents and usage data, those records are mapped into AI digital asset, usage, and CMDB tables. This subtopic identifies each target table and its purpose.

---

## LangGraph Target Tables

| Table | Purpose |
|-------|---------|
| `alm_ai_system_digital_asset` | Stores AI system digital assets |
| `alm_ai_prompt_digital_asset` | Stores AI prompt digital assets |
| `alm_ai_model_digital_asset` | Stores AI model digital assets |
| `sn_ai_disc_ai_usage` | Stores AI usage and execution data |
| `cmdb_ci_function_ai` | Stores AI function or agent CI records |

---

## How to Use This in Implementation

After the LangGraph connection is created, AI Control Tower discovers LangGraph agents and usage data, then maps those records into these target tables. Implementers should be aware that AI system, prompt, and model assets land in the corresponding `alm_ai_*_digital_asset` tables, usage and execution data lands in `sn_ai_disc_ai_usage`, and AI function or agent configuration items are stored in `cmdb_ci_function_ai`. These tables are the destinations referenced by the LangGraph Data Mapping, where import set tables (e.g., `sn_langgraph_integ_agents` and `sn_langgraph_integ_usage`) feed data into them.
