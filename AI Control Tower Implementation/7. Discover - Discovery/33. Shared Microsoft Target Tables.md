## What Is This?

This subtopic identifies the ServiceNow target tables that both Azure Foundry and Copilot discovery use to store the AI inventory data they bring into AI Control Tower. Staging data from each Microsoft data source is ultimately written into these shared target tables. They are grouped into digital asset tables, entity tables, and a usage table.

---

## Shared Microsoft Target Tables

Both Azure Foundry and Copilot use the following target tables.

### Digital Asset Tables

| Table | Purpose |
|---|---|
| `alm_ai_system_digital_asset` | Stores AI system digital assets |
| `alm_ai_prompt_digital_asset` | Stores AI prompt digital assets |
| `alm_ai_model_digital_asset` | Stores AI model digital assets |

### Entity Tables

| Table | Purpose |
|---|---|
| `sn_ent_ai_tool` | Stores AI tools |
| `sn_ent_ai_system_subcomponent_m2m` | Stores AI system subcomponent relationships |

### Usage Table

| Table | Purpose |
|---|---|
| `sn_ai_disc_ai_usage` | Stores AI usage and execution data |

---

## How to Use This in Implementation

When configuring Microsoft (Azure Foundry or Copilot) discovery in AI Control Tower, data flows from the Microsoft data sources through staging tables into these shared target tables. Implementers should expect discovered AI systems, prompts, and models to land in the `alm_ai_*_digital_asset` tables, AI tools and subcomponent relationships in the `sn_ent_ai_*` entity tables, and usage/execution data in `sn_ai_disc_ai_usage`. Because both Azure Foundry and Copilot reuse the same target tables, records from either source are consolidated into a single inventory, which is useful for reporting and validation across both Microsoft platforms.
