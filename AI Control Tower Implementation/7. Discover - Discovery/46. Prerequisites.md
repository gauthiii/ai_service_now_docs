## What Is This?

This subtopic covers the prerequisites that must be completed before creating a Google Cloud Platform (GCP) Vertex AI connection for the AI Service Graph Connector. The AI Service Graph Connector for GCP Vertex AI discovers AI assets and usage data from Google Cloud Vertex AI into ServiceNow AI Control Tower. These prerequisites ensure the required Google Cloud setup is in place so the connection can successfully discover and import data.

---

## Prerequisites

Before creating a GCP Vertex AI connection, confirm that the organization has completed the required Google Cloud setup, including:

- Creating a service account.
- Assigning and binding the required roles.
- Enabling the required APIs.
- Creating a JKS file.
- Registering the JKS file in the ServiceNow instance.

Refer to the related ServiceNow setup instructions and API documentation for the detailed external platform requirements.

---

## How to Use This in Implementation

An implementer must ensure all five Google Cloud setup steps are finished before attempting to create the GCP Vertex AI connection. This includes establishing a service account, assigning and binding the required roles to it, enabling the necessary APIs in Google Cloud, generating a JKS file, and registering that JKS file in the ServiceNow instance. Detailed external platform requirements should be obtained from the related ServiceNow setup instructions and API documentation. Completing these prerequisites enables AI Control Tower to discover Vertex AI assets and related execution data for inventory, lifecycle visibility, and value dashboard reporting.
