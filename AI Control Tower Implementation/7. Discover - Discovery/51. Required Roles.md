## What Is This?

This subtopic lists the roles required to configure and run the AI Service Graph Connector for LangGraph, which discovers AI assets and usage data from LangGraph and LangSmith into ServiceNow AI Control Tower. An administrator must hold these roles before creating a LangGraph connection and performing discovery. It fits into the Discover phase of AI Control Tower implementation, where connectors bring external AI assets into the platform.

---

## Required Roles

The following roles are required for the AI Service Graph Connector for LangGraph:

- `sn_ai_disc.discovery_admin`
- `sn_cmdb_int_util.sgc_admin`

---

## How to Use This in Implementation

Before creating a LangGraph connection, ensure the implementing user is granted both `sn_ai_disc.discovery_admin` and `sn_cmdb_int_util.sgc_admin`. These roles authorize the administrator to set up the Service Graph Connector and run discovery that maps LangGraph agents and usage data into AI digital asset, usage, and CMDB tables. Granting these roles is a prerequisite step that precedes confirming the LangSmith API key and configuring the connection.
