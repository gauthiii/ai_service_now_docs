# AI Service Graph Connector for Microsoft

> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Discover - Discovery → AI Service Graph Connector for Microsoft
> Source: AI-Control-Tower.pdf, p.70

---

## What Is This?

The AI Service Graph Connector for Microsoft is a discovery connector that brings AI assets and usage data from Microsoft sources into ServiceNow AI Control Tower. It enables AICT to discover AI systems, agents, models, prompts, tools, subcomponents, and execution data from Azure Foundry and Microsoft Copilot. This connector is part of the Discover phase, populating the AI Control Tower with externally hosted AI assets.

---

## Purpose

The AI Service Graph Connector for Microsoft is used to discover AI assets and usage data from Azure Foundry and Microsoft Copilot into ServiceNow AI Control Tower. Supported assets may include AI systems, agents, models, prompts, tools, subcomponents, and execution data.

---

## Required Roles

- `sn_ai_disc.discovery_admin`
- `sn_cmdb_int_util.sgc_admin`

---

## How to Use This in Implementation

To use the AI Service Graph Connector for Microsoft, ensure the implementing user is assigned both the `sn_ai_disc.discovery_admin` and `sn_cmdb_int_util.sgc_admin` roles. The connector is configured to pull from Microsoft sources (Azure Foundry and Microsoft Copilot), with the Azure Foundry connection set up as the next configuration step. Once configured, it discovers AI systems, agents, models, prompts, tools, subcomponents, and execution/usage data into AI Control Tower.
