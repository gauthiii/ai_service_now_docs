## What Is This?

Azure Foundry Data Mapping defines how data discovered through the Azure Foundry connection flows from its source, through intermediate staging tables, and into the target tables used by AI Control Tower. It documents the staging and target table for each Azure Foundry data source, such as discovery, agents, models, prompts, tools, subcomponents, and execution data. This mapping is the reference for understanding where discovered Azure Foundry assets and execution data land within the AICT data model.

## How to Use This in Implementation

After the Azure Foundry connection is created, AI Control Tower can discover Azure Foundry agents, models, prompts, tools, subcomponents, and execution data. Use this mapping to trace each discovered data source to its staging table (the intermediate landing table) and its target table (where the records are ultimately stored). The AI Discovery source acts as the parent data source rather than mapping to a single target table. When validating or troubleshooting discovery results, check the corresponding staging table for raw imported data and the target table for the final records (for example, models land in `alm_ai_model_digital_asset` and execution data lands in `sn_ai_disc_ai_usage`).
