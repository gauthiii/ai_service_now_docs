## What Is This?

The AI Service Graph Connector for GCP Vertex AI discovers AI assets and usage data from Google Cloud Vertex AI into ServiceNow AI Control Tower. It supports discovery of AI systems, models, prompts, tools, system subcomponents, and execution data. This connector feeds inventory and usage information into AI Control Tower for lifecycle visibility and value dashboard reporting.

---

## Purpose

The AI Service Graph Connector for GCP Vertex AI is used to discover AI assets and usage data from Google Cloud Vertex AI into ServiceNow AI Control Tower. Supported assets include AI systems, models, prompts, tools, system subcomponents, and execution data.

Usage information imported through the connector is consumed by the AI Control Tower value dashboard.

---

## Required Roles

- `sn_ai_disc.discovery_admin`
- `sn_cmdb_int_util.sgc_admin`

---

## Prerequisites

Before creating a GCP Vertex AI connection, confirm that the organization has completed the required Google Cloud setup, including:

- Creating a service account.
- Assigning and binding the required roles.
- Enabling the required APIs.
- Creating a JKS file.
- Registering the JKS file in the ServiceNow instance.

Refer to the related ServiceNow setup instructions and API documentation for the detailed external platform requirements.

---

## Connection Outcome

After the GCP Vertex AI connection is created, AI Control Tower can discover Vertex AI assets and related execution data for inventory, lifecycle visibility, and value dashboard reporting.

## How to Use This in Implementation

An implementer first ensures the two required roles (`sn_ai_disc.discovery_admin` and `sn_cmdb_int_util.sgc_admin`) are granted. Before creating the connection, complete the Google Cloud setup: create a service account, assign and bind the required roles, enable the required APIs, create a JKS file, and register that JKS file in the ServiceNow instance. Once the connection is created, the connector populates the staging tables and maps discovered data into the listed target tables, making Vertex AI systems, models, prompts, tools, subcomponents, and execution/usage data available for inventory, lifecycle visibility, and the AI Control Tower value dashboard.
