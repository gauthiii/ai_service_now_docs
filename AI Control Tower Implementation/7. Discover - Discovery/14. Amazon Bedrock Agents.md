## What Is This?

Amazon Bedrock Agents is one of the AWS API groups used by AI Control Tower to discover and import AWS AI asset metadata and usage data into the AI inventory and related usage tables. After the Amazon AI connection is created, AICT uses these actions to call the corresponding AWS Bedrock Agent APIs. This subtopic lists the AICT action names mapped to their underlying AWS APIs.

---

## AWS APIs Used — Amazon Bedrock Agents

After the Amazon AI connection is created, AI Control Tower can import AWS AI asset metadata and usage data into the AI inventory and related usage tables.

| Action Name | AWS API |
|---|---|
| `sgawsbedrock_agent` | GetAgent |
| `sgawsbedrock_get_agent` | GetAgent |
| `sgawsbedrock_list_agent` | ListAgents |
| `sgawsbedrock_list_agent_aliases` | ListAgentAliases |
| `sgawsbedrock_list_versions` | ListAgentVersions |
| `sgawsbedrock_get_agent_action_group` | GetAgentActionGroup |
| `sgawsbedrock_list_agent_action_groups` | ListAgentActionGroups |
| `sgawsbedrock_list_agent_collaborators_stream` | ListAgentCollaborators |
| `sgawsbedrock_test_connection` | ListAgents |

---

## How to Use This in Implementation

These actions are invoked by AI Control Tower once the Amazon AI connection has been established. The mapping shows which AWS Bedrock Agent API each AICT action calls — for example, `sgawsbedrock_test_connection` uses `ListAgents` to validate connectivity. Ensure the configured AWS credentials/role have permission to call the listed Bedrock Agent APIs (GetAgent, ListAgents, ListAgentAliases, ListAgentVersions, GetAgentActionGroup, ListAgentActionGroups, ListAgentCollaborators) so that agent metadata can be imported into the AI inventory. The Amazon Bedrock Foundation and Amazon Bedrock AgentCore Control groups are separate, neighboring action sets and are documented elsewhere.
