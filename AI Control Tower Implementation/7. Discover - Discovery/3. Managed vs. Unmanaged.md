## What Is This?

This subtopic explains the two categories that AI assets in the AI Assets Inventory fall into — managed and unmanaged — and how this status governs which AI Control Tower (AICT) capabilities apply to a given asset. It also describes the AI Assets API used to retrieve, update, and create AI assets out of the box. Understanding managed vs. unmanaged status is foundational to controlling which workflows and assessments run against an AI asset record during implementation.

---

## AI Assets API and Asset Inventory

The following APIs are available with OOTB capabilities to integrate with the AI Assets Inventory. AI Assets API provides endpoints to retrieve, update, and create several types of AI assets, such as AI systems, AI data sets, prompts, and models.

## Managed vs. Unmanaged Categories

AI assets in the inventory fall into two categories: managed and unmanaged. By default, assets in AI Control Tower are unmanaged — all assets created via the AI Assets API land as unmanaged. When an asset is marked managed, it gains access to AICT capabilities including governance, lifecycle management, value assessment, risk classification, security, and privacy; reverting to unmanaged removes those capabilities.

Only AI stewards can move assets between states. The AI Assets API operates on AI asset records regardless of managed/unmanaged status; the managed flag controls which workflows and assessments run against the record, not API accessibility.

## AI Assets API Requirements and Behavior

The AI Assets API supports actions that you can perform on AI Asset records in the Expanded Model and Asset Classes application. It requires the Asset Classes (sn_ent) plugin to access it. You must have the asset and model_manager roles to call the endpoints provided by the AI Assets API.

The data included in AI Asset records may be spread across several tables like Product Model, Configuration Item, and others. To reduce complexity when calling this API, however, the AI Assets API inserts data into only the Asset, Product Model, and Configuration Item tables. Specifics about table updates are provided under each endpoint.

## How to Use This in Implementation

When implementing the AI Assets Inventory, treat newly created assets as unmanaged by default — including any asset created through the AI Assets API. To enable AICT capabilities (governance, lifecycle management, value assessment, risk classification, security, and privacy) on an asset, an AI steward must explicitly mark it as managed; reverting it to unmanaged removes those capabilities. To use the AI Assets API, ensure the Asset Classes (sn_ent) plugin is activated and that the calling user holds both the asset and model_manager roles. Note that the API writes only to the Asset, Product Model, and Configuration Item tables to reduce complexity, even though AI Asset data can span additional tables.
