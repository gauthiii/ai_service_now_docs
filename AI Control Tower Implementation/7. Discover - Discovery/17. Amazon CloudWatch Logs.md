## What Is This?

This subtopic lists the discovery actions that AI Control Tower uses to discover and query Amazon CloudWatch Logs as part of AWS Discovery. Each action maps to a specific AWS API, allowing AICT to start log queries, retrieve query results, and validate connectivity. It fits into the broader AWS provider discovery flow alongside other Amazon services such as AgentCore and Amazon SageMaker.

---

## Action Name to AWS API Mappings

| Action Name | AWS API |
| --- | --- |
| `look_up_cloudwatch_startquery` | StartQuery |
| `sgawscloudwatch_startquery` | StartQuery |
| `sgawscloudwatch_getqueryresults` | GetQueryResults |
| `look_up_cloudwatch_getquery` | GetQueryResults |
| `sgawscloudwatch_test_connection` | StartQuery |
| `sgaws_agentcore_cloudwatch_test_connection` | StartQuery |

---

## How to Use This in Implementation

When configuring AWS Discovery for Amazon CloudWatch Logs, ensure the AWS credentials and IAM permissions used by AICT grant access to the underlying CloudWatch Logs APIs referenced here: `StartQuery` and `GetQueryResults`. The `*_test_connection` actions rely on `StartQuery` to validate connectivity before running full discovery. Both `look_up_*` and `sgawscloudwatch_*` action variants exist for starting queries and retrieving results, so verify that the action set expected by your discovery pattern aligns with the permissions granted.
