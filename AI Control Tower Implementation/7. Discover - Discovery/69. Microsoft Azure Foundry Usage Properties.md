## What Is This?

This subtopic defines the system properties that control how the Microsoft Azure Foundry connector retrieves usage data during scheduled discovery imports. These properties govern lookback behavior — how far back the connector reaches for thread/usage data on regular runs and on the very first run. They fit within the Discover - Discovery phase as part of the "Additional Connector Properties" configuration for connector-based discovery.

---

## Microsoft Azure Foundry Usage Properties

| Property | Description | Default Value |
|----------|-------------|---------------|
| `sn_ai_msft_integ.usage_data_lookback` | Number of days to look back for fetching threads before the run collection window start time | `3` |
| `sn_ai_msft_integ.usage_first_run_lookback_days` | Number of days to look back for usage data on the first run when no `last_success_import_time` exists | `30` |

Both properties are stored in the System Property `[sys_properties]` table.

## How to Use This in Implementation

When configuring the Microsoft Azure Foundry connector for usage/execution data discovery, review and (if needed) adjust these two system properties in the `[sys_properties]` table:

- Set `sn_ai_msft_integ.usage_data_lookback` (default `3`) to control how many days before the collection window start time the connector fetches threads on each run.
- Set `sn_ai_msft_integ.usage_first_run_lookback_days` (default `30`) to control how much historical usage data is pulled on the initial run, which applies only when no `last_success_import_time` exists.

As a dependency, ensure Discovery is run before Execution so that asset records exist before usage records are imported, and confirm the appropriate roles and Azure OAuth credentials are in place per the connector's configuration specifications.
