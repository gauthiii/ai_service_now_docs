## What Is This?

This subtopic lists the ServiceNow roles required to configure and run the AI Service Graph Connector for Google Cloud Platform (GCP) Vertex AI. These roles grant the permissions needed to set up the connector that discovers Vertex AI assets and usage data into AI Control Tower. Assigning them is a prerequisite step before creating a GCP Vertex AI connection.

---

## Required Roles

The following roles are required for the AI Service Graph Connector for Google Cloud Platform Vertex AI:

- `sn_ai_disc.discovery_admin`
- `sn_cmdb_int_util.sgc_admin`

---

## How to Use This in Implementation

Before configuring the AI Service Graph Connector for GCP Vertex AI, ensure the implementing user is granted both `sn_ai_disc.discovery_admin` and `sn_cmdb_int_util.sgc_admin`. These roles enable the discovery administration and Service Graph Connector administration capabilities needed to create the connection and import Vertex AI assets and usage data into AI Control Tower.
