> ServiceNow AI Control Tower (AICT) — Implementation Guide
> Section: Discover - Discovery → Copilot Data Mapping

---

## What Is This?

Copilot Data Mapping defines how data discovered through the Microsoft Copilot connection is moved into AI Control Tower. It maps each Copilot data source to its corresponding staging table and target table, so discovered Copilot agents, models, prompts, tools, subcomponents, and execution data land in the correct AICT records. This mapping is the reference that explains where each category of Copilot data is stored after discovery.

---

## Copilot Data Mapping

Mapping of Copilot data sources to their staging and target tables:

| Data Source | Staging Table | Target Table |
|---|---|---|
| SGC-Copilot Discovery | `sn_msft_copilot_ai_agent_staging` | Parent data source |
| SGC-Copilot AI System | `sn_msft_copilot_ai_agent_asset_staging` | `alm_ai_system_digital_asset` |
| SGC-Copilot AI Model | `sn_msft_copilot_ai_agent_model_staging` | `alm_ai_model_digital_asset` |
| SGC-Copilot AI Prompt | `sn_msft_copilot_ai_agent_prompt_staging` | `alm_ai_prompt_digital_asset` |
| SGC-Copilot AI Tools | `sn_msft_copilot_ai_agent_tool_staging` | `sn_ent_ai_tool` |
| SGC-Copilot AI Subcomponents M2M | `sn_msft_copilot_ai_agent_sbcomp_m2m_staging` | `sn_ent_ai_system_subcomponent_m2m` |
| SGC-Copilot Execution | `sn_msft_copilot_ai_agent_usage_staging` | `sn_ai_disc_ai_usage` |

---

## How to Use This in Implementation

Use this mapping to validate that Copilot discovery is populating the expected records after the Copilot connection is created. Each Copilot data source first lands in its staging table and is then promoted into the corresponding target table. When troubleshooting missing Copilot agents, models, prompts, tools, subcomponents, or usage data, inspect the relevant staging table to confirm the data was imported before it reached the target table. The "SGC-Copilot Discovery" source is the parent data source from which the other sources derive.
