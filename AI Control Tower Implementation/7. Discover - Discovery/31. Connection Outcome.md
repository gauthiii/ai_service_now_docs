## What Is This?

This subtopic describes what AI Control Tower can discover after a Microsoft Copilot connection is created. Once the Copilot connection is in place, AI Control Tower discovers Copilot agents, models, prompts, tools, subcomponents, and execution data. It also defines the Copilot data mapping that routes discovered data from staging tables into target tables.

---

## Connection Outcome

After the Copilot connection is created, AI Control Tower can discover Copilot agents, models, prompts, tools, subcomponents, and execution data.

---

## Copilot Data Mapping

| Data Source | Staging Table | Target Table |
|---|---|---|
| SGC-Copilot Discovery | `sn_msft_copilot_ai_agent_staging` | Parent data source |
| SGC-Copilot AI System | `sn_msft_copilot_ai_agent_asset_staging` | `alm_ai_system_digital_asset` |
| SGC-Copilot AI Model | `sn_msft_copilot_ai_agent_model_staging` | `alm_ai_model_digital_asset` |
| SGC-Copilot AI Prompt | `sn_msft_copilot_ai_agent_prompt_staging` | `alm_ai_prompt_digital_asset` |
| SGC-Copilot AI Tools | `sn_msft_copilot_ai_agent_tool_staging` | `sn_ent_ai_tool` |
| SGC-Copilot AI Subcomponents M2M | `sn_msft_copilot_ai_agent_sbcomp_m2m_staging` | `sn_ent_ai_system_subcomponent_m2m` |
| SGC-Copilot Execution | `sn_msft_copilot_ai_agent_usage_staging` | `sn_ai_disc_ai_usage` |

---

## How to Use This in Implementation

After completing the Copilot connection prerequisites and creating the connection, run the Discovery scheduled job (before the Execution scheduled job) to populate the staging tables. AI Control Tower then maps each data source from its staging table into the corresponding target table per the mapping above, making Copilot agents, models, prompts, tools, subcomponents, and execution data visible in AI Control Tower. Use the data mapping table to verify which staging and target tables to inspect when troubleshooting missing discovered Copilot data.
