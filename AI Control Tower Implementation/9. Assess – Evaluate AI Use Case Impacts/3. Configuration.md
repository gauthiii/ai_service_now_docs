## What Is This?

This subtopic covers how to configure and customize the Flow and associated Sub-Flows that drive the Assess phase of the AI use case lifecycle. It points implementers to the governing reference for modifying these flows and provides a safety guideline for changing system-defined flows. It fits at the configuration step of the Assess phase, after the assessment tasks and approval records have been generated.

---

## Configuration

A. For more details to modify Flow and associated Sub-Flows belonging to the Assess phase, refer to the **Govern - Lifecycle** topic.

> **Note**

## How to Use This in Implementation

When customizing the Assess phase, do not edit the system-defined Flows and Sub-Flows directly. Instead, create a copy version of the flow/sub-flow before making changes or corrections, so that platform upgrades do not overwrite or conflict with your modifications. For the detailed procedure on how to modify these flows, consult the Govern - Lifecycle topic, which is the authoritative reference for lifecycle flow configuration.
