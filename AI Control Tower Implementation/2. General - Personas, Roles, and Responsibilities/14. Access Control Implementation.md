## What Is This?

This subtopic provides guidance for configuring access control for AI Control Tower roles. It covers implementation considerations and leading practices for assigning roles, building Access Control Lists (ACLs), and enforcing data protection within AICT. It fits into implementation as the practical guidance an administrator follows when operationalizing the personas and system roles defined for AICT.

---

## Guidance

The following guidance applies when configuring access control for AI Control Tower roles.

### Implementation Considerations

- Provision multiple out-of-the-box roles to user groups (e.g., assign AI System Reader to a Compliance Reader group).
- Modify ACLs sparingly to maintain upgradeability and ensure compatibility with future AICT features.

### Leading Practices

- Apply the principle of least privilege.
- Assign roles to groups, not individual users.
- Use the condition builder and scripts for complex scenarios.
- Reuse logic using script includes for consistency.
- Comment custom ACL logic clearly and document their purpose and associated roles.
- Use field-level ACLs for sensitive data; use table-level ACLs for general control.
- Avoid duplicating ACLs for the same operation.
- Conduct thorough access testing by user type and scenario.
- Always enforce data protection with server-side ACLs—do not rely on client-side scripts or policies.

---

## How to Use This in Implementation

When configuring access for AICT, assign the out-of-the-box system roles to user groups rather than to individual users, applying least privilege. Use the condition builder, scripts, and reusable script includes for complex access scenarios, and document any custom ACL logic with its purpose and associated roles. Prefer field-level ACLs for sensitive data and table-level ACLs for general control, avoid duplicating ACLs for the same operation, and always enforce protection server-side. Modify ACLs sparingly to preserve upgradeability, and validate the configuration through thorough access testing across user types and scenarios.
